Troubleshooting

Common issues and solutions when running Jawcode. If your problem is not listed here, check the logs at ~/.jwc/logs/ or open an issue on GitHub.

Bun Version Compatibility

Symptom

Error: Jawcode requires Bun >= 1.2.x but found 1.1.x

Cause

Jawcode is built on Bun and requires a minimum version (currently >=1.2, as declared in package.json engines.bun). Older Bun versions lack APIs that Jawcode depends on, such as Bun.file() enhancements, Bun.stringWidth(), and stable shell execution.

Fix

# Update Bun to the latest version
curl -fsSL https://bun.sh/install | bash

# Verify
bun --version

If you manage Bun via a version manager (e.g. proto, mise), update through that tool instead.

API Key Authentication Failures

Symptom

Error: 401 Unauthorized — invalid api key
Error: No provider keys found. Set at least one API key.

Cause

The API key is missing, expired, malformed, or set for the wrong provider. Jawcode auto-discovers providers by checking environment variables at startup.

Fix

  1. Verify the key is exported in your shell: 
    echo $ANTHROPIC_API_KEY   # should print sk-ant-...
echo $OPENAI_API_KEY      # should print sk-...
  2. Ensure the key is in the correct variable. Anthropic keys start with sk-ant-; OpenAI keys start with sk-. Putting an Anthropic key in OPENAI_API_KEY will fail.
  3. Check that the key has not expired or been revoked in your provider dashboard.
  4. If using OAuth (ANTHROPIC_OAUTH_TOKEN), note that it takes precedence over ANTHROPIC_API_KEY.
  5. Run jwc --list-models to see which providers are active. If your provider is missing, its key is not being read.

Context Overflow Errors

Symptom

Error: context window exceeded (N tokens > limit)
Warning: conversation compacted — context was approaching the limit

Cause

The conversation has grown beyond the model's context window. Jawcode uses auto-compaction to summarize older messages, but extremely long sessions or large file reads can still overflow.

Fix

  1. Let compaction work. Jawcode automatically compacts when approaching the limit. If you see the warning, the system is handling it.
  2. Start a new session for unrelated tasks instead of continuing the same conversation indefinitely.
  3. Read file sections instead of entire files. Use read file.ts:50-100 instead of read file.ts for large files.
  4. Use a model with a larger context window. Check jwc --list-models for available context sizes.
  5. Resume from a previous session with jwc --resume — the compacted state carries forward.

Theme Loading Issues

Symptom

Warning: theme "my-theme" not found, falling back to default
Error: failed to parse theme file

Cause

Custom theme JSON is malformed, or the theme name does not match any file in ~/.jwc/agent/themes/. The built-in themes are abyss-bite (default for Jawcode) and red-claw (upstream default).

Fix

  1. Check theme file location: 
    ls ~/.jwc/agent/themes/
  2. Validate JSON syntax: 
    cat ~/.jwc/agent/themes/my-theme.json | jq .
  3. Set the theme in your config: 
    # ~/.jwc/agent/config.yml
theme: abyss-bite
  4. If using a custom theme, ensure it follows the same key structure as the built-in themes. Missing keys fall back to the default palette.

Model Not Found

Symptom

Error: model "gpt-5-turbo" not found
Error: no models available for provider "openai"

Cause

The requested model ID does not match any entry in the model registry, or the provider for that model has no valid API key. Jawcode maintains a registry of 3,600+ models across 46 providers, but model IDs must be exact or match a recognized alias.

Fix

  1. List available models: 
    # List all models
jwc --list-models

# Search for a specific model
jwc --list-models gpt-4
  2. Use the canonical model ID shown in the list output. Fuzzy matching works for well-known aliases, but if discovery fails, use the full provider/model-id format: 
    jwc --model anthropic/claude-sonnet-4-20250514
  3. Ensure the provider's API key is set. A model will not appear in the list if its provider has no credentials.

Permission Denied (Bash Tool)

Symptom

ToolError: Command blocked by restricted role-agent bash allowlist.
ToolError: Command blocked

Cause

Role agents (architect, planner, critic) run with a restricted bash allowlist. They can only execute sanctioned workflow CLI commands like jwc orchestrate and jwc state. General shell commands are blocked to enforce their read-only contract.

The main agent and executor subagents have broader bash access but may still be blocked by the command interceptor if a command matches a known-dangerous pattern (e.g., rm -rf /).

Fix

  1. Role agents: These agents are intentionally restricted. If you need shell access, delegate to an executor subagent instead.
  2. Main agent: If a legitimate command is being blocked, check the interceptor rules. The agent will ask for confirmation before running destructive commands.
  3. Custom interceptors: Project-level .jwc/config.yml can define command interception rules. Review your project config if commands are unexpectedly blocked.

Common Error Messages

ErrorCauseSolution
ECONNREFUSED Provider API is unreachable (network, firewall, VPN). Check network connectivity. Try curl https://api.anthropic.com.
Rate limit exceeded Too many API requests in a short period. Wait and retry. Consider switching to a provider with higher limits.
ENOMEM / out of memory Large file operations or too many concurrent subagents. Reduce parallelism. Close other applications. Increase swap.
Session file corrupted Incomplete write during crash or power loss. Start a new session with jwc. Old sessions are in ~/.jwc/agent/sessions/.
TUI rendering corrupted Terminal does not support required capabilities, or console.log was used in a tool. Resize the terminal (Ctrl+L to repaint). Ensure your terminal supports 256 colors and Unicode.
Puppeteer: browser closed Chrome process crashed or was killed externally. The browser tool will auto-relaunch. If persistent, set PUPPETEER_EXECUTABLE_PATH to a known-good Chrome.
natives: binary not found Pre-built native binaries not available for your platform. Run cd crates/pi-natives && cargo build --release to build from source, or check ~/.jwc/natives/ for cached binaries.

Collecting Debug Information

When reporting issues, include:

  1. Bun version: 
    bun --version
  2. Jawcode version: 
    jwc --version
  3. Log files: Located at ~/.jwc/logs/gjc.YYYY-MM-DD.log. Include the log from the day the issue occurred.
  4. Crash log: If the process crashed, check ~/.jwc/agent/gjc-crash.log.
  5. Debug log: For verbose output, check ~/.jwc/agent/gjc-debug.log.
  6. Environment: OS, terminal emulator, and shell. Redact API keys before sharing.
