MCP Integration

Jawcode supports the Model Context Protocol (MCP) for integrating external tool servers. MCP servers provide additional tools that the agent can call during a session, extending Jawcode's built-in 40+ tool surface with domain-specific capabilities.

Architecture

Jawcode's MCP integration has two distinct surfaces:

SurfaceDirectionPurpose
Client-side MCP Jawcode calls external servers Add tools from external MCP servers (Context7, computer-use, custom servers)
Coordinator bridge External coordinators call Jawcode Hermes-style orchestration of Jawcode sessions via jwc mcp-serve coordinator

Intentional isolation

The SDK surface intentionally isolates MCP runtime discovery. createAgentSession only uses an explicitly passed options.mcpManager and never performs its own discovery. The CLI path (main.ts) wires discovery for interactive sessions only.

Configuring MCP servers

MCP server configuration lives in JSON files at two scopes:

Project-level

# Either of these paths:
.jwc/mcp.json
.jwc/.mcp.json

User-level

# Either of these paths:
~/.jwc/agent/mcp.json
~/.jwc/agent/.mcp.json

Configuration format

{
  "mcpServers": {
    "context7": {
      "command": "npx",
      "args": ["-y", "@upstash/context7-mcp@latest"],
      "enabled": true
    },
    "my-custom-server": {
      "command": "node",
      "args": ["./tools/my-mcp-server.js"],
      "env": {
        "API_KEY": "..."
      },
      "enabled": true
    }
  }
}

Managed defaults

Running jwc setup defaults restores managed default MCP servers to the user-level configuration. Currently, the only managed default is context7 (Context7 documentation server).

jwc setup defaults

Runtime behavior

Startup and connection

  1. MCP servers are discovered from project and user config files.
  2. All enabled servers are connected in parallel via stdio transport.
  3. A startup timeout of 250ms applies. Servers that do not respond within this window are aborted (unless tool definitions are cached).
  4. Connected server tools are injected as custom tools into the session.

Tool caching

Server tool definitions (name, description, schema) are cached in agent.db after first successful connection. On subsequent startups, cached definitions allow the agent to proceed even if the server is slow to start. The cache is invalidated when the server configuration changes.

Schema normalization

Tool input schemas from MCP servers are normalized via normalizeSchemaForMCP. Note that $ref inlining (dereferencing) is not performed — schemas with broken $ref references will cause provider 400 errors at call time.

Conflict detection

If an MCP server provides a tool with the same name as a built-in tool, the MCP tool is skipped and an error is logged. Built-in tools always take precedence.

Discoverable tools

Jawcode prevents tool overexposure through a discovery and BM25 selection layer. Not all tools are presented to the model at once:

  • Always-loaded tools — Core tools (read, edit, bash, etc.) are always in the tool list.
  • Discoverable tools — Less common tools are hidden until the agent searches for them via search_tool_bm25. Once activated, they remain available for the session.

This applies to both built-in and MCP-provided tools. The built-in computer_use tool, for example, is discoverable rather than always-loaded.

Coordinator bridge

The coordinator bridge exposes Jawcode sessions as MCP tools for external orchestrators:

jwc mcp-serve coordinator

This is the outward MCP adapter for Hermes-style coordination. It provides tools to:

  • List and manage sessions
  • Start worktree/tmux-oriented sessions
  • Queue bounded follow-up prompts
  • Read status, tail output, and artifacts
  • Handle structured questions
  • Write coordination reports

Setup

jwc setup hermes \
  --root /path/to/repo \
  --profile my-bot \
  --repo jawcode \
  --mutation sessions,questions,reports \
  --profile-dir /path/to/hermes/profile \
  --install

Safety model

The bridge is read-only and fail-closed by default. See Permissions for the full safety model.

Smoke check

# Non-mutating contract verification
jwc setup hermes --root /path/to/repo --smoke

# JSON contract check
jwc mcp-serve coordinator --check --json
