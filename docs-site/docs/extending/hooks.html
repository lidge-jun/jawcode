Hooks & Plugins

Hooks and plugins extend Jawcode's behavior through lifecycle events and modular packages. Hooks are single TypeScript modules that react to session events. Plugins are structured packages that can bundle tools, hooks, extensions, commands, and settings together.

Hooks

What hooks can do

A hook module exports a default function that receives a hook API object. The API provides event subscription, message injection, custom rendering, and command registration:

API methodPurpose
on(event, handler)Subscribe to lifecycle events
sendMessage(msg)Inject a message into the conversation
appendEntry(entry)Append an entry to the session log
registerMessageRenderer(renderer)Add a custom message renderer for the TUI
registerCommand(cmd)Register a slash command from the hook
exec(cmd)Execute a shell command
loggerStructured logger instance
typeboxTypeBox schema builder (for validation)
zodZod schema builder (alternative validation)
piInternal runtime utilities

Creating a hook

A hook is a TypeScript file that exports a default function:

// .jwc/hooks/my-hook.ts
export default function(api) {
  api.on("session:start", () => {
    api.logger.info("Session started!");
  });

  api.on("message:sent", (msg) => {
    // React to messages sent by the user
  });

  api.registerCommand({
    name: "my-command",
    description: "Does something useful",
    handler: async (args) => {
      api.sendMessage({
        role: "user",
        content: `Running my-command with: ${args}`
      });
    }
  });
}

Hook requirements

  • Must export a default function
  • Must be a TypeScript (.ts) file — .md and .json files are rejected
  • Must not throw during initialization (errors are caught and logged)

Hook discovery

Hooks are discovered from two sources:

  1. Capability API discovery — Standard capability scan of .jwc/hooks/ (project) and ~/.jwc/agent/hooks/ (user)
  2. Explicit configured paths — Paths listed in settings under hook configuration

Both sources are merged. Hooks from explicit paths supplement (not replace) discovered hooks.

Custom tools

Custom tools are TypeScript modules that add new tool definitions to the agent's tool surface:

// .jwc/tools/my-tool.ts
export default function(api) {
  return {
    name: "my_tool",
    description: "Does something specific",
    inputSchema: api.typebox.Object({
      input: api.typebox.String()
    }),
    handler: async ({ input }) => {
      return { result: `Processed: ${input}` };
    }
  };
}

Custom tool API

The factory function receives an API object with:

FieldPurpose
cwdCurrent working directory
execShell command execution
uiTUI rendering utilities
loggerStructured logger
typeboxTypeBox schema builder
zodZod schema builder
piInternal runtime utilities

Conflict detection

If a custom tool has the same name as a built-in tool, it is skipped and an error is logged. Built-in tools always take precedence over custom tools.

Plugins

Plugins are structured packages that bundle multiple extension types together. A plugin is a directory with a manifest that declares its components:

Plugin manifest

{
  "name": "my-plugin",
  "version": "1.0.0",
  "tools": ["./tools/analyzer.ts"],
  "hooks": ["./hooks/logger.ts"],
  "extensions": ["./ext/sidebar.ts"],
  "commands": ["./commands/deploy.md"],
  "features": {
    "auto-format": true,
    "notifications": false
  },
  "settings": {}
}

Plugin locations

ScopePath
User-level~/.jwc/agent/gjc-plugins/
Project-level{cwd}/.jwc/gjc-plugins/

A directory is recognized as a plugin root if it contains a plugin manifest at the root or in a child directory.

Installed plugin record

Each installed plugin tracks:

  • name — Plugin identifier
  • version — Semantic version
  • path — Filesystem path
  • manifest — Parsed manifest object
  • enabledFeatures — Map of feature flags
  • enabled — Whether the plugin is active

Slash commands

Custom slash commands are Markdown files discovered from .jwc/commands/ directories:

# .jwc/commands/deploy.md
Deploy the current branch to staging.

Steps:
1. Run the test suite
2. Build the production bundle
3. Deploy to the staging environment

Invoke with /deploy in a session. The Markdown content is injected as context for the agent.

Built-in slash commands

Jawcode includes built-in slash commands registered in a declarative registry:

  • /settings — Open settings
  • /theme — Switch TUI theme
  • /goal — Goal management
  • /model — Model selection (2-pane keyboard UI)
  • /memory — Memory management
  • /provider — Provider configuration
  • /login — Authentication
  • /help — Docked 2-pane command catalog
  • /fork, /branch, /resume, /sessions — Session management

Extension loading

Extensions are native capability modules and plugin extension paths merged by the extension loader. They follow the same capability API discovery pattern as skills and hooks, scanning project and user directories for loadable modules.
