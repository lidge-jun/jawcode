Providers Overview

Jawcode connects to 46 providers and exposes over 3,600 models out of the box. Every provider speaks one of the supported transport APIs; you pick a model and Jawcode handles the wire format, auth, streaming, and tool-call translation.

Built-in Provider Categories

Providers fall into three categories based on how models are registered and discovered:

First-Class Providers

Built into the static catalog with full model metadata (context window, cost, thinking levels, compatibility flags). No user configuration needed beyond auth:

  • Anthropic — Claude Opus, Sonnet, Haiku families
  • OpenAI — GPT-5.x, o3, o4-mini families
  • OpenAI Codex — Codex-optimized models with WebSocket transport
  • Google — Gemini 2.x families (AI Studio, Vertex, Antigravity)
  • xAI — Grok 3, Grok Composer families
  • Amazon Bedrock — Cross-provider models via AWS
  • Azure OpenAI — OpenAI models via Azure deployments

OpenAI-Compatible Providers

Providers that expose an OpenAI-compatible API. Built-in entries exist for many, and you can add any compatible endpoint as a custom provider:

  • Groq, Together, Fireworks, Cerebras — fast inference platforms
  • DeepSeek, Mistral, Cohere, Minimax — specialized model providers
  • Perplexity, Kagi — search-augmented providers
  • OpenRouter, Vercel AI Gateway — multi-provider routing
  • Any proxy with /v1/chat/completions compatibility

Local/Discovery Providers

Automatically discovered on localhost without explicit configuration. Jawcode probes the default port and registers models it finds:

  • Ollama — http://127.0.0.1:11434
  • llama.cpp — http://127.0.0.1:8080
  • LM Studio — http://127.0.0.1:1234/v1

Model Behavior Patch Layers

When a model misbehaves in Jawcode, fixes are applied at one of four layers. Lower layers have broader scope; higher layers are more precise:

LayerWhat It FixesLocation
1. Catalog Model specs, compat flags, context windows, cost packages/ai/src/models.json
2. Request Transformer Wire parameter shaping per provider Provider-specific *.ts files
3. System Prompt Injection Behavioral correction for specific models composer-discipline.ts, cursor.ts
4. Host/Tool Defense Model-agnostic structural guards bash.ts, stream.ts, etc.

Principle: spec errors go to layers 1-2, behavioral habits go to layer 3, universal pitfalls go to layer 4.

Tokenizer Routing

Jawcode maps each provider to a BPE tokenizer family for accurate context counting and compaction. The mapping chain:

  1. resolveTokenizerFamily(model) — provider-to-family lookup
  2. resolveModelEncoding — family to native @gajae-code/natives encoding
  3. Lazy download for uncommon families

Codex Transport

OpenAI Codex uses a dedicated transport layer distinct from standard OpenAI-compatible providers:

  • WebSocket primary, SSE fallback — automatic failover
  • Watchdog — 300s idle floor matching native Codex client behavior
  • Content prewarming — generate:false prefill to amortize cold starts (measured 2m15s in production)
  • Rate-limit telemetry — captures codex.rate_limits push events, shows utilization in the status footer
  • Transport visibility — ws/sse/sse! markers in the status line

Unlisted Model Policy

Discovery-based providers may expose models that exist only in the static catalog (not returned by the provider's /v1/models endpoint). Instead of dropping these, Jawcode tags them as unlisted: hidden by default in selectors but available on explicit request. This prevents selector pollution while preserving access to catalog-only models like xai/grok-composer-2.5-fast.
