Model Configuration

Jawcode's model system handles selection, resolution, role assignment, thinking levels, canonical equivalence, and context promotion. This page covers the full configuration surface.

Model Selection Syntax

The --model flag and /model TUI command accept several selector forms:

SelectorExampleBehavior
Exact provider/model anthropic/claude-sonnet-4-6 Bypasses coalescing, selects the exact provider variant
Canonical ID anthropic-model-opus-4-6 Resolves through canonical index to the best available variant
Bare model ID claude-sonnet-4-6 Provider inferred from available models
Fuzzy/substring sonnet Substring matching after exact paths fail
Glob pattern openai/*, *sonnet* Used in --models and enabledModels
With thinking level anthropic/claude-sonnet-4-6:high Appended :off|minimal|low|medium|high|xhigh

Thinking Levels

Models that support extended thinking expose configurable reasoning effort levels. Set per-model in models.yml or as a selector suffix: 

# In models.yml
models:
  - id: claude-sonnet-4-6
    thinking:
      minLevel: low
      maxLevel: xhigh
      mode: effort        # "effort" or "budget"
      defaultLevel: high
      levels: [low, medium, high, xhigh]
# As selector suffix
jwc --model anthropic/claude-sonnet-4-6:high
jwc --model openai/o3:medium

Available levels: off, minimal, low, medium, high, xhigh. Not all models support all levels — the minLevel/maxLevel range in the catalog constrains valid selections.

Model Roles

Jawcode assigns models to roles. Each role can be pinned to a specific model in config.yml or models.yml:

RolePurpose
defaultMain conversation model
executorDefault subagent for parallel implementation
executor_extExternal/fresh model lane for independent verification
architectRead-only architecture review
plannerRead-only sequencing and planning
criticRead-only plan critique
 
# In models.yml
modelBindings:
  modelRoles:
    default: anthropic/claude-sonnet-4-6:high
  agentModelOverrides:
    executor: openai/gpt-5.2
    executor_ext: openai/o3:high
    architect: openai/o3:high

Model Profiles (--mpreset)

Profiles are pre-configured role-to-model mappings. Activate with --mpreset for a session or --default to persist: 

jwc --mpreset codex-standard
jwc --mpreset opencode-go-pro --default

Built-in Profiles

Profile GroupTiers
opencode-go-*eco, standard, pro
codex-*eco, standard, pro
opencode-go-codex-*eco, standard, pro
custom-1 through custom-4Empty slots for user override

eco favors cheaper/faster defaults, standard matches production defaults, pro raises reasoning for architect/critic/planner roles. Custom profiles defined in models.yml override built-ins by exact name.

Custom Profile Definition

# In models.yml
profiles:
  team-standard:
    required_providers: [openai, anthropic]
    model_mapping:
      default: openai/gpt-5.2
      executor: anthropic/claude-sonnet-4-6:medium
      architect: openai/o3:high
      planner: openai/o3:high
      critic: openai/o3:high

required_providers lists providers that must have credentials before the profile activates. Activation hard-blocks when any listed provider lacks auth.

wireModelId

wireModelId changes only the model ID in the upstream request body. Local selection still uses the configured provider/id. This is useful for proxies that expect a different model name than your local selector: 

providers:
  my-proxy:
    baseUrl: https://proxy.example/v1
    api: openai-completions
    models:
      - id: local-name
        wireModelId: upstream-model-v2

Canonical Model Equivalence

Multiple concrete provider variants (e.g., anthropic/claude-sonnet-4-6 and bedrock/anthropic.claude-sonnet-4-6) can share a canonical ID. When you select a canonical ID, Jawcode picks the best available concrete variant based on:

  1. Availability and auth
  2. modelProviderOrder in config
  3. Registry insertion order (if no explicit order)

Equivalence Configuration

# In models.yml
equivalence:
  overrides:
    zenmux/openai-code: gpt-5.3-openai-code
    p-openai-code/openai-code: gpt-5.3-openai-code
  exclude:
    - demo/openai-code-preview

Build order for canonical grouping: explicit user override, then bundled official-ID matches, then conservative heuristic normalization, then fallback to the concrete model's own ID.

Context Promotion

Context promotion is an overflow recovery mechanism for small-context model variants (e.g., *-spark). When a turn fails with a context length error, Jawcode can automatically promote to a larger-context sibling before falling back to compaction. 

# In models.yml model entry
models:
  - id: spark-model
    contextWindow: 32000
    contextPromotionTarget: full-model

Context promotion is off by default (contextPromotion.enabled: false). Opt in via settings to enable automatic overflow recovery.

Per-Model Compatibility Flags

The compat block controls provider-specific request behavior. Set at the provider level (baseline) or per-model (override): 

models:
  - id: my-model
    compat:
      supportsStore: true
      supportsDeveloperRole: true
      supportsReasoningEffort: false
      maxTokensField: max_completion_tokens
      reasoningContentField: reasoning_content
      openRouterRouting:
        only: [anthropic]
      extraBody:
        gateway: m1-01

compat is deep-merged for nested blocks (openRouterRouting, vercelGatewayRouting, extraBody). Model-level compat overrides provider-level keys.

Cache Retention

Control prompt caching behavior per provider, per model, or per request: 

# Provider level
providers:
  openai:
    cacheRetention: short    # none | short | long

# Model override
    modelOverrides:
      gpt-5.2:
        cacheRetention: long

Priority order: request-time options > model/modelOverride > provider > JWC_CACHE_RETENTION env > runtime default. For OpenAI Responses, this controls prompt_cache_retention only.

Scoped Model Visibility

Restrict which models are available based on working directory: 

# In config.yml
enabledModels:
  - anthropic-model-sonnet-4-5          # available everywhere
  - path: ~/work
    models:
      - anthropic/anthropic-model-opus-4-5

disabledProviders:
  - ollama                              # disabled everywhere
  - path: ~/private
    providers:
      - anthropic

String entries apply globally. Scoped entries apply when the CWD matches the configured path or any subdirectory.
