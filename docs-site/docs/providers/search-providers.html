Search Providers

Jawcode's web_search tool routes through a pluggable search provider system. 16 search engines are supported, from OAuth-unlocked native search to API-key-only services. The search provider is independent of the active model — you can use Claude with Codex search, or Grok with DuckDuckGo.

Three-Layer Architecture

The search surface has three distinct layers:

  1. Provider layer — engine-specific implementation + resolution chain
  2. Selection surface — providers.webSearch setting + /searchengine slash command
  3. Availability gating — OAuth/API key checks per engine

Provider Resolution Chain

When web_search is invoked, resolveProviderChain determines which engine to use:

  1. Explicitly selected provider (via /searchengine) if available
  2. Active model's native search engine (if auto mode and engine is available)
  3. DuckDuckGo as terminal fallback — always appended, requires no credentials

Keyed-only providers are never auto-selected — they require explicit /searchengine selection.

The /searchengine Command

Switch search providers in the TUI without restarting: 

# Check status of all search engines
/searchengine status

# Switch to a specific engine
/searchengine tavily
/searchengine brave
/searchengine codex

# Reset to auto (model-native) mode
/searchengine auto

xAI Grok — Integrated Web + X Search

The xAI search provider is unique: it sends both web_search and x_search tools to the Grok Responses API in a single round-trip. Grok decides which sources to use per query — it searches both the general web and the X (Twitter) live index simultaneously.

Verified live tools on the xAI Responses API:

  • web_search — general web search
  • x_search — X/Twitter live index
  • collections_search — collection search
  • file_search — file content search
  • code_exec — code execution
  • function — function calling

Auth: xAI OAuth, hasAuth("xai"), or XAI_API_KEY. Citations use url_citation annotations normalized to SearchSource[].

SearXNG Self-Hosted Setup

For privacy-focused or air-gapped environments, point Jawcode at your own SearXNG instance: 

# Set the endpoint
export SEARXNG_ENDPOINT=http://localhost:8888

# Select it
/searchengine searxng

SearXNG requires no API key — only the endpoint URL. It aggregates results from multiple search engines on the server side.

Response Format

All 16 search providers normalize their responses to a common SearchResponse shape: 

interface SearchResponse {
  answer?: string;           // Direct answer (if provider generates one)
  sources: SearchSource[];   // Ranked source list
  citations?: string[];      // Optional citation URLs
}

interface SearchSource {
  title: string;
  url: string;
  snippet: string;
}

This normalization means the agent sees a consistent format regardless of which search engine is active.
