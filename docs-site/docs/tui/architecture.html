TUI Architecture

Jawcode's terminal UI is a scrollback-native differential renderer. The frame is the entire transcript; the terminal scrollback buffer is the primary history store. Scroll up and you see real history, not a virtual reconstruction.

Core Constraint

Terminals cannot un-scroll. Once a line enters the scrollback buffer, it can only be erased (ESC[2K), never retrieved. This single physical constraint drives every design decision in the scroll model:

  • Frame shrinkage always leaves residual blank lines
  • Where those blanks go and when they're cleaned determines scroll UX
  • 3J (scrollback wipe) is permanently forbidden once any line is committed

Frame Layout (B2-lite)

The frame stacks top-to-bottom in this order:

[ Welcome Banner ]              ← scrolls naturally into scrollback
[ ViewportFill ]                ← stretches to fill remaining viewport height
[ chatContainer ]               ← committed transcript cells (monotonic growth)
[ pendingMessages ]             ¦
[ liveToolContainer ]           ¦ ← live zone: active tool previews
[ statusContainer (loader) ]    ¦
[ todo / btw / statusLine ]     ¦ composer cluster — always pinned to terminal bottom
[ hookAbove / editor / hookBelow ] ⌋

The fill region sits above the chat, not below. This is the B2-lite design: content and slash output cluster around the input (bottom), while shrinkage delta is absorbed by the top margin. Shrinkage only happens before content overflows the viewport.

ViewportFill Mechanism

ViewportFill is the spacer that keeps the composer pinned to the bottom when transcript content is shorter than the terminal viewport.

PhaseLocationBehavior
Sentinel emit viewport-fill.ts Emits VIEWPORT_FILL_SENTINEL (1 line; 0 if disabled for legacy compat)
Expand tui.ts #expandViewportFill After tree render, before overlay compositing. Replaces sentinel with max(0, target - contentLines) blank lines
Sticky gap #viewportFillGap / #viewportFillFloor On shrink: gap grows to keep composer at bottom (one full-render). On growth: gap freezes — frame grows from bottom, preserving append-only diff
Gap tracking #viewportFillGap Counts blank lines accumulated by floor
Compact compactViewportFill() Resets floor/gap + requestRender(true) — reprints the full transcript to rebuild scrollback without blanks. No-op when gap is 0

Compact Triggers

  • Prompt submit — input-controller.ts:457, after commitFinalizedBacklog
  • Slash dispatch — input-controller.ts:328, when slashResult === true
  • Ctrl+O toggle — current-turn output expand/collapse

Ctrl+O toggles only the current live turn; previously committed turns are unaffected. Ctrl+T opens the full transcript overlay (not a compact trigger).

Viewport Repaint Policy

When a change occurs outside the visible viewport, the renderer chooses a physical path based on isViewportAtBottom():

isViewportAtBottom()PathRationale
true append + viewportRepaint Terminal is bottom-locked; safe to append growth lines
false viewportRepaint only User is reading history; don't force-scroll to bottom
undefined viewportRepaint only Real terminal state unknown; speculative CRLF append risks duplicate/pushed rows

viewportRepaint (tui.ts:1353) reprints the visible N rows in-place without 2J/3J. The cursor moves to screen top, each row is erased (2K) and rewritten. Scrollback pixels are untouched.

fullRender(true) uses 2J+H to clear the screen and reprint the entire frame. Scrollback is rebuilt identically, so history is preserved. Triggered by width changes, clearOnShrink, or explicit requestRender(true).

3J Prohibition

buffer += isMultiplexerSession() || this.#hasCommittedHistory
    ? "\x1b[2J\x1b[H"       // 3J omitted
    : "\x1b[2J\x1b[H\x1b[3J";

Once #hasCommittedHistory is true (any line was committed to scrollback), 3J is permanently forbidden. The scrollback is now the canonical transcript and must not be wiped. Multiplexers (tmux/zellij) also suppress 3J since users scroll their own pane history.

Commit Lane (Dual-Lane Architecture)

Jawcode separates rendering into two lanes:

LaneDescriptionDefault
Virtual lane Finalized cells added to chatContainer; the differential renderer reprints every frame Always active (fallback)
Commit lane Finalized cells written to scrollback exactly once; the renderer never touches them again ON — opt-out with JWC_COMMIT_LANE=0

commitLines() Entry Conditions

Commit lane entry requires:
  #historyLane === "standard"           // not Zellij or dumb terminal
  && !stopped && terminalAvailable
  && overlayStack.length === 0          // no overlay open
  && liveZoneTop > 0                    // fill region exists (history area)
  && lines.length > 0

On success, #committedScreenRows increments and #hasCommittedHistory becomes true (3J permanently forbidden). On failure, the caller falls back to the virtual lane.

EnvironmentLane ModeReason
TERM=dumb unsupported No ANSI escape support

Everything else (incl. tmux) standard tmux pane history receives committed lines as intended

Scrollback Commit Protocol

buildInsertHistorySequence() (insert-history.ts:56) runs a 2-phase commit:

  1. Phase 1 (conditional): If the live zone doesn't touch the screen bottom, Reverse Index (\x1bM) pushes the live zone down, creating empty history rows above.
  2. Phase 2 (always): Sets a scroll region from row 1 to liveZoneTop. For each line, \r\n scrolls the region up (pushing the top line into real scrollback since top=1), \x1b[2K cleans the freed row, then the line content is written.

Critical: only scroll regions with top=1 push lines into actual scrollback. A region starting at row > 1 discards scrolled-off lines instead of preserving them.

Commit-Time Folding

The tool.renderMode setting controls how tool output enters history:

ModeBehavior
commit (jwc default) Active tools preview in liveToolContainer only. On completion, they enter chatContainer as setMinimized(true) — one-line summary. History grows monotonically; tool-induced shrinkage is eliminated.
verbose (legacy) Tool previews stream inline in the transcript. Post-completion folding relies on ViewportFill safety nets.

Ctrl+O sweeps only liveToggleEligible children in the chat and live zone. It preserves the thinking-expanded coupling (thinkingExpanded).
