Lineage

Jawcode does not exist in isolation. It is built on a family of MIT-licensed agent harnesses, each contributing a distinct piece of DNA to the final product.

gajae-code

gajae-code is the direct upstream fork source. Jawcode inherits its core architecture from this repo:

  • Multi-provider LLM client (packages/ai) with streaming support
  • Agent runtime with tool calling and state management (packages/agent)
  • Terminal UI library with differential rendering (packages/tui)
  • Native text/image/grep bindings via NAPI-RS (packages/natives, crates/pi-natives)

The internal @gajae-code/* package namespace originates from this upstream repo and is preserved for compatibility.

oh-my-pi

oh-my-pi is the upstream implementation DNA — the original agent harness that established the architectural patterns gajae-code built upon. Key contributions:

  • Tool-calling agent loop with streaming model responses
  • Theme system and TUI rendering architecture
  • Provider abstraction layer and model routing

oh-my-codex and oh-my-claudecode

Two experimental branches by Yeachan-Heo:

  • oh-my-codex — Codex-focused orchestration experiments, exploring structured multi-step workflows with OpenAI models.
  • oh-my-claudecode — Claude Code workflow exploration, investigating Anthropic-specific agent patterns and tool conventions.

cli-jaw

The cli-jaw harness contributed the core IPABCD orchestration loop that defines Jawcode's workflow identity. Key contributions from cli-jaw:

  • IPABCD pipeline — the six-stage structured workflow engine (Interview, Plan, Audit, Build, Check, Done) with 3-way reject routing at the Check stage.
  • HITL/HOTL modes — Human-in-the-Loop and Human-on-the-Loop execution modes with configurable approval gates between stages.
  • Workflow skills — the skill system architecture including jaw-interview, plan, goal, and team as composable workflow definitions.
  • jawcode/sdk embedding surface — the SDK interface that allows cli-jaw and other harnesses to embed Jawcode as a managed dependency.
  • Role agent design — the executor/architect/planner/critic subagent roles with prompt-cache-aware fork/spawn/resume lifecycle.
  • De-anchoring protocol — correction-resistant subagent behavior that generates fresh hypotheses instead of patching prior conclusions.

What Jawcode adds

Beyond its upstream lineage, Jawcode contributes:

  • 46-provider model routing — support for 3,600+ models across Anthropic, OpenAI, Google, xAI, Groq, DeepSeek, Ollama, and 39 more providers.
  • Abyss-bite design language — dark blue/cyan TUI theme with gradient sweep welcome banner and animated shine effects.
  • Goal mode — durable execution ledger with evidence checkpoints and quality-gate completion contracts.
  • Team mode — tmux-backed coordinated execution with worker lifecycle management and shared state.
  • Prompt hardening — de-anchoring, authority-bias prevention, blast-radius assessment, sunk-cost guards, and confirmation-bias countermeasures across all subagent prompts.
  • 40+ tools — read, edit, search, AST grep/edit, browser, task delegation, debug, eval, and more.
