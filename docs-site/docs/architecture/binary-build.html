Binary Build

Jawcode compiles to a single self-contained binary using bun build --compile. The output is a portable executable that bundles the TypeScript source, worker entrypoints, and native addons into one file.

Build Command

The build script lives at packages/coding-agent/scripts/build-binary.ts and produces the binary at packages/coding-agent/dist/jwc. 

bun run build:binary
# or directly:
bun packages/coding-agent/scripts/build-binary.ts

Build Pipeline

The build runs through several stages in sequence:

StepCommandPurpose
1. Stats bundle bun --cwd=../stats scripts/generate-client-bundle.ts --generate Generate the stats dashboard client bundle for embedding
2. Native embed bun --cwd=../natives run embed:native Stage native N-API addons for compile-time bundling
3. Compile bun build --compile ... Compile TypeScript to a single binary with embedded workers
4. Stage addons stageWorkspaceNativeAddons() Copy native .node files alongside the binary
5. Code sign codesign --force --sign - dist/jwc Ad-hoc sign on macOS (Bun 1.3.12+ workaround for truncated Mach-O signatures)
6. Cleanup embed:native --reset + stats --reset Restore native and stats packages to development state

Compile Flags

The bun build invocation uses these flags:

FlagPurpose
--compileProduce a standalone executable
--no-compile-autoload-bunfigSkip bunfig.toml to avoid dev-only config
--no-compile-autoload-dotenvSkip .env files
--no-compile-autoload-tsconfigSkip tsconfig resolution
--no-compile-autoload-package-jsonSkip package.json resolution
--keep-namesPreserve function/class names for stack traces
--define process.env.PI_COMPILED="true" Compile-time constant for isCompiledBinary() checks
--external mupdfExclude mupdf (optional native dependency)
--root ../..Set monorepo root so worker paths resolve correctly

Entrypoints

The main entrypoint is src/cli.ts. Three worker entrypoints are listed as additional compile entries so Bun embeds them into the binary filesystem:

EntrypointPurposeBinary Path
packages/stats/src/sync-worker.ts Stats synchronization background worker /$bunfs/root/packages/stats/src/sync-worker.js
src/tools/browser/tab-worker-entry.ts Browser tab isolation worker /$bunfs/root/packages/coding-agent/src/tools/browser/tab-worker-entry.js
src/eval/js/worker-entry.ts JavaScript eval sandbox worker /$bunfs/root/packages/coding-agent/src/eval/js/worker-entry.js

Worker Spawn Pattern

Workers must use the compile-safe hybrid pattern to work both in development and in the compiled binary: 

import { isCompiledBinary } from "@gajae-code/pi-utils";

const worker = isCompiledBinary()
  ? new Worker(
      "./packages/<pkg>/src/<worker>.ts",
      { type: "module" }
    )
  : new Worker(
      new URL("./<worker>.ts", import.meta.url).href,
      { type: "module" }
    );

In compiled mode, the path is relative to the monorepo root (matching --root). In development, import.meta.url resolves to the actual file location.

Every new worker entry must be added to the bun build arguments in build-binary.ts. Validate with jwc --smoke-test.

Native Addons

The Rust native bindings (@gajae-code/natives) compile to a .node file via napi-rs. This file cannot be embedded in the Bun binary and must be distributed alongside it.

stageWorkspaceNativeAddons() copies the compiled .node file to the dist/ directory next to the binary. At runtime, require() loads it from the adjacent path.

Compile Targets

The binary is built for the host platform by default. Cross-compilation is possible with Bun's --target flag but is not used in the standard build script.

PlatformNotes
macOS (arm64) Primary development target. Ad-hoc code signing applied automatically.
macOS (x64) Supported via Bun cross-compile. Same code signing.
Linux (x64/arm64) Supported. No code signing needed.
Windows Bun supports Windows compilation but native addon compatibility may require additional testing.

Node Distribution

In addition to the Bun binary, Jawcode provides a Node.js distribution via the jawcode npm package. The packages/jwc package includes:

  • bin/jwc.js — Node launcher that downloads and manages a Bun runtime
  • dist-node/ — esbuild bundle with Bun API shims for Node 22+ compatibility
  • src/shims/ — Bun.file, Bun.write, Bun.spawn, bun:sqlite, and other API polyfills for Node environments

The Node distribution passes adversarial audit rounds including path traversal, archive security, and TLS handling validation.

Verification

After building, verify the binary works:

# Smoke test (validates binary, workers, native addons)
./dist/jwc --smoke-test

# Version check
./dist/jwc --version

# Full test suite against compiled binary
PI_COMPILED=true bun test
