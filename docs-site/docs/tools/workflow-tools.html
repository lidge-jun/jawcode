Workflow Tools

These tools manage durable execution state: goals, skill chains, pending-action resolution, and context checkpointing. They are the control surface for IPABCD orchestration and long-running multi-turn workflows.

goal

Manage the active goal-mode objective. Goals persist across turns and drive continuation steers until paused, completed, or dropped.

Operations

opRequiredEffect
createobjectiveStart a new goal. Use only when no goal exists and none is paused.
get-Return the current goal and usage state.
resume-Re-activate a paused goal so work can continue.
complete-Mark the goal complete after verifying every deliverable.
pause-Suspend the active goal and stop continuation steers. Use when deliverables are done but programmatic completion is blocked, or when the user should decide next.
drop-Discard the current goal without completing it.

Examples

// Start a goal
goal({ op: "create", objective: "Implement feature X" })

// Check status
goal({ op: "get" })

// Pause when blocked
goal({ op: "pause" })

// Resume paused goal
goal({ op: "resume" })

// Mark done after verification
goal({ op: "complete" })

Lifecycle rules

  • Call complete only when the goal is actually done and verified.
  • If get shows a paused goal, call resume before continuing work.
  • Goal continuation steers stop when the goal is paused, completed, or dropped.
  • The CLI surface jwc goal pause --agent --audit writes to disk but does not update in-memory runtime state. Use the tool pause op instead for reliable continuation control.

skill

Invoke another available skill in the current turn. This is the chaining mechanism between workflow skills (e.g. orchestrate p to goal).

Parameters

ParamTypeDescription
namestringSkill name as it appears in /skill:<name>
argsstringFree-form argument string passed to the skill

Chaining mechanism

The skill tool dispatches the callee's SKILL.md as a user-attribution custom message. Before dispatch, it atomically demotes the caller and promotes the callee in .jwc/state/ via jwc state <caller> handoff --to <callee>.

Chain guard

The chain is refused unless the caller's current_phase is in {complete, completed, handoff, failed, cancelled, canceled, inactive}. Prepare with: 

jwc state <skill> write --input '{"current_phase":"handoff"}' --json

Rules

  • Do not use this tool to remind yourself of an already-running skill.
  • Do not chain into the same skill recursively.
  • The chained skill's planning/execution-boundary rules still apply.
  • Call once per chain step. To chain A to B to C, A calls skill(B); B calls skill(C).

Examples

// Hand off from orchestrate p to goal
skill({ name: "goal", args: "track execution of .jwc/plans/..." })

// Trigger jaw-interview
skill({ name: "jaw-interview" })

resolve

Resolves a pending action by either applying or discarding it. Pending actions include preview-style staging (e.g. ast_edit) and long-lived approval gates (e.g. plan approval).

Parameters

ParamTypeDescription
action"apply" | "discard"Persist or reject the pending action.
reasonstringOne short complete sentence explaining why. Starts with capital, ends with period.
extraobject (optional)Free-form metadata. For plan-approval gates, supply extra.title.

Behavior

  • Valid whenever a pending action exists.
  • Fails with an error when no pending action exists.
  • For plan approvals, extra.title provides the kebab/PascalCase slug for the approved plan filename.

checkpoint

Creates a context checkpoint before exploratory work so you can later rewind and keep only a concise report. This minimizes context cost for investigative work that requires many intermediate tool calls.

Parameters

ParamTypeDescription
goalstringClear explanation of what you are investigating.

Flow

1. checkpoint({ goal: "Investigate auth failure root cause" })
2. Perform exploratory work (read, search, find, lsp...)
3. rewind({ report: "Root cause: expired JWT validation..." })

Rules

  • You must call rewind before yielding after starting a checkpoint.
  • You must provide a clear goal explaining what you are investigating.
  • Never call checkpoint while another checkpoint is active.
  • Not available in subagents.

rewind

End an active checkpoint. Rewinds context to the checkpoint, replacing intermediate exploration with your report.

Parameters

ParamTypeDescription
reportstringConcise, factual, actionable summary of findings.

Report guidelines

  • Include key findings, decisions, and unresolved risks.
  • Do not include raw scratch logs unless essential.
  • On success, the session rewinds and keeps the report as retained context.
  • Errors if no checkpoint is active.
