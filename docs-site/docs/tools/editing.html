Editing Tools

Three tools handle code modification: edit for surgical text patches, ast_grep for structural code search, and ast_edit for structural rewrites. Use syntax-aware tools before falling back to text hacks.

edit (patch format)

The primary tool for modifying existing files. It uses an anchored diff format where context lines locate the edit site and +/- prefixes mark additions and deletions.

Parameters

ParameterRequiredDescription
pathYesTarget file path (applies to all entries)
editsYesArray of edit entries

Edit entry types

type Entry =
  | { op: "update", diff: string }        // Patch with diff hunks
  | { op: "create", diff: string }         // Create new file (full content)
  | { op: "delete" }                       // Delete the file
  | { op: "update", rename: string, diff: string }  // Rename + patch

Diff format

Each diff contains one or more hunks. A hunk begins with @@ (optionally followed by an anchor string) and contains lines prefixed with:

  •   (space) — context line, must match the file exactly
  • + — line to add
  • - — line to remove

Anchor selection

Anchors help the tool locate exactly where to apply the edit. Good anchors are:

  • Full function signatures
  • Class declarations
  • Unique string literals or error messages
  • Config keys with uncommon names

Avoid generic anchors like import, export, describe, function, const that match multiple locations.

Example

// Update a function
edit({
  path: "src/app.py",
  edits: [{
    op: "update",
    diff: `@@ def greet():
 def greet():
-    print('Hi')
+    print('Hello')`
  }]
})

// Create a new file
edit({
  path: "src/config.ts",
  edits: [{ op: "create", diff: "export const PORT = 3000;\n" }]
})

Error recovery

ErrorCauseFix
"Found multiple matches"Anchor/context not uniqueAdd more context lines or use a longer anchor
"No match found"Context lines do not exist in fileRe-read the file and generate a fresh patch
Syntax errorMalformed diff formatCheck +/-/  prefixes and @@ headers

ast_grep

Structural code search using AST pattern matching via native ast-grep. Use when syntax shape matters more than raw text: finding function calls, import declarations, JSX props, or control-flow patterns.

Parameters

ParameterRequiredDescription
patYesAST pattern string
pathsYesArray of files, directories, or globs to search

Pattern syntax

Patterns match AST structure, not text. Whitespace and formatting are ignored.

MetavariableMeaning
$NAMECaptures one AST node (bound by name)
$_Matches one node without binding
$$$NAMECaptures zero-or-more nodes (lazy)
$$$Matches zero-or-more without binding

When the same metavariable appears twice, both occurrences must match identical code. $A == $A matches x == x but not x == y.

Examples

// Find all console.log calls in TypeScript
ast_grep({ pat: "console.log($$$)", paths: ["src/**/*.ts"] })

// Find named imports from react
ast_grep({ pat: 'import { $$$IMPORTS } from "react"', paths: ["src/**/*.ts"] })

// Find arrow functions assigned to const
ast_grep({ pat: "const $NAME = ($$$ARGS) => $BODY", paths: ["src/utils/**/*.ts"] })

// Loosest existence check for a symbol
ast_grep({ pat: "processItems", paths: ["src/worker.ts"] })

Important rules

  • Metavariable names must be UPPERCASE
  • Use $$$NAME, never $$NAME (two-dollar form causes parse errors)
  • Partial-text patterns like prefix$VAR or "hello $NAME" do not work
  • Patterns must parse as a single valid AST node
  • Narrow paths to one language to avoid cross-language parse noise
  • Parse issues indicate a query failure, not absence of matches

ast_edit

Structural AST-aware rewrites. Use for codemods and repeated structural transformations where plain text replacement is unsafe.

Parameters

ParameterRequiredDescription
opsYesArray of { pat, out } rewrite operations
pathsYesArray of files, directories, or globs to rewrite

Metavariables captured in pat are substituted into out. An empty out deletes the matched code.

Examples

// Rename a function across a codebase
ast_edit({
  ops: [{ pat: "oldApi($$$ARGS)", out: "newApi($$$ARGS)" }],
  paths: ["src/**/*.ts"]
})

// Delete all console.log calls
ast_edit({
  ops: [{ pat: "console.log($$$ARGS)", out: "" }],
  paths: ["src/**/*.ts"]
})

// Modernize to optional chaining (same metavar enforces identity)
ast_edit({
  ops: [{ pat: "$A && $A()", out: "$A?.()" }],
  paths: ["src/**/*.ts"]
})

// Rewrite import paths
ast_edit({
  ops: [{ pat: 'import { $$$IMPORTS } from "old-package"',
          out: 'import { $$$IMPORTS } from "new-package"' }],
  paths: ["src/**/*.ts"]
})
