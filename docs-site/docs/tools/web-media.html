Web & Media

Tools for web search, image inspection, image generation, and diagram rendering. These extend the agent's reach beyond the local filesystem into web content and visual media.

web_search

Searches the web for up-to-date information beyond knowledge cutoff. Searches are performed automatically within a single API call with no pagination needed.

Parameters

ParamTypeDescription
querystringSearch query (required)
depth"fast" | "deep"fast (default): synchronous 60s. deep: async 180s with heavier reasoning for complex multi-hop queries.
limitnumberMax results to return
num_search_resultsnumberNumber of search results to fetch
recency"day" | "week" | "month" | "year"Recency filter for results
temperaturenumberSampling temperature
max_tokensnumberMax output tokens

Usage guidelines

  • Prefer primary sources (papers, official docs) and corroborate key claims with multiple sources.
  • Always include links for cited sources in the final response.
  • Use the actual current year in queries. Do not hardcode outdated years.
  • For library/framework docs, prefer Context7 MCP or official docs first.

inspect_image

Inspects an image file with a vision-capable model and returns compact text analysis. Use for OCR, UI/screenshot debugging, and scene/object questions.

Parameters

ParamTypeDescription
pathstringPath to the local image file (required)
questionstringSpecific question about the image (required)

Question guidelines

  • Specify what to inspect, constraints, and desired output format.
  • Keep questions grounded in observable evidence.
  • Ask for uncertainty indication when details are unclear.

Examples

// OCR with strict formatting
inspect_image({
  path: "screenshots/error.png",
  question: "Extract all visible text verbatim. Return as bullet list in reading order."
})

// Screenshot debugging
inspect_image({
  path: "screenshots/settings.png",
  question: "Identify the likely cause of the disabled Save button. Return: (1) observations, (2) likely cause, (3) confidence."
})

// Scene/object question
inspect_image({
  path: "photos/shelf.jpg",
  question: "List all clearly visible product labels and their shelf positions (top/middle/bottom)."
})

Limitations

  • Only path and question are allowed as parameters.
  • Returns text-only analysis; no image content blocks in output.
  • Requires a vision-capable model. Configure one if the current model does not support image input.

generate_image

Generates or edits images from text descriptions.

Parameters

ParamTypeDescription
subjectstringDetailed prompt for image generation or editing (required)
inputarrayInput images for editing. Each: { path, data, mime_type }
stylestringArtistic style
scenestringLocation or environment
actionstringWhat the subject is doing
compositionstringCamera angle and framing
lightingstringLighting setup
textstringText to render in the image
changesstring[]Edits to make (for editing mode)
aspect_ratiostring1:1, 3:4, 4:3, 9:16, 16:9
image_sizestring1024x1024, 1536x1024, 1024x1536

Tips

  • When using multiple inputs, describe each image's role in subject (e.g. "Image 1 for composition, Image 2 for lighting").
  • For text rendering, add "sharp, legible, correctly spelled" and keep text short.

render_mermaid

Convert Mermaid graph source into ASCII diagram output.

Parameters

ParamTypeDescription
mermaidstringMermaid graph text to render (required)
configobjectJSON render configuration (spacing and layout options)

Behavior

  • Returns ASCII diagram text.
  • Saves full output to artifact://<id> when storage is available.
  • Returns error when Mermaid input is invalid or rendering fails.
