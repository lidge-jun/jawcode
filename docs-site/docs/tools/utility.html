Utility Tools

Miscellaneous tools for user interaction, task tracking, inter-agent communication, remote access, arithmetic, saved recipes, and scheduled jobs.

ask

Ask the user when you need clarification or input during task execution. Defaults to action; only ask when options have materially different tradeoffs the user must decide.

Parameters

ParamTypeDescription
questionsarrayArray of question objects. Each has id, question, options (array of {label}), optional recommended (0-indexed), optional multi: true.

Rules

  • Provide 2-5 concise, distinct options per question.
  • Do not include "Other" option; the UI automatically adds "Other (type your own)".
  • Default to action. Exhaust existing sources (code, configs, docs) before asking.
  • If multiple choices are acceptable, pick the most conservative option and proceed.

Examples

// Single question with recommendation
ask({
  questions: [{
    id: "auth_method",
    question: "Which authentication method should this API use?",
    options: [
      { label: "JWT" },
      { label: "OAuth2" },
      { label: "Session cookies" }
    ],
    recommended: 0
  }]
})

// Multiple questions
ask({
  questions: [
    { id: "storage", question: "Which storage backend?",
      options: [{ label: "SQLite" }, { label: "PostgreSQL" }] },
    { id: "auth", question: "Which auth method?",
      options: [{ label: "JWT" }, { label: "Session cookies" }] }
  ]
})

todo_write

Manages a phased task list. Tasks are referenced by their verbatim content string, not by auto-generated IDs.

Operations

opRequiredEffect
initlistInitialize the full list (replaces existing). Each phase: { phase, items: string[] }
starttaskMark task in progress
donetask or phaseMark completed
droptask or phaseMark abandoned
rmtask or phaseRemove
appendphase, itemsAppend tasks to phase (lazily creates phase)
notetask, textAppend a note to a task

Anatomy

  • Task content: 5-10 words describing what is being done. Used as the identifier (must be unique).
  • Phase name: short noun phrase (e.g. "Foundation", "Auth"). No prefixes like "1." or "Phase 1:".

When to create a list

  • Task requires 3+ distinct steps
  • User explicitly requests one
  • User provides a set of tasks to complete
  • New instructions arrive mid-task

irc

Send short text messages to other live agents and receive prose replies. The primary coordination mechanism between the main agent and subagents.

Operations

opDescription
listReturn current set of visible peers
sendDeliver message to to (specific id or "all"). Recipient generates reply via ephemeral side-channel turn.

When to use

  • Unexpected state: missing file, contradictory config, broken spec. DM the spawning agent.
  • Blocked by a peer: another agent holds the file/branch you need. DM them before duplicating.
  • Decision outside scope: a fork the assignment did not pre-decide. Ask the requester.
  • Coordination: peer's in-flight work would benefit from yours.

Etiquette

  • Plain prose only. No JSON payloads.
  • Do not quote the message you are replying to.
  • One round-trip is enough. Do not retry "did you get my message?"
  • Stay terse. Share paths via local:// or artifact:// URLs.
  • Address peers by exact id from op: "list".

calc

Perform basic calculations.

Supported syntax

  • Operators: +, -, *, /, %, **, parentheses
  • Literals: decimal, hex (0x), binary (0b), octal (0o)

Returns each calculation result with its prefix and suffix applied.

recipe

Run a recipe/script/target from the project's task runners.

Parameters

ParamTypeDescription
opstringTask name plus any args, e.g. "test" or "build --release"

Monorepo namespacing

Package and Cargo target tasks use / as namespace separator:

recipe({ op: "pkg-a/test" })
recipe({ op: "crate/bin/server" })

When the same task name exists in multiple runners, prefix with the runner id (e.g. "bun:test"). Runs in the session's cwd.

Cron (CronCreate / CronList / CronDelete)

Schedule prompts to fire on recurring cron schedules or as one-shots. Poll deployments, babysit PRs, watch builds, or set reminders.

CronCreate

ParamTypeDescription
expressionstring5-field cron: minute hour day-of-month month day-of-week
promptstringThe prompt to execute on each fire
recurringbooleanWhether the job recurs or fires once

Cron expression syntax

ExpressionMeaning
*/5 * * * *Every 5 minutes
0 * * * *Every hour on the hour
0 9 * * *Every day at 9am local
0 9 * * 1-5Weekdays at 9am local

Lifecycle

  • Tasks fire between turns, never mid-response.
  • All times are in local timezone.
  • Recurring tasks auto-expire 7 days after creation; one-shots self-delete after firing.
  • Up to 50 scheduled tasks per session.
  • Closing or replacing the session clears all scheduled tasks.
  • Disable with CLAUDE_CODE_DISABLE_CRON=1.

CronList / CronDelete

CronList enumerates all scheduled tasks. CronDelete cancels by the 8-character job id returned from CronCreate.
