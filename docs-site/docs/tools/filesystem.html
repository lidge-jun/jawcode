Filesystem Tools

Four tools handle all filesystem inspection: reading files and directories, finding files by name, searching file content, and creating new files. These replace shell commands like cat, ls, grep, and find with structured, gitignore-aware alternatives.

read

A single tool for reading files, directories, archives, SQLite databases, images, documents (PDF/DOCX/PPTX/XLSX/RTF/EPUB/ipynb), internal URIs, and web URLs.

Parameters

ParameterRequiredDescription
pathYesLocal path, internal URI (agent://, artifact://, rule://, local://), or URL. Append :<sel> for line ranges or special modes.

Selectors

Append a selector to path to control what is returned:

SelectorEffect
:50 / :50-Read from line 50 onward
:50-200Lines 50 through 200 inclusive
:50+150150 lines starting at line 50
:5-16,960-973Multiple ranges in one call
:rawVerbatim text, no anchors or summary
:conflictsIndex of unresolved git merge conflicts

Behavior by path type

  • Directory — returns a depth-limited listing
  • Parseable code (no selector) — returns a structural summary with declarations kept and bodies collapsed to ..
  • Code with selector — returns lines with line+hash anchors for the edit tool
  • Documents — extracts text from PDF, Word, PowerPoint, Excel, RTF, EPUB
  • Notebooks (.ipynb) — shows editable # %% [type] cell:N text
  • Images — returns metadata (mime, bytes, dimensions, channels)
  • Archives — archive.zip:path/inside reads members directly
  • SQLite — file.db lists tables; file.db:table shows schema + samples; supports ?where=, ?order=, ?q=SELECT
  • URLs — reader-mode by default; :raw for untouched HTML

Example

// Read lines 50-200 of a TypeScript file
read({ path: "src/agent.ts:50-200" })

// Read a SQLite table with filtering
read({ path: "data.db:users?where=status='active'&order=created:desc" })

// Read a URL in reader mode
read({ path: "https://docs.example.com/api" })

find

Fast file-name pattern matching that works with any codebase size. Respects .gitignore by default.

Parameters

ParameterRequiredDefaultDescription
pathsYes—Array of globs, files, or directories
gitignoreNotrueRespect .gitignore. Set false for .env*, *.log, build outputs
hiddenNotrueInclude hidden (dot) files
limitNo1000Max results
timeoutNo5Seconds (0.5 to 60)

Example

// Find all TypeScript files under src/
find({ paths: ["src/**/*.ts"] })

// Find gitignored files
find({ paths: [".env*"], gitignore: false })

// Multiple targets as separate array elements
find({ paths: ["src/**/*.ts", "test/**/*.ts"] })

search

Regex content search using Rust regex syntax (RE2-style). No lookaround or backreferences; use line anchors or post-filters instead.

Parameters

ParameterRequiredDescription
patternYesRegex pattern (Rust syntax)
pathsYesArray of files, directories, globs, or internal URLs
iNoCase-insensitive search
skipNoFiles to skip for pagination

Output format

Results are anchor-prefixed: *5th|content for matches,  9x}|content (leading space) for context lines. The 2-char suffix is a content fingerprint used by the edit tool.

Example

// Search for a function definition
search({ pattern: "function buildPrompt", paths: ["src/"] })

// Case-insensitive search across test files
search({ pattern: "describe\\(", paths: ["test/"], i: true })

write

Creates or overwrites a file at the specified path. For modifying existing files, prefer the edit tool which is more precise and preserves formatting.

Parameters

ParameterRequiredDescription
pathYesTarget file path
contentYesFile content to write

Special targets

  • Archives — archive.zip:path/inside/archive writes to archive entries
  • SQLite — db.sqlite:table inserts a row; db.sqlite:table:key updates (JSON content) or deletes (empty content)

Example

// Create a new file
write({ path: "src/config.ts", content: "export const PORT = 3000;\n" })

// Insert a SQLite row
write({ path: "data.db:users", content: '{"name":"Alice","role":"admin"}' })
