Tools Overview

Jawcode ships 40+ built-in tools that the agent uses to read, search, edit, execute, and orchestrate work across a repository. Tools are divided into two tiers:

Essential vs Discoverable

Not every tool is loaded into every session. Jawcode splits its tool inventory into two tiers to keep the system prompt lean:

Essential tools (14)

Always loaded. These cover the core agent loop: read a file, search for content, edit code, run a command, spawn a subagent.

ToolPurpose
readRead files, directories, archives, SQLite, images, documents, URLs
editPatch existing files with anchored diff hunks
writeCreate or overwrite files
bashExecute shell commands (git, bun, cargo, python)
searchRegex content search across files
findFast file-name pattern matching
ast_grepStructural code search via AST patterns
ast_editStructural code rewrites via AST
browserControl a real Chromium tab for JS, auth, interaction
taskLaunch parallel subagents
subagentList, inspect, await, pause, resume, steer, cancel subagents
web_searchSearch the web for current information
generate_imageGenerate or edit images
search_tool_bm25Discover and activate hidden tools

Discoverable tools (11+)

Hidden by default but activatable at runtime via search_tool_bm25. The agent searches by keyword and the matching tools are loaded for the rest of the session:

ToolPurpose
backgroundManage background work rows (list, detail, follow, cancel)
computer_useControl desktop apps via the cua-driver backend
debugDAP debugger: launch, attach, breakpoints, stepping, inspection
evalRun Python or JavaScript in persistent in-process kernels
jobLegacy async job polling and cancellation
monitorStream stdout lines from long-running scripts as notifications
ircSend/receive messages between agents over IRC-like channels
CronCreateSchedule a prompt on a cron expression
CronListList scheduled cron jobs
CronDeleteCancel a scheduled cron job
recipeExecute saved bash recipes (multi-step shell presets)

MCP (Model Context Protocol) servers can add additional tools at runtime. These are also discovered via search_tool_bm25.

Tool discovery flow

When the agent needs a capability not in the essential set, it calls search_tool_bm25 with a natural-language query:

search_tool_bm25({ query: "debug breakpoint" })
// Returns: activated debug tool for the rest of the session

search_tool_bm25({ query: "background job status" })
// Returns: activated background + job tools

The search matches against tool name, description, server name (for MCP tools), and input schema keys. Repeated searches add to the active set without removing earlier activations.

Tool priority rules

Jawcode enforces strict tool-over-shell preferences to avoid common agent failure modes:

OperationUseNever
File/dir readsreadcat, ls, head, tail
Content searchsearchgrep, rg, ag, awk
File lookupfindfind (shell), fd, locate
File editseditsed, shell redirection
File creationwriteecho > file
Structure searchast_grepregex for syntax-shaped queries

The bash tool is reserved for terminal operations that no dedicated tool covers: git operations, package managers, build commands, and process management.
