Code Intelligence

Language Server Protocol integration and GitHub CLI operations. These tools provide semantic code understanding and repository management beyond what text-based search can offer.

lsp

Interacts with Language Server Protocol servers for code intelligence. Safer and more accurate than text-based alternatives for symbol-aware operations.

Operations

ActionDescription
diagnosticsGet errors/warnings for a file, glob, or workspace (file: "*")
definitionGo to symbol definition with file path, position, and 3-line context
type_definitionGo to symbol type definition
implementationFind concrete implementations
referencesFind references (first 50 with context, remaining location-only)
hoverGet type info and documentation
symbolsList symbols in a file, or workspace search with file: "*"
renameRename symbol across codebase (preview or apply)
rename_fileRename/move file; sends workspace/willRenameFiles so import paths update
code_actionsList/apply quick-fixes, refactors, import actions
statusShow active language servers
capabilitiesDump per-server capabilities (standard + experimental + executeCommand)
requestSend a raw LSP request (e.g. rust-analyzer/expandMacro)
reloadRestart a specific server or all servers

Parameters

ParamTypeDescription
filestringFile path, glob pattern (e.g. src/**/*.ts), or "*" for workspace scope
linenumber1-indexed line number for position-based actions
symbolstringSubstring on target line for column resolution. Append #N for Nth occurrence
querystringSymbol search query or code-action filter
new_namestringRequired for rename and rename_file
applybooleanApply edits (default true for rename/rename_file)
payloadstringJSON-encoded params for raw request action
timeoutnumberRequest timeout in seconds (5-60, default 20)

Critical rules

  • Use lsp for symbol-aware operations (rename, find references, go to definition) whenever a language server is available.
  • Never perform cross-file renames with ast_edit, sed, or manual edits when lsp rename can do it. Text-based renames miss shadowing, re-exports, and usages in other files.
  • Prefer lsp code_actions for imports, quick-fixes, and refactors the language server knows how to apply.

Examples

// Get workspace diagnostics
lsp({ action: "diagnostics", file: "*" })

// Go to definition
lsp({ action: "definition", file: "src/auth.ts", line: 42, symbol: "validateToken" })

// Safe cross-file rename
lsp({ action: "rename", file: "src/auth.ts", line: 10, symbol: "oldName", new_name: "newName" })

// List available code actions
lsp({ action: "code_actions", file: "src/server.ts", line: 5 })

// Raw LSP request (rust-analyzer specific)
lsp({ action: "request", file: "src/main.rs", query: "rust-analyzer/expandMacro", line: 15, symbol: "derive" })

github

GitHub CLI tool with op-based dispatch. Wraps gh for repositories, pull requests, search, checkout, push, and Actions watch workflows.

Operations

opDescription
repo_viewRead repository metadata. Optional repo (owner/repo) and branch.
pr_createCreate a PR. Provide title + optional body, or fill: true to auto-fill from commits. Supports base, head, draft, reviewer[], assignee[], label[].
pr_checkoutCheck out PRs into dedicated git worktrees. pr accepts number, URL, branch, or array for batch checkout.
pr_pushPush checked-out PR branch back to its source. Optional forceWithLease.
search_issuesSearch issues using GitHub search syntax. Optional query, repo, limit, since, until.
search_prsSearch PRs using GitHub search syntax. Same params as search_issues.
search_codeSearch code with GitHub code search syntax. Required query. No date filtering.
search_commitsSearch commits across GitHub. dateField always uses committer-date.
search_reposSearch repositories across GitHub.
run_watchWatch a GitHub Actions run. Streams snapshots, fast-fails on job failure, tails failed job logs.

URL schemes (read tool)

For reading single issues or PRs, use the read tool with URL schemes instead of github ops:

// Read an issue
read({ path: "issue://42" })

// Read a PR
read({ path: "pr://42" })

// Read PR diff (file listing)
read({ path: "pr://42/diff" })

// Read single file from PR diff (1-indexed)
read({ path: "pr://42/diff/3" })

// Full unified diff
read({ path: "pr://42/diff/all" })

Date filters

since and until accept relative durations (3d, 12h, 2w), ISO dates (2025-01-15), or ISO datetimes. Translated to GitHub search qualifiers internally.

Examples

// Create a draft PR
github({ op: "pr_create", title: "feat: add auth", draft: true })

// Search recent issues
github({ op: "search_issues", query: "bug label:critical", since: "7d" })

// Watch CI for current commit
github({ op: "run_watch" })

// Batch checkout multiple PRs
github({ op: "pr_checkout", pr: [42, 43, 44] })
