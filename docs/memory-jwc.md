# jwc memory — long-term memory surface

jwc exposes a cli-jaw-shaped verb surface over the existing memory engine.
The engine stays canonical: the local pipeline (stage1 extraction → phase2
consolidation, SQLite) or the Hindsight cloud backend. The surface adds
search/read/save/trace verbs and a per-turn Task Snapshot — it does not
change the schema or the write pipeline.

The `memory` and `chat` commands register under the jaw brand only
(`jawOnlyCommands` in `cli.ts`); the jwc engine brand keeps a zero diff.

## Storage layout (local backend)

| Piece | Location | Notes |
|---|---|---|
| Records DB | `~/.jwc/agent/agent.db` | tables `threads` / `stage1_outputs` / `jobs` |
| Artifacts | `~/.jwc/agent/memories/<encoded-project>/` | `MEMORY.md`, `memory_summary.md`, `skills/` — regenerated by phase2 |
| Sessions (chat search) | `~/.jwc/agent/sessions/<encoded-cwd>/*.jsonl` | rollout transcripts |

Backend selection: settings `memory.backend` = `local` | `hindsight` | off.

## Verbs

```
jwc memory search <query> [--cloud] [--scope path[,path...]]
jwc memory browse [--limit N]
jwc memory list [--limit N]
jwc memory status
jwc memory reindex
jwc memory read <ref> [--lines a-b]
jwc memory save <file> <content...> [--kind profile|shared|episode]
jwc memory context <ref>
jwc chat search <query> [--days N] [--recent N] [--context N]
```

### ref vocabulary

`summary` (memory_summary.md) · `memory` (MEMORY.md) · `raw` (recent stage1
digest) · `stage1:<thread_id>` · `rollout:<slug>` · artifact basenames are
accepted as aliases.

### kind mapping (federation common tongue)

| kind | local meaning | ranking weight |
|---|---|---|
| `profile` | `MEMORY.md` hits | strongest (-4) |
| `shared` | `memory_summary.md` hits | -3 |
| `episode` | stage1 rows (auto + manual) | 0, recency-boosted |

`memory save --kind` records the kind in the row's frontmatter so cross-tool
federation can map records without file-format conversion.

## Search

First-iteration contract: FTS5 (when indexed) with LIKE fallback (`hybrid`
default; `memories.searchMode` = `fts` | `like` | `hybrid`), plus kind weight
and `generated_at` recency. One seeded synonym group
(`pabcd ↔ plan/audit/build/check/done`) expands queries both ways. Results
cap at `memories.searchLimit` (8), snippets at 700 chars.

`--cloud` routes a single search through Hindsight recall regardless of the
configured backend.

## Manual save semantics

`memory save <file> <content>` writes a `stage1_outputs` row with
`thread_id = manual:<file>` (direct upsert — the job-owned write path
requires a running job's ownership token and is not applicable) plus a
`threads` row with `source_kind = "manual"`, then bumps the phase2 watermark
(`forceDirtyWhenNotAdvanced`). Consequences:

- The row is immediately searchable and is folded into `MEMORY.md` /
  `memory_summary.md` at the next phase2 consolidation.
- `claimStage1Jobs` never re-processes manual rows (it claims `cli`/`app`
  sources only).
- Saving the same `<file>` again replaces the record (upsert).

## Visibility contract (important)

`refreshBaseSystemPrompt()` is a live-session method. A standalone
`jwc memory save` process cannot refresh another process's prompt — running
sessions pick the save up on their next prompt rebuild or phase2 pass. When
the runtime is invoked in-session (optional `session` param), the refresh
happens immediately.

## Per-turn Task Snapshot

The local backend implements `beforeAgentStartPrompt`: each turn, the current
prompt is searched and the top 4 diversified hits (episodes capped at 2, one
hit per ref) are appended to the system prompt inside a `<memories>` block.
This runs alongside — not instead of — the existing `memory_summary.md`
injection (5,000-token cap). Zero hits → zero injection. Failures are
swallowed (fail-open).

## Process hygiene

CLI invocations exit explicitly (`process.exit`) after writing output —
`Settings.init` keeps the event loop alive, and without the explicit exit a
local-backend invocation lingers as a ~300MB zombie (260612 incident). When
spawning `jwc memory …` from tests or scripts, always pass a hard timeout
and never invoke subcommands that are not registered for the active brand
(unregistered args fall through to the interactive session).

## Out of scope (follow-ups)

`memory init`, FTS5 synonym tables / RRF fusion / CJK trigram, slash
`/memory` unification, dashboard federation transport (search-API based —
no markdown conversion layer).
