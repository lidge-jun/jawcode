## Why This Is A Gap

GJC has a user-facing `gjc notify setup/status` command that stores Telegram notification settings safely and verifies Telegram BotFather/private-chat pairing. JWC has no `jwc notify` command and no settings schema for notification adapters beyond cli-jaw bootstrap references.

This card owns configuration and onboarding only. Runtime delivery belongs to [10.030](./10.030_gjc_chase_telegram_managed_daemon.md).

GJC settings:

| key | meaning |
|---|---|
| `notifications.enabled` | global switch |
| `notifications.telegram.botToken` | BotFather token |
| `notifications.telegram.chatId` | paired private chat |
| `notifications.redact` | suppress sensitive streamed content, while asks stay answerable |
| `notifications.verbosity` | `lean` or `verbose` threaded mirror |
| `notifications.daemon.idleTimeoutMs` | daemon idle shutdown |

Upstream environment precedence:

1. `GJC_NOTIFICATIONS=0` hard opt-out.
2. Local `/notify off` disables only the current session.
3. `GJC_NOTIFICATIONS=1` or `GJC_NOTIFICATIONS_TOKEN` enables legacy explicit path.
4. Complete global setup enables notifications automatically.
5. Otherwise off.

JWC must translate these to `JWC_NOTIFICATIONS*` or dual-read with `JWC_*` public precedence; `GJC_NOTIFICATIONS*` should appear in JWC docs only as upstream-source citation or explicit compatibility note.

## JWC Reconcile Notes

| issue | decision pressure |
|---|---|
| command name | Public command must be `jwc notify`, not `gjc notify`. |
| token printing | Status/setup output must mask token (`first 4 chars + len`), never print raw. |
| setup chat type | GJC accepts private DM only; keep fail-closed default. |
| Threaded Mode | Setup checks `getMe.has_topics_enabled`, but runtime still falls back if `createForumTopic` fails. |
| redaction | Asks/options are not redacted; idle/stream/context/image payloads are suppressed when redaction is on. |
| config schema | Must update generated schema if settings are typed. |

## Suggested Split

1. Add settings schema and typed `getNotificationConfig`.
2. Add `jwc notify status` with masked secrets.
3. Add non-interactive `jwc notify setup --token --chat-id [--redact]`.
4. Add interactive BotFather token prompt and private chat pairing.
5. Add Threaded Mode guidance/fallback labeling.
