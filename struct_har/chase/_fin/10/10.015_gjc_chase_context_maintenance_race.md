agent-initiated custom turns(task notification 등)이 pre-prompt context maintenance를 우회해
context 사용량 추정에서 빠지던 문제 수정. 활성 notification payload를 추정에 포함하되,
display/footer/RPC 경로에서는 native tokenization 대신 cheap estimator를 유지.

설계 결정:
- Rejected: 모든 usage에 native tokenization → RSS/CPU 비용이 display 경로에 옮겨감
- Rejected: compaction만으로 monitor output 대응 → 단일 거대 monitor 라인이 단독 overflow 가능
- Adopted: pre-prompt maintenance에 notification payload 포함 + display는 cheap estimator
