## Why This Is A Gap

This is the highest-priority sub-cluster of 10.053 because it protects stored session data from corruption regardless of fork naming. GJC hardened Codex encrypted-replay handling, clamped the GPT-5.5 context window, sanitized replay image URL fields, materialized resident compaction blobs, and mitigated leaked invoke envelopes / harmony format leakage across providers. The orphaned vision model role was also removed in favor of direct image embedding.

Behavioral surface:
1. Drop invalid Codex encrypted content rather than persisting corrupt replay.
2. Preserve Codex encrypted replay strings on round-trip.
3. Materialize resident compaction replay blobs so compaction does not lose state.
4. Sanitize replay image URL fields (including object-valued image detail).
5. Clamp GPT-5.5 provider-scoped context cap.
6. Catch leaked invoke envelopes across providers + format harmony leak mitigation.
7. Embed images directly and remove the orphaned vision model role.
