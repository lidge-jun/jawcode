## Why This Is A Gap

GJC documented notifications as a broader SDK and added chat adapter abstractions beyond Telegram (Discord/Slack). JWC currently has cli-jaw channel concepts, but not a JWC-native notification SDK/adapters doc surface.

This card is mostly documentation/product alignment after implementation slices land.

## JWC Reconcile Notes

| issue | decision pressure |
|---|---|
| cli-jaw overlap | JWC is embedded into cli-jaw, which already has Telegram/Discord channel delivery. Avoid duplicating user-facing docs until product boundary is clear. |
| transport-agnostic SDK | Valuable as internal session endpoint contract even if only Telegram ships first. |
| Discord/Slack | GJC has adapter scaffolding; JWC should probably document as deferred unless a user asks for those transports. |
| release docs | Changelogs and install docs should not promise mobile notifications until implementation + smoke exists. |

## Suggested Split

1. Keep this chase card as the docs/product rollup.
2. After 10.028-10.034 implementation, write JWC docs for the supported subset only.
3. Explicitly reject/defer Discord/Slack if not shipped.
4. Add docs tests to prevent stale command names/secrets.
