## 왜 gap인가

10.021 closed the live-surface **presence** gap: live surfaces (web/cli/native) now require a
resolvable non-empty artifact **file**, not inline prose (`requireLiveSurfaceProof`, commit `cdee4f7`).
But jwc still only checks that the file **exists and is non-empty** — it does not verify the file's
**content**. gjc's red-team engine opens the bytes and rejects fakes (blank screenshots, mismatched
CLI stdout, malformed transcripts). That deep content/structural/replay engine is the residual,
consciously deferred from 10.021 because it adds ~800+ LOC, process spawning, and image parsing —
each warranting independent backend/security review.
