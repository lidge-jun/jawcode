## Why This Is A Gap

GJC added a terminal bell as a new local notification channel, plus a generic completion-notification command hook. This is a UX-tier gap: JWC may want the same "turn finished" signal, but settings must live in `.jwc` and access must be guarded so a missing config never throws.

Behavioral surface:
1. Terminal bell notification channel emitted on turn/task completion.
2. Settings guard so missing/partial bell settings do not crash access.
3. Generic completion-notification command hook (run a command when work finishes).
