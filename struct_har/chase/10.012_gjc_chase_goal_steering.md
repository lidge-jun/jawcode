## 왜 gap인가

Upstream GJC dev expanded the goal engine from a single `add_subgoal` command to a full steering subsystem: split, reorder, revise wording, annotate ledger, mark-blocked-superseded, and a unified `executeUltragoalSteeringCommand` dispatcher. JWC goal-engine still only supports `add_subgoal`.

## Source facts

| fact | upstream evidence |
|---|---|
| `addUltragoalSubgoalToPlan` refactored from inline | `goal-engine.ts` — delegates subgoal insertion to plan-aware helper |
| `splitUltragoalSubgoal` | split a subgoal into multiple children |
| `reorderPendingUltragoalGoals` | reorder pending subgoals |
| `revisePendingUltragoalWording` | revise subgoal wording in-place |
| `annotateUltragoalLedger` | add annotations to goal ledger |
| `markBlockedUltragoalSuperseded` | mark blocked goals as superseded |
| `executeUltragoalSteeringCommand` | unified dispatcher routing all steering kinds |

## JWC reconcile notes

- JWC renamed `Ultragoal*` → `Goal*` throughout; any port must apply the same rename.
- File path is `jwc-runtime/goal-engine.ts` (not `gjc-runtime/ultragoal-runtime.ts`).
- The steering subsystem is additive — existing `add_subgoal` remains, new commands extend it.
