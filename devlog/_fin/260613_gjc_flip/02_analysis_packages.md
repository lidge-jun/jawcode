# 02 — 분석 A2: 패키지 스코프·빌드 인프라 (Sonnet 파견, 260613 밤)

## 핵심 판정

1. **`@gajae-code` 스코프 플립은 npm 퍼블리시 관점에서 실행 불가** — 스코프가 upstream 소유라 포크는 그 스코프로 퍼블리시 못함. **전략 B(스코프 유지, 063.1)가 유일한 라이브 퍼블리시 경로**. 포크가 실제 npm에 내보내는 것은 비스코프 `jwc` 패키지뿐.
2. 워크스페이스 내부 import `@gajae-code/*`는 **625파일급** — 플립 시 upstream 체리픽 영구 발산세. 미래 플립을 원하면 가드 스크립트 4곳의 `WORKSPACE_SCOPE = "@gajae-code"` **상수 추출(063.1 M3)**이 선행 — 그 후엔 상수 1곳+`bun install` 2-step.
3. **`embedded:gjc/` 런타임 문자열** (`gjc-defaults.ts`): `src/defaults/gjc/` 디렉터리 리네임 시 ① text-import 경로 7곳(tsc/bun이 강제 — 무음 아님) ② `"embedded:gjc/"` 런타임 상수 2곳 동시 갱신. 1파일+1디렉터리로 자기완결, 스코프 문제와 독립.

## 플랜 반영

- **스코프 플립 제외 확정** — 내부 전용이며 퍼블리시 불가 스코프. 의도적 예외로 기록.
- defaults/gjc 디렉터리 플립은 F2에 포함 (자기완결 확인됨).
