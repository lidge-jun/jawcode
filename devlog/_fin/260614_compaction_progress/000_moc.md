# Compaction progress MOC

Date: 2026-06-14
Status: implemented; verifier DONE; chase 10.004 admitted to `_fin/10`

## Goal

Add an honest, Claude-like progress surface for manual `/compact` and automatic context-full maintenance without pretending that provider-side LLM/remote compaction exposes real streaming progress.

The target UX is:

- Existing real backend compaction remains the source of truth.
- In-flight manual and automatic compaction show an estimated percent plus fixed-width text bar in the existing status area.
- Completed compaction continues to render through the existing `CompactionSummaryMessageComponent` with `ctrl+o` expansion.
- Percent values are documented as predicted-time UI progress, not backend token/HTTP/provider progress.

## Decision summary

- Add a new progress callback/event path instead of overloading `auto_compaction_start/end`.
- Use `Loader.setMessage(...)` with a fixed-width text fill bar inside the message; do not add a new TUI `ProgressLoader` primitive in B1–B5.
- Keep extension/hook public APIs quiet by default; expose progress through `AgentSessionEvent`/wire subscribers and interactive UI only.
- Keep ACP mapping empty for the new progress event.
- Keep backend/session events to phase-boundary snapshots; the interactive UI owns local time-smoothing ticks and must not re-emit those ticks onto session/wire surfaces.
- Treat `compaction_progress` as ephemeral UI/RPC state; exclude it from coordinator sidecar state and transcript rows.
- Build order follows `001_heuristic_progress_plan.md`: B1/B2 backend types and boundary emissions, B3 session sink, B4 wire/ACP exhaustiveness, B5 UI presenter.
- Follow-up hardening fixed threshold pruning persistence: pruning now runs only after the threshold decision and writes materialized branch updates back to resident session entries.

## Non-goals

- No exact provider progress claim.
- No streaming progress from `instrumentedCompleteSimple(...)` or `/responses/compact`; those calls remain one-shot awaits.
- No new transcript row for in-flight progress.
- No changes to completed compaction summary expansion behavior beyond preserving current component use.
