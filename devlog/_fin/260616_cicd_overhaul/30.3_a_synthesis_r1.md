## Critical Finding: ARCH-A15 — workspace:* breaks CI smoke test

**Problem**: `validate:jwc-release` runs in CI BEFORE `ci-release-publish.ts`. It calls `smoke-packed-sdk.mjs` which does `npm pack` directly. `npm pack` reads `package.json` literally — if it says `workspace:*`, the tarball contains `workspace:*`, and `npm install` fails because npm can't resolve that outside a workspace context.

The plan's W1-B fix (rewrite before preBuild) only applies inside `ci-release-publish.ts`, not during CI's standalone `validate:jwc-release`.

**Resolution**: Use hard-coded `"1.0.0"` instead of `"workspace:*"`. This is simpler and solves all paths:

| Path | workspace:* | 1.0.0 |
|---|---|---|
| CI smoke test (`npm pack` direct) | ❌ FAILS — literal workspace:* | ✅ npm resolves 1.0.0 from co-installed tarball |
| Publish (`ci-release-publish.ts`) | Needs rewrite machinery | ✅ No rewrite needed — bare version passes through |
| Consumer install | Needs published dep | ✅ natives@1.0.0 published first via ci-release-publish.ts |

**Tradeoff**: On future version bumps, someone must update jwc/package.json manually. Mitigated by adding a version-match gate to `validate:jwc-release`.
