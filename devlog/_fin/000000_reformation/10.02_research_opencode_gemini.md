## opencode (TypeScript, Vercel AI SDK)

### O1. 캐시 마킹 자동화 — 마지막 2턴 고정 마킹

`applyCaching()` — 시스템 상위 2 + 비시스템 **마지막 2 메시지**에 매 요청 캐시 마커.
"마지막 N턴 고정"이라 prefix 안정성을 따로 관리할 필요가 없는 설계.
✅ (`packages/opencode/src/provider/transform.ts:351-366` cacheControl ephemeral,
`:1079` `promptCacheKey = sessionID`)

- jawcode와 동일하게 OpenAI엔 `store: false` + sessionID 캐시 키 — 우리 구성이 업계
  표준임을 확인 (00 밴드의 차별점은 WS 델타 쪽).

### O2. 헤더 우선 백오프

`retry-after-ms`(float ms) → `retry-after`(초/HTTP date) → 자체 지수(2000ms×2^n, 상한 30s)
순서로 서버 지정 지연을 우선. 5xx는 SDK가 non-retryable로 봐도 재시도.
(`packages/opencode/src/session/retry.ts:29-66`)

- 재시도마다 TUI에 `{ type: "retry", attempt, next }` 상태 송출 — **침묵 없는 백오프**.
  G3(가시성)의 선례.

### O3. 툴 출력 파일 스필 + 위임 힌트

2,000줄/50KB 초과분은 `~/.opencode/tmp/truncation/<toolID>`로 스필, 모델엔 프리뷰+경로.
task 서브에이전트가 있으면 "재독은 explore에 위임하라" 힌트까지 주입
(`packages/opencode/src/tool/truncate.ts:17-19,86-133`). CC의 50K 스필과 동일 계열 —
G8 설계 시 두 구현 비교.

### O4. 인라인 프루닝 (`compaction.prune`)

최신→과거로 툴 출력 토큰을 누적, 40K 초과 시 가장 오래된 출력부터 DB에 `compacted` 마킹
→ 직렬화 시 빈 문자열 (`packages/opencode/src/session/compaction.ts:297-341`). CC
마이크로컴팩션의 opencode판.

### O5. 기타

- 둠루프 감지: 동일 툴+동일 입력 3연속이면 ask 인터럽트 (`session/processor.ts:428-452`).
- `x-session-affinity: sessionID` 헤더 — 프록시/LB 캐시 친화 라우팅 (`session/llm/request.ts:177`).
- effort→프로바이더 파라미터 매핑을 단일 룩업 테이블로 (`transform.ts:518-1051`) —
  jawcode `model-thinking.ts`와 같은 접근, 교차 점검용.

## gemini-cli (TypeScript, GenAI SDK)

### G-1. fast-ack: 1.2초 마이크로 응답

조향(steering) 입력에 소형 모델 호출(입력 320자/출력 90자 캡, **1,200ms 하드 타임아웃**,
실패 시 정적 폴백 문자열). 메인 모델이 차가워도 사용자는 ~1.2초 안에 반응을 본다.
✅ (`packages/core/src/utils/fastAckHelper.ts:18,98` — `DEFAULT_MAX_INPUT_CHARS=1200`,
`STEERING_ACK_TIMEOUT_MS=1200`)

- jawcode 시사점: 증상 ②(턴 경계 TTFT)의 **체감** 완화책. 전송 수리(T1)와 별개로 UX 층에서
  병행 가능 — "TTFT를 줄이는 것"과 "기다림을 보이게 하는 것"은 다른 무기.

### G-2. 연결 단계 vs 스트림 중 에러 분리

첫 청크 수신 전(connection phase)과 후(mid-stream)를 구분 — 전자는 본 재시도 정책(10회,
5s 초기, 30s 상한, ±30% 지터), 후자는 별도 소예산(4회, 1s/2s/4s 지수)으로 전체 요청 재생
(`packages/core/src/core/geminiChat.ts:99-103,519-632`, `utils/retry.ts:42-47,218-433`).
재시도마다 `RETRY` 스트림 이벤트로 UI 배지.

- jawcode 시사점: 현재 워치독은 첫 이벤트/idle 두 타이머뿐 — 연결/스트림 중 구분과
  재시도 예산 분리는 G1 설계에 직접 반영.

### G-3. IDE 컨텍스트 JSON 델타 전송

첫 턴 이후 에디터 상태는 **변경분만** 전송, 무변경이면 0토큰
(`packages/core/src/core/client.ts:485-594`). "동적 컨텍스트는 델타로"의 일반 원칙 —
G4 prefix 안정성과 동전의 양면.

### G-4. 압축 전 절단 + 파일 스필 (head+tail)

압축 요약 호출 전에 50K 토큰 초과 함수 응답을 디스크 스필하고 **head 20% + tail 80%**
스니펫으로 치환 (`packages/core/src/context/chatCompressionService.ts:131-220`,
`utils/fileUtils.ts:661-712`). head만 남기는 절단보다 모델 정보 보존이 낫다 — G8 채택.

### G-5. 기타

- 컴팩션 트리거 50% (codex-rs·CC보다 공격적), 요약은 Flash-Lite 소형 모델.
- `checkNextSpeaker`: 툴콜 없는 응답 후 소형 모델로 "모델이 계속해야 하나" 판정 → 자동
  "Please continue." (`utils/nextSpeakerChecker.ts`).
- 모델 헬스 추적 + 지속 429 시 폴백 모델로 스왑하고 재시도 카운터 리셋 (`core/client.ts:764-797`).
