## service_tier echo 조사 — 정정된 결론

초기 가설("fast가 sub에서 무시된다")은 **틀렸다.** 응답의 `service_tier` echo는 fast
실현 여부의 신호가 아니다.

증거 체인:
- `/tmp/tier-ab.ts` (priority vs default, plain SSE): 둘 다 echo `default`, 속도 동일
  (57.5 vs 59.0 tok/s) → 여기서 "무시된다"고 **오판**했음.
- `/tmp/tier-orig.ts` (originator `pi` vs `codex_cli_rs`): 둘 다 echo `default` → originator
  게이트 아님.
- **네이티브 codex-cli 실행 트레이스** (`codex exec`, RUST_LOG=trace, Pro 계정
  iovesally3@naver.com, config `service_tier="priority"`): 네이티브도 `priority`를 보내고
  응답은 `auto`/`default`를 echo한다. 그리고 `codex-rs`는 **응답 `service_tier`를 어디서도
  읽지 않는다** — realized 체크도, 경고도 없음 (`grep service_tier core/src/client.rs` →
  요청 설정만 존재).

결론: ChatGPT/Codex 백엔드는 priority가 적용돼도 echo를 `auto`/`default`로 돌려준다 (API
키 종량 응답과 의미가 다름). echo ≠ 거부. 사용자 확인: **codex-cli(Pro)에서 fast는 실제로
작동한다.** jawcode도 `priority`를 동일하게 보내므로 동일하게 적용된다.

→ 잘못된 `⚡?` "ignored" 표시는 **되돌렸다** (footer 무조건 ⚡, fastTier 해석 제거).
requested/realized 원시 값은 디버그용으로만 유지. 첫 A/B의 "속도 동일"은 plain SSE·비혼잡
시간대라 양쪽 모두 빠른 구간을 측정한 것으로 추정 (priority의 이점은 혼잡 시 큐 우선순위).

### 라이브 스모크 (tmux `jwcsmoke`, `/tmp/jwc-smoke`)

```
프롬프트1: "Read sample.txt and tell me how many lines"
  → ✔ Read sample.txt → "sample.txt has 4 lines." (정답)
  → 푸터: ⬢ GPT-5.5 ⚡? · ws · ◕ high   (fast 미실현 + ws 전송 동시 표시)
  → 로그: content prewarm completed {mode:cold, seededResponseId:true}
          codex websocket request {requestMode:delta, inputItems:1}
프롬프트2: "append a fourth line ... line four"
  → ✔ Edit → 파일에 line four 정확히 추가
  → 로그: requestMode:delta items:1 (턴 경계에서도 델타 유지)
```

핵심 입증: **프리웜이 컨텍스트를 미리 prefill → 실제 첫 메시지가 full이 아니라 delta(1
아이템)로 나갔다.** 이것이 T1이 노린 콜드스타트 TTFT 제거의 직접 증거다.

## 동시 세션 주의 (운영 메모)

이 작업 내내 `jwc` 라이브 세션이 같은 서브모듈에 동시 커밋·편집함. 모든 커밋은 pathspec
(`git commit -- <paths>`)으로 내 파일만 분리 커밋, `git reset` 미사용. 한 번 mixed 커밋 +
reset이 jaw 세션 커밋을 orphan시켜 reflog로 복구한 사례 있음 → [[jawcode-concurrent-jaw-session-git]].

## 종합 판정

전송 계층(00 밴드) 핵심은 전부 landed·라이브 검증됐다. "1.7 tok/s"의 클라이언트측 주범
(G5 O(n²) 렌더)은 제거됐고, 콜드스타트 TTFT는 콘텐츠 프리웜으로 delta화됐다. 남은 체감
지연 중 **서버측 저하(#24585)와 fast 미실현은 클라이언트가 못 고치는 외부 요인**임이
A/B로 확정됐다. G8/G9/G10/G2/P2는 효용·위험 대비 후속 사이클로 분리.
