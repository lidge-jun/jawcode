# A Synthesis R1 — Scroll second patch

Revision: `scroll-r1`

## Verdicts

- Planner lens: FAIL (`agent://78-ScrollPlannerAudit`)
- Architect lens: FAIL (`agent://79-ScrollArchitectAudit`)

## Shared root cause

Both audits agree the P plan correctly identified the bad branch, but the proposed replacement was underspecified and still used the same stale cursor source in the append phase:

- `viewportRepaint()` was home-anchored, but `appendGrowthThenViewportRepaint()` still derived position from `hardwareCursorRow - prevViewportTop`.
- The helper was split into two synchronized-output writes, creating a state gap between append and repaint.
- The mandatory test covered bottom/overflow behavior but not the user-reported scrolled-up viewport.
- The plan changed `structure/31_scroll.md` policy without owning the SoT update.

## Finding decisions

### Planner findings

- PLANNER-A1: accept. Make scrolled-up VirtualTerminal regression mandatory rather than optional.
- PLANNER-A2: accept. Replace stale relative append positioning with an absolute, single synchronized append+viewport repaint sequence and explicit state reconciliation.
- PLANNER-A3: accept. Keep multiplexer behavior explicit: legacy multiplexer flag still full-renders; normal multiplexer remains viewportRepaint-only unless a later tmux-specific plan proves append is safe/needed.
- PLANNER-A4: accept. Replace “composer floor” with concrete `viewport[viewport.length - 1] === "> input"` style assertion.
- PLANNER-A5: accept. Add `structure/31_scroll.md` doc sync to this cycle.
- PLANNER-A6: accept. Restrict Biome gate to this cycle’s touched files plus first-patch regression only when actually touched.

### Architect findings

- ARCH-A1: accept. Append phase must not use stale `hardwareCursorRow - prevViewportTop`.
- ARCH-A2: accept. Merge append and viewport repaint into one synchronized sequence and update renderer state once at the end.
- ARCH-A3: accept. Make off-bottom viewport regression mandatory with `scrollViewportUp()` / `getViewportY()` helpers.
- ARCH-A4: accept. Emit already-prepared `newLines` bytes for appended rows, matching `fullRender()` contract; do not re-truncate through a second helper unless tests prove required.
- ARCH-A5: accept. Modify `structure/31_scroll.md` in this cycle.
- ARCH-A6: accept. Restrict touched-file hygiene to actual files.
- ARCH-A7: accept. Existing shrink / extraLines / compact tests remain mandatory acceptance coverage for broader home-anchor change.

## Plan changes for revision scroll-r2

- Replace `appendGrowthThenViewportRepaint()` with one `appendGrowthAndRepaintViewport()` synchronized sequence:
  - absolute `\x1b[${height};1H` before append, not stale relative cursor math;
  - append prepared `newLines` tail bytes directly;
  - `\x1b[H` and repaint the target viewport in the same buffer;
  - compute final cursor state using the same cursor logic as `viewportRepaint()`;
  - update `#previousLines`, `#maxLinesRendered`, `#viewportTopRow`, width/height once after successful write.
- Branch policy:
  - `useLegacyMultiplexerFullRender()` remains full render;
  - `grew && !isMultiplexerSession()` uses append+viewport repaint;
  - normal multiplexer remains viewportRepaint-only.
- Add mandatory `VirtualTerminal` scroll helpers and scrolled-up regression.
- Add `structure/31_scroll.md` §5 doc update.
- Tighten acceptance commands.
