# A synthesis r1 — agent background contract

Revision audited: a-r1
Planner report: `41.6_a_planner_audit_r1.md`
Architect report: `41.7_a_architect_audit_r1.md`
Decision: accept all findings; revise plan to a-r2 and delta re-audit.

## Shared root cause

The P-stage plan was coherent at the feature level but left several integration contracts underspecified: background row ids are not uniformly subagent ids, canonical row sorting was still conditional, settings/keybinding discovery promised more runtime access than `ToolSession` has, and prompt scope did not include every legacy bg-adjacent tool/system prompt.

## Local evidence checked

- `BackgroundRowView` has only `id`, not a dedicated `subagentId` field; live job rows use `job.id`, queued subagent rows use `record.subagentId`.
- `AsyncJobManager.cancelSubagent(subagentId, filter)` expects a stable subagent id, while `readOutputSince(jobId, offset, filter)` expects an async job id.
- `ToolSession` exposes `settings` but no `KeybindingsManager` accessor.
- `tools/renderers.ts` has an explicit renderer map.
- Existing tool prompt imports use `with { type: "text" }`.

## Finding decisions

- PLANNER-A1: accepted. Add explicit `follow` result status shape with `unsupported` and tests.
- PLANNER-A2 / ARCH-A2: accepted. Add `resolveBackgroundIds()` algorithm for `sub` rows: first treat `row.id` as subagent id, then job id metadata, then scan current/historical job ids. Separate `jobIds` for follow from `subagentId` for cancel.
- PLANNER-A3: accepted. Add explicit attention flag/detail item.
- PLANNER-A4: accepted. Add `system-prompt.md` conditional guidance.
- PLANNER-A5: accepted. Add `monitor.md` cross-reference.
- PLANNER-A6 / ARCH-A4: accepted. Add verified output-ref helper contract; omit refs when proof is absent.
- PLANNER-A7 / ARCH-A1: accepted. Mandate new neutral `modes/background-row-model.ts` canonical helper.
- PLANNER-A8: accepted. Require `try/finally observer.dispose()` and leak-guard test.
- PLANNER-A9: accepted. Pin `tools.test.ts` registry cases.
- ARCH-A3: accepted. V1 settings returns `KEYBINDINGS` default for `app.background.expand` with source `keybinding-default`; no ToolSession keybinding accessor.
- ARCH-A5: accepted. Mirror JobTool factory gating only; no extra `isToolAllowed` branch.
- ARCH-A6: accepted. Specify text import syntax.

Subagent verified output refs may require careful artifact metadata lookup. The revised plan explicitly says omit refs when verification is not provable, so implementation can pass without fabricated refs.
