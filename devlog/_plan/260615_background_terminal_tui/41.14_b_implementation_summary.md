## Implemented

- Added discoverable built-in `background` tool with `list`, `detail`, `follow`, `cancel`, and `settings` operations.
- Preserved existing launch paths (`task`, `bash`, `monitor`, cron tools, and legacy `job`) and made `background` the canonical agent-facing row management surface.
- Added neutral `sortBackgroundRows()` / `isBackgroundAttention()` row model shared by JobsObserver and footer panel model.
- Implemented owner-scoped row lookup and id resolution for job-backed rows, queued subagent records, and task-backed subagent job metadata.
- Implemented verified `agent://<subagentId>` detail refs only when artifact output and metadata both exist.
- Implemented bounded `follow` output with cursor offsets and byte truncation.
- Implemented cancel matrix:
  - running `sh`/`mon` rows cancel through `AsyncJobManager.cancel(jobId, ownerFilter)`;
  - paused `sh`/`mon` rows return `unsupported`;
  - `cron` rows return `unsupported` and leave deletion to CronDelete;
  - queued/running/paused subagent records cancel through `cancelSubagent(subagentId, ownerFilter)`;
  - metadata-only task/sub rows cancel through their job id, not through `cancelSubagent`;
  - terminal/latched rows return `already_terminal`;
  - cross-owner/unknown ids return `not_found`.
- Added prompt guidance for background launch timing, TUI footer parity, factual progress commentary, and legacy `job` vs canonical `background` split.
- Registered renderer and built-in factory without making `background` essential by default.
