## WONDER — what's still missing?

- No explicit synthetic `q` row test exists because current `AsyncJob` rows are `bash` or `task`; the implementation has the defensive `q` unsupported branch, but a future producer of `q` rows should add a direct fixture.
- The list-order test proves failed/attention rows sort first but could be hardened with an explicit full expected id sequence instead of also comparing against `sortBackgroundRows(rows)`.
- `detail` for unknown ids returns text plus `{ op: "detail" }`; follow/cancel have structured `not_found`. The plan did not require structured detail-not-found, but adding it would make the API more uniform.

## REFLECT — how to improve the spec

- Acceptance criteria should explicitly state whether `detail` needs a structured `not_found` object.
- The cancel matrix should define how to construct/test future `q` rows, not just their desired behavior.
- The workflow spec should handle missing `/skill:dev-testing` gracefully; this repo currently points C-stage at a skill that is not installed.
- Commit/staging guidance should account for files with mixed unrelated hunks when multiple agent sessions share one working tree.
