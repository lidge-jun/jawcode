## Resolutions

- Handler lifecycle owner: `InteractiveMode.toggleBackgroundFooterPanel()` is the single lifecycle owner. It calls `InputController.installBackgroundFooterPanelHandlers()` on expand and `removeBackgroundFooterPanelHandlers()` on collapse. InputController does not independently decide lifecycle.
- Esc precedence: install wraps current `editor.onInterruptPriority`; collapse restores previous handler. Wrapper order: if detail overlay active, no-op/let detail handle; else if panel expanded, collapse and return true; else call previous handler. This preserves BTW/tool-focus behavior.
- Prompt helper: `#prepareRealUserAgentPromptSubmission()` is sole sequence/ack helper. It is called by `withLocalSubmission` and direct prompt paths, including `packages/coding-agent/src/modes/utils/ui-helpers.ts` compaction first-prompt path or a routed replacement through `withLocalSubmission`.
- Retained rows: `#retainedTerminalRows` stores denormalized row snapshot plus `outputPreview`/`errorPreview`/`resultPreview`; detail reads retained data after AsyncJob eviction.
- Status line: cycle 1 suppresses/removes `status-line.ts` non-monitor `runningBackgroundJobs` block when footer-owned `sh/sub` rows are visible. Segment docs repeat the rule.
- Detail precedence: detail Esc/back closes detail and restores editor focus first; panel interrupt wrapper no-ops while detail overlay is active.
