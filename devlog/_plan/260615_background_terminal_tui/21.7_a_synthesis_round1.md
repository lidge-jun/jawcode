## Decision summary

All blocking Planner and Architect findings are accepted. No waivers. The plan is revised for `a-r2` delta audit.

## Shared root causes

1. **Snapshot reshape too broad.** The plan removed/renamed `JobsSnapshot` fields while existing `jobs-overlay-model.ts`, `status-line.ts`, `status-line/segments.ts`, and tests still depend on monitor/cron fields.
2. **Panel input path was conceptual, not code-real.** The plan described `handleInput(): boolean` on an unfocused component, conflicting with TUI `Component.handleInput?: void` and `CustomEditor` key consumption.
3. **Terminal latch lifecycle lacked an owner.** The plan required `visibleSinceUserMessageSeq` and a later real user-agent message but did not name the field, sequence owner, mark-before-render hook, or all prompt callsites.
4. **Detail UI lacked a concrete host.** The plan allowed conditional overlay reuse vs a small panel without assigning a file/contract.
5. **Older MOC slices remained authoritative-looking.** The MOC still contained superseded `BackgroundJobsStrip` / Ctrl+B / above-status-container design fragments.
