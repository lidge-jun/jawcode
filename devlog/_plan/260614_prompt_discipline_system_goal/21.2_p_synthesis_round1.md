# 21.2 P Synthesis Round 1 — Jaw-Interview Handoff Cleanup

Date: 2026-06-15
Critic receipt: `21.1_p_critic_round1.md`
Original plan: `21_p_jaw_interview_handoff_cleanup_plan.md`
Revised plan: `21.3_p_jaw_interview_handoff_cleanup_plan_r2.md`

## Decision summary

All Critic findings are accepted. The revised plan tightens insertion points, reset ordering, concrete test helpers, concrete jaw-interview CLI argv, HUD assertions, and error-contract language. The unused `idle` reason is removed from the planned type surface. The `complete` hook is removed from implementation scope because it was not part of the user's requested P/idle cleanup and expanded acceptance without direct evidence.

## Finding decisions

### P21-1 — stage hook not anchored to `runNativeOrchestrateCommand` post-checkpoint

Accepted. The revised plan anchors P cleanup immediately after `recordGoalCheckpointForTransition(...)` and before JSON/text return in `runNativeOrchestrateCommand()`. This ensures PABCD persistence and goal fusion have already succeeded.

### P21-2 — reset retire must run only after successful unlink per target

Accepted. The revised plan places reset cleanup inside the per-target non-dry-run success branch after `fs.rm(target.path, { force: true })` succeeds and after `removed.push(...)`, not before or outside the target loop.

### P21-3 — seed/read helpers are placeholders

Accepted. The revised plan gives concrete helper bodies for session path encoding, jaw-interview state path, active-state snapshot seeding, and JSON reads in `orchestrate-state.test.ts`.

### P21-4 — missing concrete `--write`/`--json` argv

Accepted. The revised plan uses explicit `runNativeJawInterviewCommand(["--write", "--stage", "final", "--slug", "retire-me", "--spec", "# Spec", "--json"], root)` and similar argv for active interview seeding.

### P21-5 — redundant double-write in guard test seed

Accepted. The revised plan updates the existing `writeActiveJawInterview()` helper to accept an optional mode-state extension instead of overwriting the same state twice inside the test.

### P21-6 — HUD/skill-active-state not asserted in orchestrate tests

Accepted. The revised plan asserts both the mode state and `skill-active-state.json` active entry are inactive after P/reset cleanup.

### P21-7 — idle exit reason unused

Accepted. The revised type is narrowed to `"orchestrate-p" | "orchestrate-reset"`. No unused `idle` reason remains.

### P21-8 — fail-open read vs fail-closed write not unified

Accepted. The revised contract is explicit: absent/corrupt jaw-interview state returns `false` and does not block PABCD; write/sync failures throw and fail the PABCD command because a discovered stale state could not be retired safely.

### P21-9 — complete retire wired but not in acceptance

Accepted. The revised plan removes `orchestrate-complete` wiring from this patch. P and reset/idle are sufficient for the user report.
