# 20.14 B — hidden-thinking full transcript patch

## Root cause

The Ctrl+T full transcript protocol expanded thinking blocks but still respected the normal UI `hideThinkingBlock` flag. When a session was reopened with hidden thinking enabled, historical assistant thinking segments could render only `Thinking...` instead of the actual thinking body.

## Code changes

- `packages/coding-agent/src/modes/components/assistant-message.ts`
  - `renderFullTranscript(width)` now records both `#thinkingExpanded` and `hideThinkingBlock`.
  - It temporarily sets `hideThinkingBlock=false` and `#thinkingExpanded=true` while rendering the full transcript.
  - It restores both values in `finally`, preserving normal Ctrl+O/live UI hidden-thinking state after Ctrl+T closes/rerenders.

## Test changes

- `packages/coding-agent/test/full-transcript-overlay.test.ts`
  - Live assistant full transcript coverage now constructs `AssistantMessageComponent` with `hideThinkingBlock=true`.
  - Asserts Ctrl+T/full transcript output contains `LIVE_THINK_2`.
  - Asserts normal rendering after full transcript hides `LIVE_THINK_2` and returns to `Thinking...`.
  - Adds historical session replay coverage with `hideThinkingBlock=true`, asserting `HIST_THINK_2` appears in the full transcript and normal render state is restored afterward.
- `packages/coding-agent/test/input-controller-keybindings.test.ts`
  - Persisted reopened-session Ctrl+T regression now sets `ctx.hideThinkingBlock=true` and still expects `OLD_THINK_2` in the overlay.
