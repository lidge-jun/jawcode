# 20.16 B — stale hidden-thinking placeholder patch

## Root cause

The prior hidden-thinking fix used `setThinkingExpanded(true)` after setting `hideThinkingBlock=false`. If `#thinkingExpanded` was already true, the setter returned early and did not rebuild content. That left the normal hidden `Thinking...` placeholder cached in the component when Ctrl+T rendered.

## Code changes

- `packages/coding-agent/src/modes/components/assistant-message.ts`
  - `renderFullTranscript(width)` now writes `this.#thinkingExpanded = true` directly after `hideThinkingBlock=false`.
  - It calls `updateContent(this.#lastMessage)` when a last message exists, forcing a rebuild even if thinking was already expanded.
  - In `finally`, it restores both `hideThinkingBlock` and `#thinkingExpanded`, then rebuilds content again so normal UI state is restored.

## Test changes

- `packages/coding-agent/test/full-transcript-overlay.test.ts`
  - Adds `shows hidden live thinking in full transcript even when already expanded`.
  - The test creates a live assistant component with `hideThinkingBlock=true`, calls `setThinkingExpanded(true)` before opening the overlay, verifies normal render shows `Thinking...` and not `LIVE_ALREADY_2`, verifies Ctrl+T/full transcript shows `LIVE_ALREADY_2`, and verifies normal hidden render is restored after overlay render.
