# P Synthesis R2 — Critic R2 Response

## R2-1 (raw registry ops) — Accept, impl-level
`attemptFreshFallback` allocates a NEW actor (different from closure-bound `actorRouting.actor`), so `updateWorkflowActor` (which mutates the original actor) is not appropriate for the fresh actor's operations. Raw registry ops are the correct approach here. Will add a comment.

## R2-2 (stale closure rebind) — Accept, critical fix
On isolated fallthrough after fresh success, `finalizeWorkflowActorResult` must NOT be called because it's bound to the old actor. Solution: `attemptFreshFallback` handles ALL finalization for both failed-resume and fresh actors internally. On success, it returns `{ result, freshActor }` so the caller can set up a new `finalizeWorkflowActorResult` closure. On the isolated fallthrough path, rebind `finalizeWorkflowActorResult` to the fresh actor before merge/patch post-processing.

## R2-3 (next-call AC) — Accept, add to ACs
"When the registry has a `failed` actor, the next call allocates `runMode: initial` directly (Change 1A guarantees this)."

## R2-4 (integration tests) — Partially accept
Will add targeted tests during implementation. Not all test shapes need plan-level specification.

## R2-5 (double finalize) — Accept
Isolated fresh-failure path should NOT call `finalizeWorkflowActorResult` again — `attemptFreshFallback` already finalized. Remove the redundant call.

## R2-6 (aborted) — Accept
Add `result.aborted` exclusion to the predicate if the `aborted` field exists on SingleResult.

## R2-7 (thrown errors) — Waive
Thrown `runSubprocess` errors are infrastructure failures (not resume-specific), out of scope for resume→fresh fallback.
