## REFLECT — how to improve the spec next time

- Acceptance criteria should distinguish “exact stale literal absent” from “negative/removal guidance may mention a concept,” because verifier parsers can overmatch literal strings in receipts or plans.
- Rollback/default override variables should never be named until the implementation repo verifies them; use `TBD — cli-jaw checkout required` first.
- The ontology should separate three surfaces from the start: native Code mode, existing Jaw mode JWC runtime attach, and shared auth/model inventory. Mixing those caused most interview churn.
