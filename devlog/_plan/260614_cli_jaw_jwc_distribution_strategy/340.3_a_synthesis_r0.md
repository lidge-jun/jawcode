# 340.3 — A-stage synthesis r0

Status: both Stage-A lenses FAIL for revision `a-r0`; all findings accepted.

## Original design intent

Make `jawcode` npm distribution release-ready without turning `jawcode` into a cli-jaw session manager:

- `npm install -g jawcode` should install the `jwc` bin and package-local Bun path.
- `jawcode` postinstall should bootstrap the shared cli-jaw-compatible home contract (`CLI_JAW_HOME` or `~/.cli-jaw`) like cli-jaw postinstall does.
- The bootstrap should create a reproducible non-secret baseline: deterministic settings seed, heartbeat, empty MCP config, GitHub skills catalog, and default/orchestration active skills.
- No `./cli-jaw` checkout, no copying the maintainer's live home, no secrets/personal state.
- Release readiness includes CI/release gates and packed install proof; actual npm publish remains out of scope until explicit publish approval.

## Shared root cause

The P-stage plan was revised through multiple interview corrections, but the gate artifact still pointed implementers at the older `330` slice and stale `240` assumptions. The result was a mixed plan: strategy decisions live in `340`/pending approval, while implementation details in `330` still say `cli = "claude"`, omit package-local Bun smoke hardening, and do not enumerate CI/release/native gates.

## Chosen compromise

Revise the plan, not the product code, in this A-stage:

1. Keep `jawcode` package-boundary scope.
2. Make `settings.json` postinstall seed an intentional `jawcode` extension for the no-cli-jaw install requirement.
3. Canonicalize `cli = "codex"` across `330`, pending approval, and acceptance tests.
4. Add a `CLI_JAW_HOME` resolver/discovery slice before claiming custom-home support.
5. Constrain skill bootstrap implementation to a minimal Node CJS surface plus drift-check tests against cli-jaw behavior, rather than unbounded copy-paste.
6. Make release validation canonical across local script, CI workflow, release workflow, and publish preBuild.
7. Mark native heavy probes as macOS-arm64 release gates where practical, with report-only/non-mac skip semantics where appropriate.
8. Explicitly defer generated `.d.ts` as not required for this release.
