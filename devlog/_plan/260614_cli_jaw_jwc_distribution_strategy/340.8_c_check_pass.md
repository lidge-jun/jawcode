## Scope checked

Release-readiness for the `jawcode` npm package, including:

- first install for users without pre-existing `cli-jaw`;
- cli-jaw-compatible home bootstrap under `~/.cli-jaw` or `CLI_JAW_HOME`;
- non-secret settings/heartbeat/mcp seeding;
- GitHub skills download/activation where git/network are available;
- safe/CI/skip bootstrap modes;
- exclusion of heavy skill artifacts such as `node_modules`;
- package-local Bun proof;
- registry-faithful install proof;
- macOS arm64 native dependency probes;
- CI and release workflow gates;
- no publish until an explicit release approval/manual workflow publish.

## C-stage adversarial review findings and fixes

Initial read-only adversarial review found concrete issues:

1. `bootstrap-cli-jaw-home.cjs` created empty `skills/` before a successful skills download, which could suppress native user-skill fallback after a failed clone.
2. The packed smoke inherited `CI=true` on GitHub Actions, causing postinstall to safe-skip skills while the smoke expected activated fixture skills.
3. Native probes were only meaningful on macOS arm64 but CI only had Ubuntu package gates.
4. Release workflow lacked the CI package file-count/bin syntax guards.
5. Publisher contract test did not lock the full JWC prebuild gate list.
6. `session-transcript-replay.ts` had a current-tree missing brace in the read-tool replay branch, causing `bun --cwd=packages/jwc run bundle` to fail with `Unexpected end of file`.

Applied fixes:

- Bootstrap no longer creates empty `skills/`/`skills_ref/` before successful source merge/activation.
- Failed clone in postinstall remains nonfatal but does not leave an active empty skills root.
- Safe/CI mode skips network while activating from existing `skills_ref` when present.
- Clone cooldown no longer blocks retry when there is no existing `skills_ref/registry.json`.
- Packed smoke and bootstrap tests clear inherited `CI`/safe/skip env for normal install-contract assertions; matrix cases opt into those modes explicitly.
- CI and release workflows now include `mac-native-probes` on `macos-14`; GitHub documentation identifies `macos-14` as an arm64 hosted runner label: https://docs.github.com/en/actions/reference/runners/github-hosted-runners and https://github.com/actions/runner-images.
- Release workflow now gates publish on mac native probes and includes Node bin syntax + pack file-count guards.
- `release-publish-order.test.ts` now asserts the full eight-step JWC prebuild contract.
- The read-tool replay branch in `session-transcript-replay.ts` now closes the error branch and outer read branch correctly.

Final read-only recheck receipts:

- `agent://17-BootstrapFixRecheck`: PASS, no findings.
- `agent://18-ReleaseFixRecheck`: PASS, only P3 hardening note; hardening applied by clearing CI/safe/skip in `scrubbedPathEnv` too.
