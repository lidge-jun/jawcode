# 100.02 — Bun 글로벌 셰임 주입 골격

> 상태: ✅ **완료 (260613)** — shims/{index,types,bun-object}.ts 신설, esbuild inject 배선.
> 빌드 green·번들에 셰임 포함(`__jwcNodeShim` 마커 확인). Node import 실패 지점이
> `globalThis.Bun` 부재가 아니라 `import "bun"` 모듈 resolve로 이동 (100.03+ 별칭 대상).
> 전 스텁은 API명을 박은 명시 throw — 첫 런타임 실패가 누락 API를 자가 보고.

> 상위 문서: [100_moc_node_porting.md](./100_moc_node_porting.md)
> 인벤토리: [phase1/100.1_plan_bun_shim_inventory.md](./phase1/100.1_plan_bun_shim_inventory.md)

## 목표

Node 빌드가 `globalThis.Bun` 부재로 즉시 깨지지 않도록 `packages/jwc/src/shims/`를 만들고, esbuild `inject`로 Node 번들 시작 전에 최소 Bun 호환 객체를 주입한다.

100.02는 “모든 API 동작 완성”이 아니라 **셰임 주입 경로와 인터페이스 골격 고정**이 목표다.

## 범위

### 포함

- `packages/jwc/src/shims/index.ts` 생성
- `globalThis.Bun` 존재 여부 감지
- Node일 때 shim object 설치
- Bun일 때 native Bun 그대로 통과
- esbuild config에 `inject: ["src/shims/index.ts"]` 연결
- 타입 충돌을 피하기 위한 최소 global 선언 방식 결정

### 제외

- SQLite/hash/JSONL/serve/archive의 완전 구현
- 테스트 러너 전환
- 실 프로바이더 스트리밍

## 설계

```ts
const nativeBun = globalThis.Bun;

if (!nativeBun) {
  Object.defineProperty(globalThis, "Bun", {
    value: buildNodeBunShim(),
    configurable: true,
    writable: false,
  });
}
```

원칙:

- Bun 런타임에서는 절대 override하지 않는다.
- Node 런타임에서만 설치한다.
- 100.02의 shim method는 필요한 곳에서 명시적으로 `throw new Error("Bun.X shim not implemented")`를 내도 된다.
- 단, `Bun.env`는 esbuild define으로 `process.env` 처리하므로 shim object에 의존하지 않는다.

## 파일

| 파일 | 역할 |
|------|------|
| `packages/jwc/src/shims/index.ts` | global install entry |
| `packages/jwc/src/shims/types.ts` | shim 내부 타입 |
| `packages/jwc/scripts/build-node.ts` | inject 설정 추가 |

## 구현 순서

1. `packages/jwc/src/shims/` 생성
2. `index.ts`에서 Node 감지 + shim install
3. `build-node.ts`에 inject 연결
4. `bun --cwd=packages/jwc run build:node` 실행
5. 실패가 특정 API stub error 또는 resolve error로 이동했는지 기록

## 완료 기준

- `packages/jwc/src/shims/index.ts` 존재
- Node 빌드가 shim entry를 포함함
- Bun 실행 경로에서는 native Bun을 덮어쓰지 않음
- 첫 실패 지점이 `globalThis.Bun is undefined`가 아님
