# 112.4 — design: Code 모드 UI 시안 (jaw ↔ Code)

> 상태: 🟡 시안 v1 (260613) — 사용자 보정 전제 초안. **목업**: [112.4_preview.html](./112.4_preview.html) (브라우저로 열기).
> 상위: [112_moc_gui.md](./112_moc_gui.md) D112-1 · 구현 짝: [112.3_plan_code_mode_impl.md](./112.3_plan_code_mode_impl.md) S2~S4 컴포넌트가 본 시안을 따른다.
> 디자인 근거: cli-jaw `public/css/variables.css` 토큰 **전량 재사용** (신규 토큰 금지) + Claude Code 데스크톱의 Chat|Code 전환 패턴 차용.

## 1. 디자인 규칙 (불변)

1. **토큰 재사용**: 색·간격·라운드·트랜지션 전부 `variables.css` 기존 변수 (`--bg`/`--surface`/`--accent`(cyan 200)/`--space-N`/`--radius-N`/`--ease-out-expo`). 신규 CSS 변수 0개 — Code 모드 전용 값이 필요하면 기존 토큰 조합(`color-mix`)으로.
2. **폰트**: 디스플레이 = Chakra Petch, UI = Outfit/Pretendard, 코드·경로·도구 출력 = `--font-mono` (`index.html:31` 로딩 체계 그대로).
3. **다크/라이트**: `[data-theme="light"]` 오버라이드 자동 상속 — Code 모드 전용 색 분기 금지.
4. **모드 명칭**: **`jaw`** | **`Code`** — "Chat"이라는 단어는 UI 어디에도 쓰지 않는다 (우리는 chat이 아니고 jaw다).
5. **jaw 모드 무변경** (D112-1): 기존 UI에 추가되는 것은 **모드 토글 1개뿐**. 나머지는 Code 모드 신규 표면.
6. 레트로 터치(`--noise-opacity`/`--scanline-pct`/shark 🦈)는 jaw 모드 아이덴티티 — Code 모드는 **밀도 우선**으로 노이즈/스캔라인 미적용, 단 상어 액센트(로고·빈 상태)는 유지.

## 2. 토글 위치 (결정)

- **위치**: 좌측 사이드바 로고(`CLI-JAW`, `index.html:61`) **바로 아래** 세그먼트 컨트롤 `[ jaw | Code ]` — Claude Code 데스크톱이 Chat|Code를 최상위 내비로 두는 패턴과 동형. 헤더(`chat-header`)는 검색/테마 액션으로 이미 포화라 기각.
- 형태: pill 세그먼트 (`--radius-full`), 활성 = `--accent` 배경 + `--text-on-accent`, 비활성 = `--text-dim`. 사이드바 collapse(48px) 시 아이콘 2개(🦈/`</>`)로 축약.
- 전환 시 좌측 사이드바 **하부 콘텐츠가 모드별로 스왑** (jaw: 상태/메모리/통계/CLI 상태 그대로 · Code: 워크스페이스+세션 목록). 우측 사이드바(에이전트/스킬/설정)는 jaw 전용 — Code 모드에서 숨김(2컬럼 그리드).

## 3. Code 모드 레이아웃

```
┌──────────┬─────────────────────────────────────────────┐
│ CLI-JAW  │ code-header: ~/repo ⎇ branch · model · usage │
│ [jaw|Code]├─────────────────────────────────────────────┤
│──────────│ transcript                                   │
│ WORKSPACE│   user bubble (--user-bg)                    │
│  repo ⎇  │   ▸ ThoughtCell (접힘, --text-dim)           │
│ SESSIONS │   ▸ ToolCell completed (컴팩트 1줄)           │
│  ● conv1 │   ▾ ToolCell running (--status-running)      │
│  ○ conv2 │   ✕ ToolCell failed (--error)                │
│  ↳ fork  │   assistant text (markdown)                  │
│ [+ 세션] │   [PermissionDialog — 인라인 카드]            │
│          ├─────────────────────────────────────────────┤
│          │ composer (mono placeholder · ⏎ 전송)         │
└──────────┴─────────────────────────────────────────────┘
```

- **세션 항목**: 활성 `●`(accent) / idle `○` / fork는 `↳` 들여쓰기 + `forkedFrom` 칩 (D112-4). 상태 점은 `--status-running-color`(스트리밍)/`--success`(완료).
- **ToolCell 상태 머신** (112.3 S2): `pending`(dim) → `in_progress`(running 배경 + 스피너) → `completed`(1줄 컴팩트: 아이콘·라벨·소요·⌄) / `failed`(error 테두리). 클릭 토글로 출력 확장 — 확장부는 `--code-bg` mono.
- **ThoughtCell**: 기본 접힘 `▸ 추론 n초` — 클릭 확장, 본문 `--text-dim`.
- **PermissionDialog**: 모달 아닌 **트랜스크립트 인라인 카드** (`--warning` 좌측 보더) — "이번만 허용 / 세션 전체 허용 / 거부" 3버튼. 동시 요청은 세션별 큐 (112.3 Q3 권장안).
- **WorkspacePicker**: 세션 0개 빈 상태에서 중앙 카드 — 최근 레포 목록(`git-info` API) + 경로 입력. 🦈 빈 상태 일러스트 자리.
- **code-header**: 워크스페이스 경로(mono) + `⎇ branch` 칩 + 모델 칩(클릭=S4 팝업) + 사용량 배지(`usage_update`, USD 4자리). jaw 헤더(`CLI-JAW ● claude`)와 정보 위계 동형.

## 3.1 변경 팝업 3종 (S4 — 목업 포함)

- **모델 팝업** (모델 칩 앵커 팝오버): **그룹 프리셋** 구조 — Claude/Codex/Local 그룹 라벨 + 모델 행(mono 이름 · capability 칩(1M/think/fast) · `● 현재`). upstream `a12a751` group preset UX(사용자 패치 중)와 컨셉 정합. 하단에 **Service Tier** 섹션 병합 (standard/priority) — 별도 팝업 안 만든다.
- **턴 중 변경**: 행에 `다음 턴 적용` pending 배지 (`--status-running-color`) — 113.1 `202 {pending:true}` 와이어의 UI 짝.
- **thinking 팝업** (think 칩 앵커): off/low/med/high 세그먼트 (`--radius-full` pill) — 즉시 적용, `config_option_update` 수신으로 확정 표기.
- **상호배제**: 팝오버는 동시에 1개만 열림 + 외부 클릭 닫기 — 113.1 "409 미사용, UI 모달 상호배제" 원칙의 구현.
- 워크스페이스 피커는 §3 (모달), 권한은 인라인 카드 — 팝업 위계: 인라인 카드 > 앵커 팝오버 > 중앙 모달 순으로 가볍게.
