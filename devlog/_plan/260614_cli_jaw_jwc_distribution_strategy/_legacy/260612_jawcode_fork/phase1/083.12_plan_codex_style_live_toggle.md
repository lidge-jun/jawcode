# 083.12 — plan: Codex식 live-only 출력 토글 정렬

> 상태: ⚪ 이동됨 — canonical 기록은 `devlog/_plan/260614_tui_codex_live_toggle/00_moc.md`
> 입력: 사용자 결정 — “Codex 식으로 가자. thinking 매핑은 없애고 그냥 현재 thinking/tool 전부 ctrl+o로 열고, ctrl+t는 후속 phase로.”
> 관계: [083.11](./083.11_issue_global_toggle_commit_lane.md) 조사 결론의 실행 phase.

## 결정

jwc TUI의 출력 접기 UX를 Codex RS 계열로 정렬한다.

- normal prompt scrollback은 **항상 canonical collapsed 상태**로 커밋한다.
- `ctrl+o`는 **현재 live / 미커밋 구간**의 tool + thinking 전체 토글만 담당한다.
- `ctrl+t`의 thinking 전용 토글 매핑은 제거한다.
- full transcript overlay/screen은 별도 후속 phase로 남긴다.

이 결정은 “이미 scrollback에 찍힌 과거 픽셀은 일반 토글로 되돌릴 수 없다”는 083.11 원인 분석을 전제로 한다. 전역 가역 토글을 유지하려면 Codex RS의 raw-output/resize reflow처럼 source-backed full clear+rewrite가 필요하지만, 그건 083.9 commit lane의 핵심 목표(긴 세션 무깜빡임, scrollback 보존, 3J 회피)와 충돌한다.

## 목표 UX

### Prompt screen / normal chat

- 과거 턴:
  - tool output, read/search group, thinking block은 collapsed canonical 형태로 scrollback에 남는다.
  - 한 번 과거로 넘어간 구간은 `ctrl+o`로 다시 열거나 닫지 않는다.
- 현재 턴 / live zone / 아직 커밋되지 않은 마지막 턴:
  - `ctrl+o`: tool + thinking 전체 펼침/접힘.
  - 새 사용자 메시지 제출 시 해당 구간은 collapsed canonical 형태로 커밋된다.
- `ctrl+t`:
  - 이 phase에서는 thinking toggle로 쓰지 않는다.
  - full transcript overlay/screen 후속 구현을 위해 예약하거나 unbound 처리한다.

### Full transcript 보기

- 현재 jwc의 `alt+t`는 tool transcript overlay다. Codex RS의 `ctrl+t` transcript overlay는 전체 conversation transcript라 범위가 다르다.
- 따라서 `ctrl+t`를 곧바로 `alt+t`에 매핑하지 않는다.
- 후속 phase에서 full transcript overlay/screen을 설계한다.

## 비교 기준

### Codex RS 정렬점

- Codex RS normal chat은 finalized `HistoryCell`을 terminal scrollback에 append한다.
- `ctrl+t`는 접기 토글이 아니라 alternate-screen transcript overlay다.
- `alt+r` raw-output toggle이나 resize reflow 같은 특수 이벤트에서만 source-backed clear+rewrite를 수행한다.
- jwc 083.12는 이 중 normal scrollback 불변성과 overlay 분리를 따른다.

### Claude Code와의 차이

- Claude Code의 `ctrl+o`는 prompt ↔ transcript screen 전환이다.
- jwc는 이번 phase에서 screen 전환까지 가지 않는다.
- jwc의 작은 변경안은 `ctrl+o`를 live-only output toggle로 유지하고, full transcript는 별도 후속이다.

## 구현 범위

### 1. `ctrl+t` thinking toggle 제거

대상:

- `packages/coding-agent/src/config/keybindings.ts`
  - `app.thinking.toggle` 기본키 `ctrl+t` 제거 또는 unbound 처리.
  - 설명도 “Toggle thinking mode”/“Toggle thinking block visibility”류에서 제거 또는 deprecated 표시.
- `packages/coding-agent/src/modes/components/custom-editor.ts`
  - default action key 목록에서 `app.thinking.toggle: ["ctrl+t"]` 제거.
- `packages/coding-agent/src/modes/controllers/input-controller.ts`
  - `onToggleThinking` 배선은 당장 남길 수 있지만 기본키가 없어야 한다.
  - 장기적으로 `toggleThinkingBlockVisibility()`는 삭제 후보. 단, 사용자 커스텀 keybinding 호환을 당장 유지할지 별도 판단 필요.
- 문서/핫키:
  - `packages/coding-agent/src/modes/utils/hotkeys-markdown.ts`
  - `packages/coding-agent/src/modes/components/keybinding-hints.ts` 관련 노출 경로 확인.

원칙:

- 기본 UX에서 `ctrl+t`가 thinking 전용 토글로 동작하면 안 된다.
- 커스텀 keybinding까지 즉시 hard-delete할지는 별도 선택이다. 기본 동작 제거가 필수.

### 2. `ctrl+o`를 live-only tool + thinking toggle로 정의

현재 코드:

- `InputController.toggleToolOutputExpansion()`
  - `const expanded = !this.ctx.toolOutputExpanded`
  - `this.ctx.thinkingExpanded = expanded`
  - `setToolsExpanded(expanded)`
- `setToolsExpanded()`
  - `chatContainer.children`와 `liveToolContainer.children` 전체에 `setExpanded(expanded)` 호출.

수정 방향:

- `ctrl+o`는 커밋된 과거 컴포넌트를 대상으로 삼지 않는다.
- 대상은 다음 중 하나로 제한한다.
  1. `liveToolContainer.children`
  2. `chatContainer.children` 중 `!child.committed`인 미커밋 컴포넌트
  3. `streamingComponent`
- thinking도 같은 live-only 규칙을 따른다.
- status 문구는 “current turn” 또는 “live output” 범위를 드러낸다.

예상 패치 형태:

```ts
for (const child of this.ctx.chatContainer.children) {
  if (child.committed) continue;
  if (isExpandable(child)) child.setExpanded(expanded);
}
for (const child of this.ctx.liveToolContainer.children) {
  if (isExpandable(child)) child.setExpanded(expanded);
}
this.ctx.streamingComponent?.setThinkingExpanded(expanded);
```

단, `AssistantMessageComponent.setExpanded()`가 thinking만 바꾸므로 tool component와 assistant component가 같은 protocol을 공유하는 현재 구조는 유지 가능하다.

### 3. commit 시 canonical collapsed 강제

문제:

- 사용자가 `ctrl+o`로 펼친 상태에서 새 prompt를 보내면, `commitFinalizedBacklog()`가 펼친 렌더 결과를 `commitLines()`에 넘길 수 있다.
- 그러면 과거 scrollback이 열린 상태로 굳는다.

수정 방향:

- `commitFinalizedBacklog()`가 커밋할 때는 화면의 live expanded 상태와 무관하게 collapsed canonical lines를 생성해야 한다.

후보 구현:

#### A. 일시 상태 전환

- `child`가 expandable이면 현재 expanded 상태를 저장하고 `setExpanded(false)` 후 render, 다시 원복한다.
- 단순하지만 모든 component가 현재 expanded 상태 getter를 제공하지 않는다.
- `ToolExecutionComponent`는 `expanded` getter가 있는지 확인 필요. `AssistantMessageComponent`는 thinking expanded getter가 없을 가능성이 있다.

#### B. `renderCommitted(width)` 프로토콜 추가

- 커밋 가능한 component에 optional method를 추가한다.
- `commitFinalizedBacklog()`는 `renderCommitted?.(width) ?? render(width)`를 사용한다.
- 각 component가 자기 canonical collapsed 렌더를 책임진다.
- 명시적이고 테스트하기 쉽다.

권장: B.

예상 인터페이스:

```ts
interface CommitRenderable {
  renderCommitted(width: number): string[];
}
```

또는 기존 `Expandable` 근처에 optional guard:

```ts
function hasCommittedRenderer(obj: unknown): obj is { renderCommitted(width: number): string[] } { ... }
```

대상 component:

- `ToolExecutionComponent`
- `ReadToolGroupComponent`
- `AssistantMessageComponent` thinking summary
- `CustomMessageComponent` / `SkillMessageComponent` 등 expandable류는 필요 여부 확인
- `BashExecutionComponent` / `EvalExecutionComponent`는 083.10 잔여 셀 타입 커밋과 연결됨

### 4. full transcript overlay 후속 phase 예약

별도 문서로 분리할 후속:

- `ctrl+t`를 full transcript overlay/screen으로 재사용할지 결정.
- 현재 `alt+t` tool transcript overlay와 통합/대체 여부 설계.
- Codex RS식 alternate-screen transcript overlay 후보:
  - committed session messages + active live tail
  - tool output full text
  - thinking full text
  - search/scroll support
- Claude Code식 screen 전환 후보:
  - prompt screen과 transcript screen 상태 분리
  - source messages 기반 virtual render
  - per-message expanded keys

## 비목표

- 이번 phase에서 full transcript overlay를 만들지 않는다.
- 이번 phase에서 terminal scrollback을 일반 `ctrl+o`로 clear+rewrite하지 않는다.
- 이번 phase에서 visual banner/scroll model을 단순화하지 않는다.
- 이번 phase에서 `alt+t` tool transcript overlay를 제거하지 않는다.

## 테스트 계획

### Unit/headless

1. `ctrl+o` live-only 대상 테스트
   - committed child + uncommitted child + liveTool child fixture 구성.
   - toggle 후 committed child에는 `setExpanded`가 호출되지 않아야 한다.
   - uncommitted/live child에는 호출되어야 한다.

2. commit canonical collapsed 테스트
   - `toolOutputExpanded=true` 또는 component expanded 상태에서 `commitFinalizedBacklog()` 실행.
   - fake `ui.commitLines(lines)`에 들어간 lines가 collapsed height/summary인지 검증.

3. thinking canonical collapsed 테스트
   - `thinkingExpanded=true` 상태의 assistant thinking message를 commit.
   - commit lines가 full thinking trace가 아니라 summary인지 검증.

4. keybinding 테스트
   - `ctrl+t`가 기본 `app.thinking.toggle`에 묶이지 않는지 검증.
   - hotkey markdown/help에 thinking toggle 기본키가 사라졌는지 검증.

### Manual / 실기기

1. 긴 세션에서 도구/thinking 출력 생성.
2. `ctrl+o`로 현재 턴 펼침.
3. 새 prompt 제출.
4. 이전 턴이 collapsed summary로 scrollback에 고정되는지 확인.
5. 다시 `ctrl+o`를 눌렀을 때 현재 live/mutable 구간만 변하고 과거 scrollback이 열린 상태로 남지 않는지 확인.
6. `ctrl+t`가 thinking toggle로 동작하지 않는지 확인.
7. `alt+t` tool transcript overlay가 기존대로 동작하는지 확인.

## 수용 기준

- 사용자가 `ctrl+o`로 펼친 상태에서 턴을 넘겨도 과거 scrollback에 full tool/thinking이 굳지 않는다.
- `ctrl+o`는 현재 미커밋/live 출력에 대해서만 열고 닫는다.
- `ctrl+t`는 더 이상 thinking toggle 기본키가 아니다.
- full transcript는 후속 phase로 명시 보류되어 있다.
- 083.11의 “닫을 수 없는 과거 픽셀” 문제가 UX상 재발하지 않는다.
