## 요약

`/quota` — OAuth 로그인된 프로바이더의 할당량(quota/usage limit)을 드롭다운 선택 후 TUI로 표시.
API key 상태에서는 비활성 (OAuth만 usage endpoint 지원).

---

## 기존 `/usage`와의 차이

| | `/usage` (기존) | `/quota` (신규) |
|---|---|---|
| 표시 방식 | 텍스트 덤프 (전 provider 일괄) | **드롭다운 선택 → 단일 provider TUI 렌더** |
| 대상 | 현재 세션의 active provider | OAuth 로그인된 모든 provider (inactive 포함) |
| 조건 | 제한 없음 | **OAuth credential만** (api_key 제외) |
| TUI | 채팅 영역에 텍스트 출력 | SelectList 드롭다운 → 전용 usage 패널 |
| ACP | 텍스트 반환 | 텍스트 반환 (드롭다운 없이) |

## 조건: OAuth 전용

```typescript
// auth-storage에서 OAuth credential만 필터
const oauthCredentials = authStorage.list()
  .filter(c => c.credential.type === "oauth" && !c.disabledCause);
```

API key 방식 (`type: "api_key"`)에서는 usage endpoint가 없으므로:
- `/quota` 실행 시 "Quota tracking requires OAuth login. API key providers don't expose usage limits." 메시지

## 참조 패턴: `/effort` dropdown

`/effort` bare 호출 시 `showEffortSelector()` → SelectList 표시 패턴을 재사용:

```typescript
// builtin-registry.ts
{
  name: "quota",
  description: "Show provider quota and usage limits",
  acpDescription: "Show quota for an OAuth provider",
  acpInputHint: "[provider]",
  subcommands: [
    { name: "anthropic", description: "Anthropic (Claude Pro/Max)" },
    { name: "openai-codex", description: "OpenAI (Codex/ChatGPT Plus)" },
    { name: "xai", description: "xAI (Grok)" },
    { name: "kiro", description: "Kiro" },
  ],
  allowArgs: true,
  handle: async (command, runtime) => {
    // ACP mode — text output
    const provider = command.args.trim() || null;
    const text = await buildQuotaText(runtime, provider);
    await runtime.output(text);
    return commandConsumed();
  },
  handleTui: (command, runtime) => {
    const provider = command.args.trim();
    if (provider) {
      runtime.ctx.showQuotaForProvider(provider);
    } else {
      runtime.ctx.showQuotaSelector();  // dropdown
    }
    runtime.ctx.editor.setText("");
  },
}
```

## Usage Fetch 구현

### Provider별 endpoint

| Provider | Endpoint | Auth | 응답 |
|---|---|---|---|
| `anthropic` | `api.anthropic.com/api/oauth/usage` | Bearer access_token | `{five_hour, seven_day, seven_day_opus, seven_day_sonnet}` |
| `openai-codex` | `chatgpt.com/backend-api/wham/usage` | Bearer access_token + chatgpt-account-id header | `{primary_window, secondary_window}` |
| `xai` | ❓ (미확인 — endpoint 발견 필요) | — | — |
| `kiro` | ❓ (미확인 — Kiro API에 usage endpoint 없을 수 있음) | — | — |

> 기존 `packages/ai/src/usage/claude.ts`, `openai-codex.ts`에 fetch 로직 이미 구현됨.
> `/quota`는 이 기존 `UsageProvider.fetchUsage()` 인터페이스를 호출하면 됨.

## 엣지 케이스

| 상황 | 처리 |
|---|---|
| OAuth credential 없음 | "No OAuth credentials. Use /login first." |
| OAuth 1개만 있음 | 드롭다운 스킵, 바로 해당 provider 표시 |
| 선택한 provider의 usage fetch 실패 | "Failed to fetch quota for {provider}: {error}" |
| provider가 usage endpoint 미지원 | "Quota tracking not available for {provider}." |
| 토큰 만료 (401) | auto-refresh 시도 → 실패 시 "Token expired. Run /login {provider} to re-authenticate." |
| `/quota unknown-provider` | "Unknown provider. Available: anthropic, openai-codex, xai, kiro" |

## Codex CLI 참조 (~/Developer/codex 리서치 결과)

### 핵심 발견

Codex CLI는 `codex-rs/backend-client/src/client.rs`에서 `/wham/usage` endpoint를 호출:

```rust
// PathStyle::ChatGptApi → "{base_url}/wham/usage"
// PathStyle::CodexApi → "{base_url}/api/codex/usage"
pub async fn get_rate_limits(&self) -> Result<RateLimitSnapshot>
pub async fn get_rate_limits_many(&self) -> Result<Vec<RateLimitSnapshot>>
```

### API 응답 구조 (OpenAPI 모델)

```rust
// RateLimitStatusPayload — GET /wham/usage 응답
struct RateLimitStatusPayload {
    plan_type: PlanType,           // guest|free|plus|pro|team|business|enterprise...
    rate_limit: Option<RateLimitStatusDetails>,
    credits: Option<CreditStatusDetails>,
    additional_rate_limits: Option<Vec<AdditionalRateLimitDetails>>,
    rate_limit_reached_type: Option<RateLimitReachedType>,
}

// RateLimitStatusDetails — 각 윈도우 스냅샷
struct RateLimitStatusDetails {
    allowed: bool,
    limit_reached: bool,
    primary_window: Option<RateLimitWindowSnapshot>,
    secondary_window: Option<RateLimitWindowSnapshot>,
}

// RateLimitWindowSnapshot — 개별 윈도우
struct RateLimitWindowSnapshot {
    used_percent: i32,
    limit_window_seconds: i32,
    reset_after_seconds: i32,
    reset_at: i32,
}
```

### jwc 기존 인프라와의 일치

jwc의 `packages/ai/src/usage/openai-codex.ts`가 **정확히** 동일한 구조를 이미 파싱:
- `CodexUsageWindowPayload` = Codex CLI의 `RateLimitWindowSnapshot`
- `CodexUsageRateLimitPayload` = `RateLimitStatusDetails`
- `CodexUsagePayload` = `RateLimitStatusPayload`

→ `/quota`는 새로운 파싱 로직 불필요. 기존 `fetchUsageReports()` 인프라를 provider 필터만 추가해서 재사용.

### Codex protocol의 `RateLimitSnapshot` (TUI 표시용)

```rust
struct RateLimitSnapshot {
    limit_id: Option<String>,      // "codex"
    limit_name: Option<String>,
    primary: Option<RateLimitWindow>,
    secondary: Option<RateLimitWindow>,
    credits: Option<CreditsSnapshot>,
    plan_type: Option<PlanType>,
    rate_limit_reached_type: Option<RateLimitReachedType>,
}

struct RateLimitWindow {
    used_percent: f64,       // 0-100
    window_minutes: Option<i64>,
    resets_at: Option<i64>,  // unix timestamp
}
```

## 참고

- `/usage` (기존)은 "현재 세션의 active provider 전체 보기" — 그대로 유지
- `/quota` (신규)는 "특정 OAuth provider 선택해서 상세 보기" — 보완 관계
- 향후: status line에 quota % 표시 가능 (083 밴드 확장)
