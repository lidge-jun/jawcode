## 1. 표시 사양

```
활성 (b 단계, 감사 통과):     I·P·A·▶B·C·D
a 단계 + 감사 대기:           I·P·▶A·B·C·D ⏳audit
a 단계 + 감사 FAIL(2R):       I·P·▶A·B·C·D ✗a2
complete / 상태 없음:         (세그먼트 invisible — 0폭)
```

- 지난 단계 = `dim`, 현재 단계 = `▶` + `accent` 굵게, 남은 단계 = `muted`
- 단계 문자 `I P A B C D` 대문자 + `·` 구분 (전부 1폭 ASCII/중점 — 한글 폭 이슈 회피, 99.04.01 §5)
- 게이트 chip 1개: `ctx.audit_status`/`verification_status`가 `pending`이면 `⏳audit`/`⏳verify`(warning), `fail`/`needs_fix`면 `✗a{a_round}`/`✗b`(error), 그 외 생략
- 전체 폭 ≤ 22 cols (status-line elastic 규칙 내)

```typescript
const PABCD_ORDER = ["i", "p", "a", "b", "c", "d"] as const;

const pabcdSegment: StatusLineSegment = {
	id: "pabcd",
	render(ctx) {
		const state = ctx.pabcd; // 99.04.01에서 SegmentContext에 추가
		if (!state?.active || !PABCD_ORDER.includes(state.stage as (typeof PABCD_ORDER)[number])) {
			return { content: "", visible: false };
		}
		const idx = PABCD_ORDER.indexOf(state.stage as (typeof PABCD_ORDER)[number]);
		const band = PABCD_ORDER.map((s, i) => {
			const ch = s.toUpperCase();
			if (i < idx) return theme.fg("dim", ch);
			if (i === idx) return theme.fg("accent", `▶${ch}`);
			return theme.fg("muted", ch);
		}).join(theme.fg("statusLineSep", "·"));
		const chip = pabcdGateChip(state); // §아래
		return { content: chip ? `${band} ${chip}` : band, visible: true };
	},
};

function pabcdGateChip(state: NonNullable<SegmentContext["pabcd"]>): string | null {
	if (state.stage === "a") {
		if (state.auditStatus === "fail") return theme.fg("error", `✗a${state.aRound ?? 1}`);
		if (state.auditStatus !== "pass") return theme.fg("warning", "⏳audit");
	}
	if (state.stage === "b" && state.verificationStatus !== "done") {
		return state.verificationStatus === "needs_fix" ? theme.fg("error", "✗b") : theme.fg("warning", "⏳verify");
	}
	return null;
}
```

## 4. 비고

- pabcd는 native 상태(D050-22)라 스킬 HUD 바(skill-active-state)에는 **올리지 않는다** — canonical 레일 무접촉 원칙 유지. 단계 띠는 세그먼트로만.
- cli-jaw 동형 포인트: cli-jaw는 단계를 프롬프트 헤더(`[PABCD — B: BUILD]`)로만 노출 — jwc는 이를 **상시 띠로 승격**하는 것이 차별점이자 080 §B의 의도.
