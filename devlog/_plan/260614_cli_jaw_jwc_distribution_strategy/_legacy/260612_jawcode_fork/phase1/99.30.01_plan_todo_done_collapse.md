2. **TUI 갭 (주원인)**: `todoExpanded === false`도 활성/마지막 페이즈 **최대 5줄** 노출; terminal-only일 때 **요약 1줄 없음**.

## 제안 슬라이스

### M1 — 프롬프트 · stop 리마인더 (저위험)

- `todo-write.md` 또는 신규 `prompts/system/todo-stop-hygiene.md` (before-agent-start / 리마인더 공용 문단):
  - stop/yield 전: 남은 `pending`/`in_progress` → `done` 또는 `drop`.
  - 전 페이즈 완료 시: `{"op":"done","phase":"<name>"}` 권장.
- `#checkTodoCompletion` 리마인더 본문에 **복사 가능한 `ops` 예시** 1~2개 (incomplete task content 그대로 인용).
- (선택) `todo.eager` 기본값은 유지; 대신 `system-prompt`에 3+ step 작업 시 `todo_write` 권장 한 줄 (99.03 표면 개정과 충돌 시 99.03에 편입).

**AC**: 단위/스냅샷 — 리마인더 문자열에 `todo_write` + `op":"done"` 포함; 기존 `agent-session` todo 테스트 green.

### M2 — TUI 전부 `completed` 시 자동 접힘 (**1차 구현 — 사용자 확정**)


```ts
// interactive-mode.ts (제안) — structure/todo_pipeline.md §TUI 와 동기화
function allTasksTerminal(phases: TodoPhase[]): boolean {
  const tasks = phases.flatMap(p => p.tasks);
  return tasks.length > 0 && tasks.every(t => t.status === "completed" || t.status === "abandoned");
}
```
- `interactive-mode.ts`:
  - 헬퍼 `allTasksTerminal(phases)` — 전 task가 `completed` | `abandoned`.
  - `allTasksTerminal && !todoExpanded` → hermes 스타일 한 줄:  
    `▸ Todos (completed/total)` (+ optional `· complete`).
  - `pending`/`in_progress` 존재 시: 기존 축약/확장 동작 유지; incomplete면 헤더에 pending 카운트 (hermes `incomplete` 힌트).
- `todo_write` 성공 후 `reloadTodos()` 시: `allTasksTerminal`이면 `todoExpanded = false` 강제 (사용자가 수동 펼친 경우는 토글로 복구 가능).

**AC**: 수동 — 3항목 todo 전부 `done` 후 composer 위 1줄만; 미완료 1개면 펼침/축약 기존과 동일. 회귀: `packages/coding-agent` 관련 tui/test.

### M3 — stop 시 “전부 completed인데 stop” (경계)

- `#checkTodoCompletion` 앞단: incomplete 0이면 리마인더 스킵 (현행 유지).
- (선택) `todo_auto_clear` / `rm` 정책은 **기본 비활성** — completed 생존은 devlog 5459 주석과 일치; UI만 접기.

### M4 — 설정 · 문서

- `settings-schema`: `todo.collapseWhenAllComplete` (boolean, default true) — M2 동작 끄기.
- devlog 본 문서 §검증에 스크린샷 경로 재첨부.
