TUI 전 표면에서 찾았다. 셀렉터 계열 13종 + 입력 표면 4종을 비교한 결과 **불일치 8건**이 확인됐다.
1위(ask v1/v2 분기)는 99.20.01에서 해소 완료 — 잔여 7건이 본 문서의 수리 대상이다.

## 2. 입력 표면 4종 격차 (main composer 대비)

| 기능 | main CustomEditor | v2 listSlot | legacy prompt Editor | HookInput |
|------|------------------|-------------|---------------------|-----------|
| 다중행 | ✅ | ✅ | ✅ | ❌ 단일행 |
| ctrl+g 외부 에디터 | ✅ | ✅ (`hook-selector.ts:777-779`) | ✅ (동일 경로) | ❌ |
| 슬래시 autocomplete | ✅ | ❌ | ❌ | ❌ |
| 플레이스홀더 | `"Type your message..."` | `""` | 없음 | 없음 |

- autocomplete 부재는 082.3 §9.3에서 "허용(삽입만)"으로 설계됐으나 미배선 — 수리 후보.
- HookInput(ui.input 훅 API)만 단일행+ctrl+g 부재 — 외부 확장 표면이라 영향 낮음.

## 3. fork-local vs upstream

v2 슬롯·numberOptionLabels·jaw-interview 계열은 **jwc 고유** (수리 자유). 셀렉터 본체들
(TreeSelector·ModelSelector·SelectList 등)은 upstream 공유 가능성 — 수리 시 diff 최소화 +
업스트림 PR 후보 분리 (99.11 연계). 단정엔 git blame 필요.
