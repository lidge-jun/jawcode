## 2. 상태 읽기 레일 — `status-line.ts`

### pabcd: fs.watch + 폴백

- git HEAD 워처 선례(`:259-278`) 미러: 생성자에서 `pabcdStatePath(cwd, sessionId)` 디렉터리(`.jwc/state/`)에 `fs.watch` — 파일 단위 watch는 atomic rename(tmp→rename)에 끊기므로 **디렉터리 watch + 파일명 필터**
- 콜백: `#pabcdState` 무효화 → 다음 렌더에서 `readPabcdState()`(lenient, fail-open) 재호출
- 워처 불가 환경(네트워크 fs 등) 폴백: 스킬 HUD와 동일 1초 TTL 폴링 레일에 합류(`:438-458`)
- 세션 스코프: TUI의 현재 sessionId가 있으면 session-scoped 경로 우선, 없으면 unscoped — `orchestrate-state.ts` `pabcdStatePath` 시그니처 그대로

### ultragoal: mtime+size 캐시 tail-scan

- `readUltragoalLedgerStats(cwd)`: `.jwc/ultragoal/ledger.jsonl` stat → (mtime,size) 미변경 시 캐시 반환; 변경 시 마지막 ≤200행만 파싱해 `goal_checkpointed` 카운트·마지막 evidence 공백 여부 산출
- 배치 위치: `gjc-runtime/` 신규 헬퍼가 아니라 **status-line 전용 경량 리더** (`modes/components/status-line/workflow-readers.ts` 신규, ≤120줄) — 렌더 경로에서 엔진 모듈 그래프를 끌어오지 않기 위함 (orchestrate-state import는 타입+경로 함수만)
