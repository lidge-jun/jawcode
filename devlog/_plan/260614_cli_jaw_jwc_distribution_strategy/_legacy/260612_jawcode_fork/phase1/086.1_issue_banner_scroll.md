> 하단(로고·세션 트레일·하단 보더)이 thinking 텍스트로 덮여 "찢어진" 상태. "스크롤이 완전 올라가야 되는데

짧은 터미널에서는 jaw 풀 배너(세션 트레일 포함 ~16행 + 주변 스페이서/에디터/상태줄 ≈ 22행+)가 뷰포트보다
높다. 첫 페인트 순간부터 배너 상단은 스크롤백으로 밀려나고, 이후 인트로 애니메이션·비동기 갱신·리사이즈가
얽히면 스크롤백 경계에서 배너가 시각적으로 찢긴 채 박제될 수 있다. 패치는 두 갈래: 원인 조사(에뮬레이터
재현)와, 원인과 무관하게 표면 자체를 없애는 수직 반응형 컴팩트 배너.

## 조사 — xterm 헤드리스 재현 7종 전부 무결

VS Code 터미널과 동일 엔진(`@xterm/headless`, 기존 `packages/tui/test/virtual-terminal.ts` 하네스)으로
60×12 단말에서 22행 배너 + 채팅 append를 재생하고 스크롤백 버퍼를 전수 검사:

| 시나리오 | 결과 |
|---|---|
| A. 첫 페인트(배너>뷰포트) + 채팅 append 6배치 | 22/22행 보존 ✓ |
| B. 배너가 스크롤백으로 나간 뒤 변이(인트로 프레임/모델 pill) | 22/22 ✓ (fullRender(true) 경로가 수복) |
| C. 배너가 늦게 성장(setRecentSessions 비동기 도착) | 22/22 ✓ |
| D. 본문 축소(clearOnShrink: 도구 접힘) 후 재성장 | 22/22 ✓ |
| E. 리사이즈 3종(너비/높이/둘 다) mid-session | 22/22 ✓ |

→ **렌더러의 기본 경로(차등/fullRender/clearOnShrink/리사이즈)는 스크롤백을 일관 유지**한다.
`tui.ts:1369` "firstChanged < viewportTop → fullRender" 가드와 fullRender의 절대 홈(`2J H 3J`) 재동기화가
드리프트를 자가치유하는 구조. 재현 스크립트: `/tmp/repro_banner_scroll.ts` (필요 시 tui 테스트로 승격).

미재현 = 실환경 변수 잔존 (§잔여 가설): VS Code 스크롤백 1000행 리밋/reflow, IME 하드웨어 커서 위치
시퀀스(`tui.ts:1519 #cursorControlSequence`)의 상대 이동 클램프 드리프트(한글 입력 상시 사용),
suspend/resume·외부 에디터의 ui.stop/start 재인쇄. 단정 불가 — 라이브 로그가 필요.

## 패치 — 수직 반응형 컴팩트 배너 (BRANDED 한정)

가로 반응형(우측 컬럼 숨김)은 이미 있었으나 **세로 반응형이 없었다**. 터미널 높이를 알면 풀 배너가
뷰포트+컴포저(예약 7행: 스페이서 2 + 에디터 3 + 상태줄 1 + 여유 1)에 안 들어갈 때 5행 컴팩트 배너로 접는다
→ 배너가 첫 페인트에서 스크롤백 경계에 걸칠 일 자체가 없어진다.

```
╭─── jwc v0.4.4 · Jawcode ───────────────╮
│ Jawcode · bite · build · ship · v0.4.4 │
│ [ ⬢ fable-5 ] [ 📦 anthropic ]         │
│ ▸ 최근세션 (2m) · /resume               │
╰────────────────────────────────────────╯
```

- `welcome.ts` — `getViewportRows?: () => number` 주입형 프로바이더(미주입 = 기존 동작 그대로) +
  `#renderCompact()`(타이틀 보더·아이덴티티 줄·pill 줄·최근 세션+`/resume` 줄) + 타이틀 보더 로직을
  `#titleBorderLine()`으로 공통화. 판정: `풀배너행수 + 7 > rows → 컴팩트`.
- `interactive-mode.ts` — 배너 생성 직후 `getViewportRows = () => this.ui.terminal.rows` 배선.
- **gjc diff-0**: 컴팩트 전환은 `BRANDED` 게이트 — gjc는 rows=12에서도 풀 배너 유지(테스트로 고정).
- 테스트: `brand-visual-identity.test.ts` +2건 — jwc rows=12 → ≤5행 + 아이덴티티 유지 / gjc rows 무시.
  기존 redesigned-shell·가드(`verify-gjc-ui-redesign`, `rebrand-inventory --strict`) 그린.

## 잔여 (라이브 디버그 레시피)

실환경에서 찢어짐이 재발하면: `PI_DEBUG_REDRAW=1 jwc` 로 재현 → 디버그 로그의 fullRender 사유
(`firstChanged < viewportTop` / `terminal width changed` / `clearOnShrink` …) 타임라인이 어느 경로에서
찢겼는지 알려준다. 그 로그가 와야 렌더러 측 단일 원인(특히 IME 커서 드리프트 가설)을 확정/기각할 수 있다.

## 컴팩트 임계 완화 (260613 00:5x — "최초 로딩이 밋밋해졌다")

사용자 패널(~21행)에서 풀 배너(15~16행)가 컴팩트로 떨어짐 — 임계 `풀배너+7 > rows`의 예약 7행은
**핀 이전 시대의 보수값**(첫 페인트에서 배너가 스크롤백 경계에 걸려 찢어지던 시절). 083.7 핀+§12
sticky gap으로 스크롤이 깨끗해진 지금은 컴포저 클러스터 실측 최소만 예약하면 충분.

- `welcome.ts COMPACT_BANNER_RESERVED_ROWS` **7 → 4** (에디터 3 + 상태줄 1) — 풀 배너가 ~20행
  패널까지 생존. 검증: rows=18 → 컴팩트(5행), rows=21 → 풀(16행). brand-visual 테스트 무회귀.
