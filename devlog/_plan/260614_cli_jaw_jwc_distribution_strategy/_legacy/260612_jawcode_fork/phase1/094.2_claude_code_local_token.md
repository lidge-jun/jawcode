## 토큰 위치 및 구조

### 저장소: OS Secure Storage (`SystemCredentialStore`)

Claude Code CLI는 `SystemCredentialStore` 추상화를 사용하여 OS별 secure storage에 credential을 보관.
바이너리 분석 결과 `security find-generic-password` (macOS), `secret-tool` (Linux) 호출 확인.

| OS | Backend | Service Name | Account |
|---|---|---|---|
| **macOS** | Keychain (`security` CLI) | `"Claude Code-credentials"` | `"{username}"` (e.g., `"jun"`) |
| **Linux** | libsecret/gnome-keyring (`secret-tool`) | `"Claude Code-credentials"` | `"{username}"` |
| **Windows** | Windows Credential Manager (DPAPI) | `"Claude Code-credentials"` | `"{username}"` |

### 값 (JSON) — 전 플랫폼 동일 포맷

```jsonc
{
  "claudeAiOauth": {
    "accessToken": "sk-ant-oat01-...",        // Anthropic OAuth access token
    "refreshToken": "sk-ant-ort01-...",       // Anthropic OAuth refresh token
    "expiresAt": 1781257465670,               // epoch milliseconds
    "scopes": [
      "user:file_upload",
      "user:inference",
      "user:mcp_servers",
      "user:profile",
      "user:sessions:claude_code"
    ],
    "subscriptionType": "max",                // "pro" | "max" | "team"
    "rateLimitTier": "default_claude_max_20x"
  }
}
```

## OS별 접근 방법

### macOS: `security` CLI

```bash
security find-generic-password -s "Claude Code-credentials" -w
# → JSON string (credential 값)
```

검증 완료: 실제 macOS Keychain에서 `claudeAiOauth` JSON 추출 성공.

### Linux: `secret-tool` (libsecret)

```bash
secret-tool lookup service "Claude Code-credentials"
# → JSON string (동일 포맷)
```

**전제조건**: `secret-tool` 설치 (`libsecret-tools` 패키지) + gnome-keyring 또는 KDE Wallet 실행 중.

| Distro | 패키지 | 비고 |
|---|---|---|
| Ubuntu/Debian | `libsecret-tools` (보통 기본 설치) | GNOME 세션에서 자동 unlock |
| Fedora/RHEL | `libsecret` | 동일 |
| Arch | `libsecret` | 동일 |
| Headless/SSH | 미지원 | gnome-keyring 데몬 없음 → `null` fallback |

**Headless 서버 대응**: `secret-tool`이 없거나 keyring 데몬이 없으면 `execSync`가 실패 → `null` 반환 → 기존 OAuth 플로우. 별도 처리 불필요.

### Windows: PowerShell `Get-StoredCredential` 또는 `cmdkey`

```powershell
# Option 1: cmdkey (Windows 내장)
cmdkey /list:Claude Code-credentials

# Option 2: CredentialManager 모듈 (PowerShell)
(Get-StoredCredential -Target "Claude Code-credentials").Password
```

**문제점**: Windows Credential Manager는 값을 바이너리(DPAPI 암호화)로 저장.
`cmdkey`로는 평문 읽기 불가 — `CredentialManager` PowerShell 모듈 또는 네이티브 API 필요.

| 방법 | 실현성 | 비고 |
|---|---|---|
| `cmdkey /list` | ❌ | 존재 확인만 가능, 값 추출 불가 |
| PowerShell `Get-StoredCredential` | ⚠️ | `CredentialManager` 모듈 설치 필요 |
| `dpapi-addon` (npm) | ❌ | 네이티브 의존성 추가 필요 |
| Direct Win32 API (FFI) | ⚠️ | Bun FFI로 가능하나 복잡 |

> **Windows는 P3**: DPAPI 복호화가 필요하여 간단한 CLI 호출로 불가.
> 대안: Windows 사용자는 기존 browser OAuth 플로우 사용.

## 구현

### 감지 함수 (cross-platform)

```typescript
import { execSync } from "node:child_process";
import { userInfo } from "node:os";

const SERVICE_NAME = "Claude Code-credentials";

function readFromSecureStorage(): string | null {
  const platform = process.platform;
  try {
    if (platform === "darwin") {
      return execSync(
        `security find-generic-password -s "${SERVICE_NAME}" -w`,
        { encoding: "utf8", timeout: 5000, stdio: ["pipe", "pipe", "pipe"] }
      ).trim();
    }
    if (platform === "linux") {
      return execSync(
        `secret-tool lookup service "${SERVICE_NAME}"`,
        { encoding: "utf8", timeout: 5000, stdio: ["pipe", "pipe", "pipe"] }
      ).trim();
    }
    // Windows: P3, not implemented
    return null;
  } catch {
    return null;
  }
}

export function detectClaudeCodeToken(): OAuthCredentials | null {
  const raw = readFromSecureStorage();
  if (!raw) return null;

  try {
    const data = JSON.parse(raw);
    const oauth = data?.claudeAiOauth;
    if (!oauth?.accessToken || !oauth?.refreshToken) return null;

    return {
      refresh: oauth.refreshToken,
      access: oauth.accessToken,
      expires: oauth.expiresAt ?? 0,
    };
  } catch {
    return null;
  }
}
```

### login 함수 수정 (`anthropic.ts`)

```typescript
export async function loginAnthropic(ctrl: OAuthController): Promise<OAuthCredentials> {
  // Auto-detect: Claude Code secure storage token
  const local = detectClaudeCodeToken();
  if (local) {
    ctrl.log?.("Found Claude Code token, importing automatically");
    if (local.expires < Date.now() + 60_000) {
      try { return await refreshAnthropicToken(local.refresh); }
      catch { /* refresh 실패 → 기존 플로우 */ }
    } else {
      return local;
    }
  }

  // 기존 browser OAuth 플로우
  const flow = new AnthropicOAuthFlow(ctrl);
  return flow.login();
}
```

### auth-storage.ts 수정


## 엣지 케이스

| 상황 | 처리 |
|---|---|
| macOS — Keychain 잠김 | `execSync` timeout 5s → `null` → 기존 플로우 |
| macOS — 첫 접근 (허용 팝업) | 사용자 "허용" 클릭 필요, "Always Allow" 가능 |
| Linux — `secret-tool` 미설치 | `execSync` 실패 → `null` → 기존 플로우 |
| Linux — headless (keyring 없음) | `secret-tool` 실패 → `null` → 기존 플로우 |
| Linux — Wayland + gnome-keyring | 정상 작동 (`secret-tool`은 D-Bus 경유) |
| Linux — KDE + KWallet | `secret-tool`이 KWallet과 호환 (kwallet 지원 시) |
| Windows | 미구현 → `null` → 기존 플로우 |
| Claude Code 미설치 / 미로그인 | entry 없음 → `null` |
| `claudeAiOauth` 필드 없음 | `null` |
| 토큰 만료 + refresh 성공 | 새 토큰 반환 (accountId/email 포함) |
| 토큰 만료 + refresh 실패 | `null` → 기존 플로우 |
| Claude Code Pro vs Max | subscriptionType 무관, inference scope만 필요 |

## UX 고려

### macOS
- `security` CLI 실행 시 Keychain 접근 **허용 팝업** 발생 가능
- 첫 실행에서 "jwc가 Keychain에 접근하려 합니다" → 사용자 "허용" 클릭
- "Always Allow" 선택 시 이후 묻지 않음
- 보안상 올바른 동작이나 사용자 경험이 약간 어색할 수 있음

### Linux
- `secret-tool`은 **팝업 없이** 조용히 작동 (세션 keyring이 이미 unlock 상태)
- SSH 세션이나 headless 서버에서는 keyring 없음 → 자동 fallback
- 로그 1줄: "Claude Code token not found (no keyring available)"

### Windows (P3)
- 미구현이므로 기존 브라우저 OAuth 플로우로 진행
- 추후 PowerShell `Get-StoredCredential` 방식 추가 검토

## 참고: CLI-jaw와의 차이

cli-jaw는 각 CLI 바이너리를 **직접 실행**하는 구조라 토큰 읽기가 불필요.
jwc는 **직접 API 호출** 구조이므로 토큰을 직접 획득해야 함.

```
cli-jaw: "claude야 이거 해줘" → claude 프로세스가 자체 auth 처리
jwc:     "Anthropic API에 직접 요청" → 토큰 필요 → Claude Code에서 빌려오기
```
