## 설계 (제안)

### A안 (권장·최소 위험): 3-state + "새 도구 시작 시 이전 도구 minimize"
1. `ToolExecutionComponent`에 `#minimized` 상태 + `setMinimized(bool)` 추가. 렌더에서 minimized면
   **헤더 1줄만** 출력(인자/결과 미리보기 생략). `tool-execution.ts:#updateDisplay`에 분기 추가.
2. `event-controller.ts:#handleToolExecutionStart`에서 새 컴포넌트 추가 직전, **직전 활성 도구 컴포넌트를
   `setMinimized(true)`**. "직전 활성"은 `pendingTools`/별도 `#lastToolComponent` 참조로 추적.
3. 전역 `toolOutputExpanded`(ctrl+o)가 true면 minimized 무시하고 전부 펼침 → "나중에 열기" 충족.
   ctrl+o off로 돌아오면 다시 최신만 preview, 나머지 minimized.
4. 정적 재렌더(`ui-helpers.ts`)도 동일 규칙 적용: 마지막 도구만 preview, 그 외 minimized (전역 펼침 아닐 때).
- 손볼 파일: `tool-execution.ts`(+ `bash-execution.ts`/`eval-execution.ts`/`read-tool-group.ts`가 자체
  setExpanded 가지면 minimized도 위임), `event-controller.ts`, `ui-helpers.ts`. 타입: `modes/types.ts` 컨텍스트.

### B안 (대안): 완료 시점 접힘
- `tool_execution_end`에서 해당 도구를 minimize. 단 "마지막 도구도 접힘"이 되어 활성 도구가 안 보일 수 있어,
  "가장 최근 완료 1개는 preview 유지" 규칙 필요 → A안이 더 단순.

### 패턴 B 추가 — 개별 재오픈(포커스 모델) [확정 요구]
A안의 minimize는 "접기"만 해결. **"모든 도구 개별 재오픈"** 요구를 위해 Codex `display_lines/transcript_lines`
이원화를 차용:
1. 접힌 도구 한 줄에 **인라인 힌트** `… +N lines` 표시(Codex `render.rs:254` 모방) — 발견성 확보.
2. **셀 포커스/선택 모델**: chatContainer에 "현재 선택된 ToolExecutionComponent" 인덱스 + 상하 이동 키 +
   선택 도구만 `setExpanded(toggle)`. → 원하는 도구 하나만 제자리 펼침. (jwc `setExpanded` 그대로 재사용.)
3. 펼침 시 높이 변동 → 부모 재레이아웃 트리거 필요(차등 렌더러 reflow).

### 패턴 A 추가 (2차) — Codex식 Ctrl+T 전체 transcript 오버레이
모든 도구(완료분 포함)의 **전체 출력**을 풀스크린 pager로. 요구 "전부 다시 보기"를 한 화면에서 충족.
- 전제: `event-controller.ts:506/519/546/561`의 `pendingTools.delete`가 완료 컴포넌트를 폐기 → **완료 후 보관
  리스트**로 라이프사이클 변경 필요. 신규 오버레이 레이어 + Ctrl+T 키 + pager 스크롤 상태. 난이도 上 → 후속.

### 열린 질문 (구현 전)
1. minimized 한 줄 포맷: 도구명 + 인자 요약 어디까지 (bash=명령, read=경로, edit=파일)?
2. 에러 도구는 minimized라도 에러 표식 유지 ([기본값] ✘ + 메시지 첫 줄).
3. 포커스 이동 키 선택(↑↓ vs j/k) + 펼침 키(Enter vs Space) — 기존 키바인딩 충돌 점검.
4. 스냅샷 테스트 영향 (tui 896개 중 tool-execution 렌더) — minimized/truncate 기본화 시 갱신 범위.
5. 단계: **B의 인라인 minimize+포커스 펼침을 1차**(가성비·요구 직접 충족), A 오버레이는 2차.

## 적용된 수정 (1차 = A안)

- `tool-execution.ts`: `#minimized` + `setMinimized()` + `render()` 오버라이드 — minimized이고 `#expanded`
  아닐 때 **한 줄 요약**(아이콘 + 라벨 + `formatArgsInline` 인자 요약 + Codex식 `… +N lines` 힌트)만 출력.
  에러 도구는 ✘ + 에러 첫 줄 유지. `ToolExecutionHandle`에 `setMinimized?` 옵션 추가.
- `event-controller.ts` (스트리밍 :353 / 실행시작 :472 두 생성 지점) + `ui-helpers.ts`(리플레이): 새
  ToolExecutionComponent 추가 직전 `ctx.lastToolComponent?.setMinimized(true)` → **마지막 도구만 preview**.
- **후속 수정 (사용자 실테스트 "bash가 안 접힘")**: "다음 도구 시작" 트리거만으로는 **턴의 마지막 도구가
  영영 안 접힘** → 트리거 2개 추가: ① 새 assistant 메시지 시작(`#handleMessageStart`) = 도구 배치 종료
  (최종 텍스트 스트리밍 시작 시점에 접힘), ② `#handleAgentEnd` 안전망(abort/error로 끝나는 턴).
  리플레이(`ui-helpers`)도 assistant 메시지 경계에서 동일 적용. 단, 리플레이가 도구로 끝나는 세션(진행 중
  턴 재개)은 마지막 도구를 열어 둠 — 의도적.
- `types.ts`/`interactive-mode.ts`: `lastToolComponent` 컨텍스트 필드, `rebuildChatFromMessages()`에서 리셋.
- ctrl+o(전역 `toolOutputExpanded`)는 렌더 시 minimized보다 우선 → 전체 펼침/복귀 그대로 동작 ("나중에 열기" 1차 충족).
- read 그룹(ReadToolGroupComponent)은 1차 제외 — 자체가 한 줄/파일 요약 형태라 minimize 가치 낮음.
