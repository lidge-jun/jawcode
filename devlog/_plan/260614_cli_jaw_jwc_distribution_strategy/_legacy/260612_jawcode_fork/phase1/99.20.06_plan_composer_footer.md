### 메인 REPL 푸터 합성 (`e0z`, cli.pretty.js:576820–577070)

- 컨테이너: `Box { height: 1, overflow: "hidden" }` + `Text { wrap: "truncate" }` — **정확히 1행**.
- 세그먼트 배열을 우선순위로 합성, `" · "` 구분: ① remote 표시 ② **권한 모드**
  (`"bypass permissions on"` + 폭 여유 시에만 `"(shift+tab to cycle)"` — 폭 티어 `l<2`/`j>=80`)
  ③ teams ④ PR 상태 ⑤ 스트리밍 중 힌트(`esc to interrupt`·`stop agents`) ⑥ **전부 없으면
  `"? for shortcuts"` 폴백** ⑦ voice/tasks 힌트.
- **입력 모드 표시**: `mode === "bash"`면 푸터 전체가 `"! for bash mode"`(bashBorder 색)로 교체
  (cli.pretty.js:576893) — 푸터가 입력 모드 인디케이터를 겸한다.
- **빈 푸터는 `return null`** — CC는 alt-screen Yoga 앱이라 행 추가/제거 reflow가 공짜.
  jwc는 스크롤백-네이티브 차등 렌더러이므로 이 부분만은 못 따라가고 **상시 1행 예약**(§1.5)이
  올바른 번역이다. 나머지(교대 문법·우선순위 합성·truncate·` · ` 구분·폭 반응)는 그대로 채택.

### 알림 큐 시스템 (`Zq`, cli.pretty.js:471133–471315) — 2차 심층 실측 (260613)

CC의 일시 메시지는 푸터와 **별도의 1행 알림 라인**(`pWz`, cli.pretty.js:573490–573505,
입력창 아래 정보 스택의 첫 줄)에 그려지며, 전역 상태 `notifications: { current, queue }`로
관리된다:

- **스키마**: `{ key, text | jsx, color?, priority, timeoutMs?, invalidates?: key[], fold? }`.
- **우선순위**: `immediate(0) > high(1) > medium(2) > low(3)` (cli.pretty.js:471322).
  `immediate`는 현재 알림을 선점(현재 것은 큐 선두로 복귀), 나머지는 큐에서 우선순위순 드레인.
- **타임아웃**: 기본 `Gr1 = 8000ms`, 항목별 `timeoutMs` 오버라이드. 만료 시 current=null →
  다음 항목 드레인. `key` 중복은 무시(또는 `fold`로 병합), `invalidates`로 타 키 제거.
- **표시**: `Text { wrap: "truncate", color: current.color, dimColor: !color }` — 1행 truncate.
- **실측 인벤토리** (발신 예): `escape-again-to-clear` "Esc again to clear" (immediate, 1000ms) ·
  `external-editor-hint` "ctrl+g to edit in $EDITOR" (immediate, 5000ms) ·
  `error-compacting-conversation` (immediate, error색) · `auto-mode-gate-notification` (high, warning색) ·
  `kill-agents-confirm` (푸터 e0z가 current.key를 읽어 합성 변경 — 두 표면의 연동 전례).

### 에디터 더블프레스 실제 와이어링 (cli.pretty.js:471416–471450)

- **ctrl+c**: `pI` 1차 부수효과 = 텍스트 있으면 클리어, pending은 콜백으로 **부모에 상승** →
  부모 푸터가 "Press Ctrl-C again to exit"로 교대 (알림 큐 경유 아님).
- **esc esc**: 메인 REPL에서는 **종료가 아니라 클리어** — 1차에 알림
  `"Esc again to clear"`(immediate, 1000ms), 2차에 히스토리 저장 + 클리어.
  ※ jwc의 esc esc는 082.1 IME 안전망(종료)이므로 의미가 다름 — jwc 의미 유지 (§2.2).
- **ctrl+d**: **텍스트가 빈 경우에만** pending/종료 동작 (텍스트 있으면 del 문자 처리).
  → jwc W2 구현 시 동일 가드 채택: 빈 에디터에서만 더블프레스 종료, 텍스트 있으면 무시.

### jwc 번역 판정 — 두 표면을 한 행으로 접기

CC는 [푸터 힌트 라인(e0z)] + [알림 라인(pWz)] **2행 체계**다. jwc v1은 행 예산 1행이므로
둘을 우선순위 스택(transient ▸ mode ▸ hint)으로 **한 행에 접는다** — transient 슬롯이 CC
알림 라인의 축소판(단일 current + 타임아웃), mode/hint 슬롯이 CC 푸터의 축소판. CC급
우선순위 큐(4단계·fold·invalidates)는 v1 비범위 — 필요해지면 후속 슬라이스로 승격.

## 2. 설계 — `ComposerFooter` 컴포넌트 + 우선순위 스택

### 2.1 NEW `packages/coding-agent/src/modes/components/composer-footer.ts`

```ts
export type FooterSlot = "transient" | "mode" | "hint";

export class ComposerFooter implements Component {
    #transient: { text: string; expiresAt?: number } | null = null; // ctrl+c 안내 등
    #mode: string | null = null;       // plan/goal/bypass 등 지속 상태
    #hint: string | null = null;       // 평시 기본 힌트 ("/help for help")
    // render(width): 우선순위 transient > mode > hint > "" — 항상 정확히 1줄 반환
}
```

- **항상 1줄 반환** (§1.5 행 불변식 — CC와 유일하게 갈리는 결정, §1.6). `truncateToWidth` +
  `sanitizeStatusText` 재사용.
- transient는 만료 타이머 내장 (한글 IME 힌트의 setTimeout 패턴 일반화 + CC `pI`의 800ms
  자동 해제와 동형). 타이머 `unref()` 필수 (기존 `#hangulImeHintTimer` 동일).
- 스타일: dim 전경, 들여쓰기 1칸, 다중 세그먼트는 **`" · "` 구분 (CC 실측 규격)** — 99.20.02
  ⑤의 단일 힌트 규격을 여기서 확정하고, 셀렉터 푸터들은 후속 슬라이스에서 이 규격으로 수렴.
- 키 이름은 하드코딩 금지 — `keybinding-hints.ts`의 `appKey()/editorKey()`로 실바인딩 표기
  (리바인드 시 자동 추종). CC도 `keyName`을 상태로 들고 다니며 같은 원칙.
- 폭 반응 (CC 폭 티어 채택): 좁은 단말에서는 보조 힌트(`(shift+tab …)` 류)부터 탈락 —
  1차는 truncate만, 티어 탈락은 후속.

### 2.4 확정 구현 범위 (260613 CC 2차 실측 후 동결)

**컴포넌트 API** (CC `Zq`/`e0z`의 1행 축소판):

```ts
export class ComposerFooter implements Component {
    /** 일시 메시지 — 최신 우선(latest-wins), durationMs 후 자동 소거. CC 알림 라인 축소판. */
    setTransient(text: string, opts?: { durationMs?: number; color?: "warning" | "danger" | "dim" }): void;
    clearTransient(): void;
    /** 지속 모드 표시 — F4 보류로 v1에서는 호출부 없음, 슬롯만 존재. */
    setMode(text: string | undefined): void;
    /** 평시 폴백 힌트 — 기동 시 1회 설정. */
    setHint(text: string | undefined): void;
```

## 6. 구현 기록 (260613)

§2.4 동결 범위 그대로 랜딩. 구현 중 추가된 설계 결정 3건:

1. **`onForcedExit` 분리 (핵심)** — 더블-esc 안전망(082.1)은 `onExit`을 호출했는데, `onExit`
   (=ctrl+d 핸들러)이 더블프레스로 바뀌면 esc esc가 "즉시 종료" 대신 "ctrl+d 안내 arm"이 되어
   안전망이 깨진다. `custom-editor.ts`에 `onForcedExit`(즉시 종료 전용, 미설정 시 `onExit`
   폴백) 신설 — 안전망은 `onForcedExit ?? onExit`을 타고, 체인은 `onExit`만 더블프레스화.
2. **푸터 off = 레거시 거동 보존** — ctrl+d는 푸터 비활성(gjc 브랜드/설정 off) 시 기존

   hook-status 표면 폴백. ctrl+c/esc의 800ms 윈도우 정렬은 전역 (W1, 거동 무해).
3. **esc 1차 안내의 조용한 맥락 필터** — `onExitPending`은 arm 시 항상 발화하되, 컨트롤러가
