### 왜 모델이 Cursor 네이티브 도구명(Shell/Glob/Grep)을 말했나

cursor 프로바이더는 요청 바디에 tools를 넣지 않고, 백엔드의 `requestContextArgs` 요청에 응답하며
도구를 advertise한다 (`cursor.ts:976-997`). 이때 네이티브 도구는 의도적으로 제외된다
(`cursor.ts:2069` `CURSOR_NATIVE_TOOL_NAMES`, `:2076` filter — Cursor 백엔드가 server-driven exec로
직접 수행하므로 MCP 도구만 advertise). `Shell`/`Glob`/`Grep`/`mcp_<server>_<tool>` 네이밍은
Cursor IDE 학습 데이터에 각인된 자사 도구명으로, 모델이 텍스트로 재현한 환각이다.

   보내는지, `log("exec","dispatch",…)`(`cursor.ts:975`)가 찍히는지 확인. 정상이면 본 이슈는 100% 타이틀 환각 단독.
2. cursor 디버그 로그(`appendCursorDebugLog`, `cursor.ts:154`) 활성화 후 advertise 도구 목록 캡처.
3. **081 조사와의 교차점**: 081은 "와이어에 네이티브 toolCallStarted가 실제 도착했으나 파서가 드롭"
   가설(95%)이고, 본 조사는 "에이전트 턴에 toolCall emit 없음"(90%)이다. 둘 다 "세션에 toolCall 블록

   맞아도 둘 다 유효(타이틀 가드 + 파서 폴백 모두 방어적으로 타당).
