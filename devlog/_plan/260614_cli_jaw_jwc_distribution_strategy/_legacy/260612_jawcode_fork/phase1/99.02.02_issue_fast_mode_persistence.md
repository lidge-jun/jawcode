# 99.02.02 — issue: /fast 토글이 재시작 후 보존되지 않음

> 상태: ✅ 수정 완료 (260613 00:4x). 소속: [99.02 정리](./99.02.00_plan_parallel_landing.md) 패키지 (소형 마감 이슈군).
> 입력: 사용자 "fast 모드가 토글되었을 때 종료하고 다시 실행했을 때 상태가 보존되지 않아" (260613).

`/fast`는 `setServiceTier`로 agent 상태를 바꾸고 **세션 jsonl에만** 기록했다(`appendServiceTierChange`).
같은 세션 resume은 복원되지만, 새 실행(새 세션)은 tier 엔트리가 없어 `serviceTier` **설정**으로
폴백하는데(`sdk.ts:1792`, `agent-session.ts:9095` — 인프라는 이미 존재), `/fast`가 그 설정을
쓰지 않아서 항상 "none"으로 시작했다.

## 수정

`agent-session.ts setFastMode`에 영속 1줄: `settings.set("serviceTier", enabled ? "priority" : "none")`.

- **`setServiceTier`가 아니라 `setFastMode`에 넣은 이유 (중요)**: anthropic fast-mode 자동 폴백
  (`agent-session.ts:1952-1954`)이 미지원 모델에서 `setServiceTier(undefined)`를 호출한다 —
  거기에 영속을 걸면 자동 폴백이 사용자 선호를 조용히 지운다. 영속은 **사용자 의도 경로**
  (`/fast on|off|toggle` → setFastMode/toggleFastMode)에만.
- scoped tier(`openai-only` 등)가 이미 켜진 상태의 `/fast on`은 기존대로 early-return(스코프 보존,
  설정 무변경). `/fast off`는 전역 off로 영속 — 대칭적 의미론.
- 선례: `/think` 레벨 영속(`defaultThinkingLevel`, :5999)과 동일 패턴.

## 검증

- 신규 세션 생성 경로가 설정을 읽는 것은 기존 코드로 보장 (`hasServiceTierEntry`가 false면
  `configuredServiceTier` 사용). tsc 0, acp-builtins/acp-agent 92 pass.
- 수동 e2e: `/fast on` → 종료 → 재실행 → 상태줄 fast 표시 유지 + `/fast status` = on (사용자 확인 대기).
