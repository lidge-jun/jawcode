## 1. 기본 on 전환 기준

**무엇**: `JWC_COMMIT_LANE` 옵트인 환경변수 게이트를 제거하고 `commitLaneEnabled()`가 능력 감지만으로 작동하게 한다.

**왜**: 옵트인은 초기 검증용이고, "standard" 레인 감지 터미널에서는 폴백이 구조적으로 보장되어 있다
(`insert-history.ts:21` `HistoryLaneMode` 분기, `tui.ts:1169` `#historyLane !== "standard"` 조건).
폴백 경로가 견고하면 기본 활성화가 안전하다.

**앵커**:
- `packages/coding-agent/src/modes/utils/ui-helpers.ts:47-48` — `commitLaneEnabled()` 플래그 게이트
- `packages/tui/src/insert-history.ts:30-34` — `detectHistoryLaneMode()` env 분기
- `packages/tui/src/tui.ts:1169` — `#historyLane !== "standard"` 조기 탈출

**검증 시나리오 매트릭스**:

| 환경 | 확인 항목 | 합격 기준 |
|------|-----------|-----------|
| iTerm2 | 스크롤백에 커밋 픽셀 실착 여부 | 도구 폴딩 후 스크롤업 → 텍스트 보임 |
| kitty | fullRedrawCount / PI_DEBUG_REDRAW 0 어서션 | `tui.ts:1408-1413` 로그 0건 |
| tmux (네이티브) | 커밋 라인이 tmux 자체 `tmux capture-pane` 히스토리에 적재 | capture 후 grep 확인 |
| Ghostty | pre-wrap 렌더 깨짐 없음 | 80열 → 120열 리사이즈 후 커밋 텍스트 가독성 |
| conpty (Windows) | `HistoryLaneMode = "unsupported"` 자동 폴백 → virtual 레인 동작 | fullRedraw 정상, 스크롤백 손상 없음 |

**텔레메트리**: `PI_DEBUG_REDRAW=1 jwc` → `~/.jwc/agent/state/gjc-debug.log`(`utils/src/dirs.ts:449`). 긴 세션(≥20턴, thinking 포함)에서 `fullRender` 기록 0건 + 3J 없음. `commitOrAppend()`(`ui-helpers.ts:57-63`) false 반환 폴백률 < 5% 이면 전환 허용.

**완료 기준**: `commitLaneEnabled()`가 플래그 없이 `detectHistoryLaneMode() === "standard"`만으로 true를 반환하고, 매트릭스 전 항목 합격.

## 3. 리사이즈 UX — pre-wrap 고정 정책

**무엇**: 커밋된 라인은 write 시점 폭으로 pre-wrap 고정되며 리사이즈 시 리플로우하지 않는다(083.9 §3-3, Codex `PreWrap` 동일). width 변경 시 `widthChanged` 분기(`tui.ts:1305, 1424-1427`)는 라이브 존만 fullRender하고, `#committedScreenRows`를 포함한 스크롤백 픽셀은 건드리지 않는다.

**실제 UX**: 폭을 줄이면 커밋 텍스트가 잘릴 수 있다. 폭을 늘리면 커밋 텍스트가 여백을 낭비하는 채 남는다. 이는 알려진 트레이드오프이며 Codex와 동일 수용.

**검증 포인트**:
1. `width` 변경 시 `#committedScreenRows` 처리: `tui.ts:1314-1316` — `clear && committedScreenRows > 0`이면 `#scrollOutCommittedRows`로 선행 플러시 후 `committedScreenRows = 0`. width 변경 fullRender 경로에서 이 선행 플러시가 실행되는지 확인.
2. 플러시 후 스크롤백에 커밋 픽셀이 old-width 형태로 남고 새 라이브 존이 new-width로 그려지는 시각적 불일치는 수용.

**완료 기준**: 80→120열 리사이즈 후 스크롤백 손상 없음(커밋 픽셀 보존), 라이브 존 정상 리렌더.

## 4. zellij raw 모드 구현

**무엇**: `detectHistoryLaneMode()`가 Zellij 환경에서 `"zellij-raw"`를 반환하지만, `tui.ts:1169`의 `commitLines()`는 `#historyLane !== "standard"` 조건으로 false를 반환해 **현재 Zellij는 항상 virtual 레인 폴백**이다.

**왜**: Zellij는 부분 스크롤 리전에서 soft-wrap을 보장하지 않아 표준 DECSTBM 경로가 깨진다(083.9 §0b-5). Codex는 "raw-append" 특수 모드를 구현한다 — 리전 없이 커서를 히스토리 바닥에 배치하고 `\r\n`으로 직접 append.

**구현 방침**: `buildInsertHistorySequence`에 `mode: "zellij-raw"` 분기 추가 — Phase 2(DECSTBM 리전) 대신 `\x1b[{liveZoneTop};1H` + `\r\n`×N으로 raw append. `commitLines()`에서 `#historyLane === "zellij-raw"` 분기 허용.

**앵커**:
- `packages/tui/src/insert-history.ts:21` — `HistoryLaneMode` 타입 (`"zellij-raw"` 이미 정의)
- `packages/tui/src/insert-history.ts:32` — Zellij 감지 분기 (이미 "zellij-raw" 반환)
- `packages/tui/src/tui.ts:1169` — `!== "standard"` 가드 → "zellij-raw" 허용으로 확장할 지점

**완료 기준**: Zellij 터미널에서 `JWC_COMMIT_LANE=1` 세션의 fullRedrawCount = 0, 스크롤백 텍스트 가독성 확인.

## 5. 멀티플렉서 실증 (tmux)

**무엇**: tmux는 표준 경로(standard)로 처리되고 커밋 라인이 tmux 자체 pane 히스토리에 적재된다는 것이 의도된 동작이다(`insert-history.ts:27-28` 주석).

**실증 절차**: `JWC_COMMIT_LANE=1 jwc`를 tmux에서 실행 → 도구 완료 후 `tmux capture-pane -p -S -200 | grep <committed_text>` 로 커밋 텍스트 확인. `isMultiplexerSession()`(`tui.ts:136`) + `#hasCommittedHistory`(`tui.ts:1325`) 플래그 상호작용으로 3J 제외.

**앵커**:
- `packages/tui/src/tui.ts:136` — `isMultiplexerSession()` (TMUX/STY/ZELLIJ env)
- `packages/tui/src/tui.ts:1321-1325` — `#hasCommittedHistory` → 3J 제외 분기

**완료 기준**: tmux capture-pane에서 커밋 텍스트 grep 성공 + fullRender 3J 로그 0건.

## 6. `/resume` replay 전량 커밋 방출

**무엇**: `--resume` 세션에서 이전 히스토리를 재생할 때 모든 과거 셀을 커밋 형태로 방출해 replay가 라이브 존을 overflow하지 않도록 한다.

**현황**: replay 경로(`interactive-mode.ts:913` `loadSessionContext` → `addMessageToChat` with `populateHistory`)는 `chatContainer.addChild`로 셀을 쌓는다. 커밋 레인이 기본 on이면 replay 도중 lagging fill로 `commitLines()` false 반환이 빈번해진다.

**방침**: `addMessageToChat`의 replay 경로(옵션 `populateHistory: true`)에서 커밋 레인 활성 시 각 셀을 완성 상태로 즉시 `commitOrAppend` — virtual 레인 경로보다 렌더 비용이 낮고 스크롤백에 전량 착지하면 세션 시작 시 fullRender가 라이브 존만 그린다.

**앵커**:
- `packages/coding-agent/src/modes/interactive-mode.ts:2212-2213` — `addMessageToChat` → `#uiHelpers.addMessageToChat`
- `packages/coding-agent/src/modes/utils/ui-helpers.ts:109` — `addMessageToChat` 케이스 switch
- `packages/coding-agent/src/sdk.ts:913` — `loadSessionContext` (replay 진입점)

**완료 기준**: `--resume`으로 30턴 이상 세션 재진입 시 프레임 길이 ≤ viewport+ε, fullRedrawCount = 0.
