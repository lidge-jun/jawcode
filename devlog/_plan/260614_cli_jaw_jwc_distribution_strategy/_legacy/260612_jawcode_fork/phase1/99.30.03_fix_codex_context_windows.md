# 99.30.03 — fix: codex 컨텍스트 윈도우 카탈로그 정정 (gpt-5.5 272K / gpt-5.4 1M)

> 상태: 조사 ✅ (260613, 백엔드 라이브 조회 2회로 확정) / **구현 ✅ (260613)** — ai 스위트
> 1271 pass·신규 테스트 2·tsc 0. 잔여: 재시작 후 HUD 확인 + 5.4 캐시 갱신 여부 (§5).
> 소속: [99.30.00 MOC](./99.30.00_moc_feature_improvement.md). 입력: 사용자 260613 "HUD가
> `◫ 70.1%/400K`인데 272K 제한인가봐, 공식문서 보고 기본 설정 패치. 5.4는 1M이 확실."

HUD가 gpt-5.5 세션을 `70.1%/400K`(≈280K 토큰)로 표시하던 시점에 이미 요청이 실패하고
있었다. 원인은 jwc가 gpt-5.5를 400K 모델로 믿고 있었던 것 — **codex 백엔드의 실제 한도는
272K**이고, 70.1%는 그 한도를 넘긴 뒤였다. 컴팩션 임계값도 400K 기준이라 작동하기 전에
세션이 죽는다. 반대로 gpt-5.4는 백엔드가 **1M 용량**을 광고하는데 jwc는 272K로만 알고
있었다. 사용자의 진단("5.5는 272K, 5.4는 1M — codex 한도 기준")이 정확했다.

## 1. 확정 근거 — 백엔드 라이브 조회 (260613, HTTP 200 × 2)

`GET chatgpt.com/backend-api/codex/models?client_version=0.139.0` (저장된 codex OAuth 토큰):

| slug | context_window | max_context_window |
|------|---------------|--------------------|
| gpt-5.5 | 272000 | **272000** (하드캡) |
| gpt-5.4 | 272000 | **1000000** (1M 용량) |
| gpt-5.4-mini | 272000 | 272000 |
| gpt-5.3-codex-spark | 128000 | 128000 |

- 백엔드는 기본 윈도우(`context_window`)와 세션 용량(`max_context_window`)을 분리 보고.
- client_version에 따라 모델 목록·필드가 달라짐 (구버전 0.99.0은 5.5 미노출) — 디스커버리
  버전 매개변수의 중요성 재확인.
- API(api.openai.com) 쪽은 별개: 5.4·5.5 모두 1,050,000 — jwc `openai` provider 번들은
  이미 정확해서 무변경.

> 출처: [openai/codex#19319 — GPT-5.5 reports 258400 despite published 400K](https://github.com/openai/codex/issues/19319) · [openai/codex#20761 — needs opt-in 1M context](https://github.com/openai/codex/issues/20761) · [GPT-5.5 Model | OpenAI API](https://developers.openai.com/api/docs/models/gpt-5.5) · [GPT-5.4 Model | OpenAI API](https://developers.openai.com/api/docs/models/gpt-5.4)

## 2. 결함 3건

1. **`model-thinking.ts:433` `applyGpt55ContextWindow`** — "gpt-5.5는 400K 모델" 전제로
   디스커버리가 보고한 272000을 모든 provider에서 **400000으로 강제 덮어씀**. HUD 과대
   표시·컴팩션 미작동·~272K에서 4xx의 직접 원인. (API쪽 1.05M도 400K로 깎고 있었음.)
2. **`models.json` openai-codex 번들** — gpt-5.5: 400000(과대), gpt-5.4: 272000(과소).
3. **`utils/discovery/codex.ts:18` 스키마** — `max_context_window` 필드를 파싱하지 않아
   5.4의 1M 용량을 인지할 경로 자체가 없음.

## 3. 패치 내역

| 파일 | 변경 |
|------|------|
| `model-thinking.ts` | 400K 핀 → **codex 전송로(`openai-codex-responses`) 한정 272K 핀** (낡은 캐시/번들의 400K 방어). 타 provider 카탈로그 값 보존. `linkOpenAIPromotionTargets` 주석의 400K 서술 정정 (5.5→5.4 자동 승격은 모델 계열이 달라 비도입 — 사용자 결정 영역) |
| `utils/discovery/codex.ts` | `max_context_window` 파싱 추가, `max(context_window, max_context_window)` 채택 → 5.4가 디스커버리에서 자동으로 1M |
| `models.json` | openai-codex: gpt-5.5 400000→272000 · gpt-5.4 272000→**1000000** |
| 테스트 | 구 400K 기대 2곳 갱신 (`model-thinking.test.ts` · `openai-codex-default.test.ts`) + 신규 `codex-discovery-context.test.ts` 2케이스 (1M 채택 / 272K 기본값 폴백) |
