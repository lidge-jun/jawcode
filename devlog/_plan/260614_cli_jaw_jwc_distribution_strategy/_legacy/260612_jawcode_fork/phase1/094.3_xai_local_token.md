## 토큰 위치 및 구조

### 경로 (전 플랫폼 동일)

| OS | 경로 | 비고 |
|---|---|---|
| macOS | `~/.grok/auth.json` | 검증 완료 |
| Linux | `~/.grok/auth.json` | 동일 (파일 기반, keychain 불필요) |
| Windows | `%USERPROFILE%\.grok\auth.json` | 동일 |

## 호환성 검증

| 항목 | Grok CLI | jwc `xai` provider | 결과 |
|---|---|---|---|
| OIDC Client ID | `b1a00492-073a-47ea-816f-4c329264a828` | `XAI_OAUTH_CLIENT_ID` (동일) | ✅ |
| OIDC Issuer | `https://auth.x.ai` | `XAI_OAUTH_ISSUER` (동일) | ✅ |
| Token endpoint | Discovery → `auth.x.ai/oauth/token` | 동일 (OIDC discovery) | ✅ |
| Scope | `openid profile email offline_access grok-cli:access api:access` | 동일 `XAI_OAUTH_SCOPE` | ✅ |
| Access token | JWT (typ: `at+jwt`, alg: ES256) | JWT | ✅ |
| Refresh token | opaque string | 동일 | ✅ |

**결론**: 완전 동일 OAuth app + 동일 scope. 변환 없이 그대로 사용.

## 구현

### 감지 함수

```typescript
import { existsSync, readFileSync } from "node:fs";
import { join } from "node:path";
import { homedir } from "node:os";

const XAI_AUTH_KEY_PREFIX = "https://auth.x.ai::";

export function detectGrokCliToken(): OAuthCredentials | null {
  const authPath = join(homedir(), ".grok", "auth.json");
  if (!existsSync(authPath)) return null;

  try {
    const raw = JSON.parse(readFileSync(authPath, "utf8"));

    // Find the entry matching xAI OIDC issuer
    const entry = Object.entries(raw).find(
      ([key]) => key.startsWith(XAI_AUTH_KEY_PREFIX)
    )?.[1] as Record<string, unknown> | undefined;

    if (!entry?.key || !entry?.refresh_token) return null;

    const accessToken = entry.key as string;
    const refreshToken = entry.refresh_token as string;
    const expiresAt = entry.expires_at
      ? new Date(entry.expires_at as string).getTime()
      : 0;

    return {
      refresh: refreshToken,
      access: accessToken,
      expires: expiresAt,
      accountId: entry.user_id as string | undefined,
      email: entry.email as string | undefined,
    };
  } catch {
    return null;
  }
}
```

### login 함수 수정 (`xai.ts`)

```typescript
export async function loginXai(ctrl: OAuthController): Promise<OAuthCredentials> {
  // Auto-detect: Grok CLI 로컬 토큰
  const local = detectGrokCliToken();
  if (local) {
    ctrl.log?.("Found Grok CLI token, importing automatically");
    if (local.expires < Date.now() + 60_000) {
      try { return await refreshXaiToken(local.refresh, ctrl.signal); }
      catch { /* refresh 실패 → 기존 플로우 */ }
    } else {
      return local;
    }
  }

  // 기존 browser OAuth 플로우
  return new XaiOAuthFlow(ctrl).login();
}
```

## 특이사항

### JSON 키 포맷

Grok CLI는 일반적인 flat JSON이 아닌 **compound key** (`issuer::client_id`) 구조 사용.
향후 Grok이 다른 provider를 추가하면 여러 엔트리가 공존 가능.
`XAI_AUTH_KEY_PREFIX` 매칭으로 정확히 xAI 엔트리만 추출.

### Access Token 필드명

Grok CLI는 access token을 `"key"` 필드에 저장 (비표준).
`"access_token"` 아님에 주의.

### 토큰 수명

xAI access token 수명: **6시간** (`expires_at` - `create_time` ≈ 6h).
Codex (10일)나 Anthropic (1시간) 대비 중간 수명.
refresh는 OIDC standard flow.
