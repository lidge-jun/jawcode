### 선행 사례 — cli-jaw가 같은 계열 문제를 이미 해결함 (260528)

cli-jaw 커밋 `0ff4e544` "fix(cursor): parse nested stream-json tool labels" — Cursor가 flat
name/input 대신 `tool_call.readToolCall`/`shellToolCall` 중첩 페이로드를 보내서 Tool 패널에
generic `tool {}`로 표시되던 문제를, **중첩 키(oneof variant 이름)를 순회해 도구명을 도출**하는
`parseCursorToolPayload` + `cursorToolKindLabel`(`shell→Bash`, `glob→Glob`, `mcp→MCP` 별칭 테이블)로
해결 (`/Users/jun/Developer/new/700_projects/cli-jaw/src/agent/events/cursor.ts:58-127`,
테스트 `tests/unit/cursor-events.test.ts`).

본 이슈와의 차이: cli-jaw는 **cursor-agent CLI의 stream-json 텍스트 레벨**이라 행 자체는 떴고
라벨만 망가졌지만, gjc는 **connect-RPC protobuf 레벨**에서 미처리 variant를 통째로 드롭해 행이
아예 없다. 같은 와이어 스키마(ToolCall oneof) — A안 구현 시 cli-jaw의 kindKey→라벨 별칭 테이블을
그대로 차용하면 된다 (사실상 A안의 검증된 템플릿).
