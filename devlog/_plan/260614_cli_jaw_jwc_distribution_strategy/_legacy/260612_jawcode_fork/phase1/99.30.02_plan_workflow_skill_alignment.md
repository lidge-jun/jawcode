## 2. 1단계 — `/` 자동완성 1번 = 네이티브 워크플로우 (✅ 260613 랜딩)

- `extensibility/slash-commands.ts` — `WORKFLOW_COMMAND_PRIORITIES { orchestrate: 110, goal: 105 }`,
  primary 항목에만 부여 (alias `/pabcd`는 일반 랭크). tui 자동완성 정렬이
  `matchRank → priority desc → score → index`이므로 빈 접두사 `/`에서 **1번 /orchestrate ·
  2번 /goal · 3번 티어(100) 번들 스킬 · 이후 레지스트리 순**.
- `interactive-mode.ts #rebuildSkillSlashCommands` — 핀 집합에서 **ralplan 제거**
  (`defaultGjcNames.delete("ralplan")`): 스킬은 호출 가능하게 유지, 노출만 일반 풀로 강등.
- 검증: NEW `test/slash-command-priority.test.ts` 2케이스 (핀 계층 110>105>100 + alias 비전파)
  — **2 pass**, tsc 0. orchestrate는 jaw-브랜드 게이트라 테스트는 존재 시에만 단언.

## 6. "chase로 충분한가?" 판정 (260613 — 사용자 "struct_har/chase로 따라가니까 충분할 수도?")

**불충분 — 흡수 문제에 한해. 단 3단계 보류 직감은 정확.** 근거 실측:

1. **업스트림은 정반대 방향**: `_upstream_gjc` HEAD(`75d103f`, 260613 — 오늘도 활발)에
   orchestrate **부재** (gjc-runtime에 orchestrate-runtime 없음, ralplan SKILL에 orchestrate
   언급 0). 최근 스킬 커밋 `7db68d6` 제목이 그대로 "**prioritize ralplan refinement**" (#395)
   — 업스트림은 ralplan을 계속 강화 중. **chase를 따라갈수록 ralplan 중심 문구가 더 들어온다**
   — 흡수는 심화되지 해소되지 않는다.
2. **네이티브 orchestrate/goal은 jwc 고유** (cli-jaw 포팅) — 업스트림이 이 정합을 대신
   만들어줄 가능성 없음.
3. **chase 비용 우려는 이미 예산 잡혀 있음**: [fork-delta.md:41](../../../structure/fork-delta.md)이
   SKILL.md 4종을 `HARD-EDIT / CONFLICT-EXPECTED`로 등재 (085.5-M4·C13) — 본 수술은 새
   충돌면 개설이 아니라 **기존 관리 항목에 줄이 늘어나는 것**. chase 사이클의 재적용
   정책(포크 논리 우선)이 그대로 적용된다.
4. jaw-interview는 파일명부터 포크 고유(upstream은 deep-interview) — 충돌 위험 최소.

**최종 정리**: 2단계(SKILL.md 재배선)는 chase가 대체 불가 → 마저 진행. 3단계(구조 축소)와
리포 개편은 chase 추적 비용 고려로 보류 — §5 권고 유지. 수술 완료 시 fork-delta :41 행의
merge 지침에 "ralplan→orchestrate 재배선 보존" 1구절 추가로 동행 갱신.
