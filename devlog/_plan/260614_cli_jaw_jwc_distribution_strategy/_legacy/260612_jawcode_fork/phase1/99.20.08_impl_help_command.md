`/help`를 신설해 힌트를 실명제로 만든다.

## 1. 유사 표면 조사 (260613)

| 표면 | 내용 | 판정 |
|---|---|---|
| `/hotkeys` (`builtin-registry.ts:751` → `handleHotkeysCommand` → `buildHotkeysMarkdown` + `showMarkdownPanel`) | 키보드 단축키 전용 카탈로그 | **가장 근접 — 패턴 차용** (마크다운 빌더 + read-once 패널) |
| `?` (빈 에디터, `custom-editor.ts:431`) | `/hotkeys`와 동일 핸들러 | 단축키 한정 — 커맨드 목록 없음 |
| `/tools` · `/context` · `/usage` · `/quota` | 도구/컨텍스트/사용량 패널 | 각자 전문 표면 — help 아님 |
| 슬래시 자동완성 (`/` 입력 시) | 이름+설명 드롭다운 | **탐색 가능하지만 카탈로그 아님** — 한눈에 전체를 볼 표면 부재 |
| CC `/help` (디컴파일) | local-jsx 다이얼로그 — 사용법 + 커맨드 | 컨셉 레퍼런스 |

결론: **전체 커맨드 카탈로그 + 입문 안내**를 겸하는 표면이 없음 → 신설.

## 2. 설계

- **데이터 소스 = 자동완성과 동일 목록**: `refreshSlashCommandState()`가 조립하는
  `[builtin + hook + custom + skill + file]` 결합 리스트를 `allSlashCommands` 필드로 보존
  (`interactive-mode.ts`, ctx 노출). `/help`가 이걸 렌더하므로 **자동완성과 카탈로그가
  구조적으로 어긋날 수 없다**. 초기화 전 호출은 `BUILTIN_SLASH_COMMAND_DEFS` 폴백.
- **표면 = `showMarkdownPanel`** (`/hotkeys`·`/tools` 동형): 99.20.07이 이 패널군을 도킹
  후보(b)로 분류해 두었으므로, `/help`도 같은 경로를 타면 **도킹 변환 시 자동 동승**한다 —
  지금 별도 도킹을 만들지 않는 것이 의도적 결정.
- **본문 2부**: ① Getting around — `Enter` 전송 · `!command` bash 모드(실측
  `input-controller.ts:262`) · `/command` · `?`/`/hotkeys` · 종료 더블프레스(99.20.06 연동,
  키 표기는 `getDisplayString` 실바인딩) ② Commands — 이름 정렬 + 중복 제거 + 설명의 `|`
  이스케이프(테이블 보호) 카탈로그.
- **ACP/텍스트 경로 분리**: `handle`은 빌트인 목록을 plain text로 출력
  (`acpDescription ?? description`). file/skill 커맨드는 ACP가
  `available_commands_update`로 별도 광고하므로 중복 나열하지 않는다.

## 3. 비범위

- 도킹 패널 변환 — 99.20.07 P-수리 소관 (showMarkdownPanel군 일괄).
- 커맨드 출처별 그룹핑(builtin/skill/custom 섹션 분리) — 결합 리스트의 description에 이미
  출처 표기(`(project)` 등)가 섞여 있어 v1은 단일 정렬 테이블. 필요 시 후속.
- `/help <command>` 상세 보기 — 서브커맨드/usage 표시는 자동완성 ghost hint가 담당 중.

### 5.1 Claude Code — `/help` = local-jsx 탭 다이얼로그 (cli.pretty.js:486436–486790)
