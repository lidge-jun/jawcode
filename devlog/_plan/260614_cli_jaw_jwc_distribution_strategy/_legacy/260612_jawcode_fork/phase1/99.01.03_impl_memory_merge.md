## 0. 실측 보정 (설계 대비 변경점 — 이 2건 외 설계 그대로)

1. **`refreshBaseSystemPrompt()`는 세션 인스턴스 메서드** (`agent-session.ts:3805`) — standalone CLI 프로세스(`jwc memory save`)에서 타 프로세스의 라이브 세션을 갱신할 방법은 **없다**.
   - 구현: M1 runtime이 optional `session?: AgentSession`을 받아 **있으면** `await session.refreshBaseSystemPrompt()` 호출(설계 ⑦ 유지), CLI 경로(M3)는 세션이 없으므로 스킵하고 `enqueueGlobalWatermark(forceDirtyWhenNotAdvanced:true)`로 다음 rebuild/phase2 합류 보장. M10 규약 문서에 이 동작을 명문화.
2. **Task Snapshot(M6) 주입 레일 = `beforeAgentStartPrompt` 훅** — `MemoryBackend.beforeAgentStartPrompt?(session, promptText)` (`memory-backend/types.ts:63`)가 `agent-session.ts:3822`에서 소비되어 그 턴의 시스템 프롬프트에 append(fail-open, hindsight 선례 `hindsight/backend.ts:131`).
   - `buildMemoryToolDeveloperInstructions` 시그니처 변경(설계의 optional promptText 안) 대신 **localBackend에 beforeAgentStartPrompt 구현**을 채택 — 기존 함수 무변경, promptText 자연 수신, 기존 5000토큰 summary 주입과 병행이라는 설계 의도(99.01.02 §4 [확정])에 정확히 부합.

## 2. 모듈 사양 (diff-수준)

### M2 — `memories/local-query.ts` (전부 from-scratch, 검색면 코드베이스에 부재 실측)

재사용 배관 (전부 export 확인): `openMemoryDb`/`closeMemoryDb`(`storage.ts:47/91`), `upsertThreads`(`:103`, `MemoryThread = {id,updatedAt,rolloutPath,cwd,sourceKind}`), `enqueueGlobalWatermark`(`:247`, `forceDirtyWhenNotAdvanced` 확인), `getMemoryRoot`(`index.ts:1111`), `getAgentDbPath`(`dirs.ts:384`), `getMemoriesDir`(`dirs.ts:434`).

- `searchLocalMemories(agentDir, cwd, query, limit=8)`:
  - `stage1_outputs` LIKE (`raw_memory`/`rollout_summary`, threads JOIN cwd 필터 — `listStage1OutputsForGlobal`(`storage.ts:492`) WHERE 패턴 동형) + artifact 텍스트 스캔(`MEMORY.md`/`memory_summary.md` 라인 스캔).
  - 랭킹 2요소: kind 가중(profile(-4)>shared(-3)>episode(0) — MEMORY.md→profile상당, memory_summary.md→shared상당, stage1/manual→episode상당, 99.01.02 §1 매핑) + recency(`generated_at` 컬럼 — 파일명 regex 비의존, 99.01.01 §6(b)).
  - 동의어 시드 1그룹 하드코드: `pabcd ↔ plan/audit/build/check/done` (99.01.01 §6(a)).
  - 반환 `{ref, kind, line?, snippet(≤700자), score}` 상위 8.
- `readLocalMemoryArtifact(agentDir, cwd, ref, lines?)`: ref 어휘 `summary|memory|raw|stage1:<thread_id>|rollout:<slug>` + basename 호환 + `--lines a-b`.
- `saveLocalMemoryManual(agentDir, cwd, file, content, kind?)`: **직접 `INSERT...ON CONFLICT(thread_id) DO UPDATE`** (ownershipToken 함정 — `markStage1SucceededWithOutput`(`storage.ts:298`)은 running job 토큰 검증이라 부적합, 실측 확인) + `upsertThreads([{id:"manual:<file>", sourceKind:"manual", ...}])` + frontmatter `kind` 기록 + `enqueueGlobalWatermark(db, now, cwd, {forceDirtyWhenNotAdvanced:true})`. manual row는 `claimStage1Jobs`의 `cli|app` 필터(`storage.ts:184`) 비대상·`listStage1OutputsForGlobal` cwd 필터에는 포함 → phase2 자연 합류 (설계 [확정]③).
- `contextLocalMemory(agentDir, cwd, ref)`: ref의 thread/cwd/timestamp → rollout jsonl 역참조로 주변 근거 반환.
- `buildLocalTaskSnapshot(agentDir, cwd, promptText, topN=4)`: promptText 키워드로 search 상위 4, 다양화(episode 최대 2, 파일/ref당 1 — 99.01.02 §4).

### M1 — `gjc-runtime/memory-runtime.ts`

- `resolveMemoryBackend(settings)`(`resolve.ts:19` — `memory.backend`: hindsight/local/else off) id로 분기.
