지금 thinking 블럭은 "보이거나(전문) 숨거나(0줄)" 둘뿐이다. 도구 출력이 083.1에서 얻은 중간 상태 —
한 줄로 접혀 있다가 원할 때 개별로 펼치는 — 가 thinking에는 없어서, 긴 reasoning이 화면을 통째로
밀어내거나(보임) 아예 사라진다(숨김). 이 플랜은 도구 접힘과 **같은 문법**으로 thinking을 접고,
thinking만 골라 펼치는 입구를 더한다.

## 현재 동작 (코드 사실, 260612 조사)

- thinking은 별도 컴포넌트가 아니라 **`AssistantMessageComponent` 내부 렌더**다
  (`assistant-message.ts:166` — `hideThinkingBlock`이면 thinking 파트 skip).
- 전역 토글은 **ctrl+t** (`app.thinking.toggle` → `input-controller.ts:1032
  toggleThinkingBlockVisibility`): `hideThinkingBlock` 불리언 반전 → 세팅 영속
  (`settings.set("hideThinkingBlock", …)`) → `session.agent.hideThinkingSummary` 동기 → 채팅 전체 리빌드.
  즉 **2-상태 전역 스위치**(전문 ↔ 비표시), 접힘(1줄 요약) 상태·개별 제어 없음.
- 083.1이 만들어 둔 재사용 자산: 도구 셀 3-상태(minimized/preview/expanded), **ctrl+↑ 포커스 링**(셀
  탐색 + 개별 펼침), **alt+t 전체 transcript 오버레이**(`tool-transcript-overlay.ts`).

## 설계 (제안)

### 1. thinking 셀 3-상태 (도구와 동형)

`hideThinkingBlock` 불리언을 **표시 모드 3값**으로 승격: `expanded`(현 "보임") / `collapsed`(신규 기본
후보) / `hidden`(현 "숨김"). `collapsed`는 도구 minimized와 같은 1줄 요약:

```
✻ thinking · 84 lines — 첫 문장 미리보기…
```

- 구현 위치: `assistant-message.ts` thinking 파트 렌더 분기 + 모드 전달
  (`ui-helpers.ts:264` 생성자 경로, 스트리밍 중에는 현행대로 전문 표시 후 완료 시 접힘).
- 세팅: `hideThinkingBlock`(불리언) → `thinking.display`("expanded"|"collapsed"|"hidden") 마이그레이션,
  구 키는 읽기 호환(true→hidden, false→expanded).
- 스트리밍 중 예외: 진행 중인 thinking은 접지 않는다(언제나 tail 표시) — 완료 시점에 collapsed로 정착.
  083.1 "새 도구 시작 시 이전 도구 minimize"와 같은 리듬.

### 2. 개별 펼침 — 083.1 포커스 링에 thinking 셀 합류

ctrl+↑ 포커스 링이 도구 셀만 돌고 있다면 thinking 셀을 같은 링에 편입 — 포커스 + Enter/Space로
그 thinking만 expand/collapse. 신규 키 없음, 학습 비용 0.

### 3. thinking 전용 펼침 입구 (커맨드 — 사용자 결정 대기)

"thinking만 한꺼번에 펼치는 버튼" 후보 (장단 비교, **추천 = a**):

| 안 | 입구 | 장점 | 단점 |
|----|------|------|------|
| **a (추천)** | **ctrl+t를 3-상태 순환으로 승격**: expanded → collapsed → hidden → … | 기존 키 그대로, 키맵 불변, "thinking 보기 조절 = ctrl+t" 멘탈모델 유지 | 2-상태에 익숙한 손이 한 칸 더 눌러야 함 |
| b | 신규 키 (예: alt+shift+t) "모든 thinking 펼침/접힘" | ctrl+t 의미 보존 | 키 추가 + 키맵 혼잡, alt+t(transcript)와 혼동 |
| c | `/thinking` 슬래시 커맨드 (`/thinking expand\|collapse\|hide`) | 디스커버리 좋음, 인자로 정밀 제어 | 키보드 흐름이 끊김 — 보조 입구로는 a와 병행 가능 |
| d | alt+t transcript 오버레이에 thinking 포함 + 필터 키 | "전부 다시 보기" 동선 일원화 | 인라인 채팅에서 바로 펼치는 요구와는 결이 다름 |

[기본값] 제안 조합: **a (ctrl+t 3-상태) + 2 (포커스 링 개별 펼침)**, c는 보조로 후속.
→ **커맨드 확정은 사용자 응답 대기** — 본 문서는 여기까지 (구현 착수 금지).

## 구현 기록 (260612)

사용자 결정으로 §설계가 단순화됐다: 3-상태(hidden 포함) 순환 대신 **collapsed ↔ expanded 2-상태 토글**,
hidden은 legacy 세팅 전용으로 격하. 포커스 링 편입(§설계 2)은 후속(아래 잔여).

- `assistant-message.ts` — `#thinkingExpanded`(기본 false=접힘) + `#streaming` 플래그.
  완료된 thinking은 1줄 요약 `Thinking … +N lines`(italic thinkingText + dim, 도구 minimized와 정렬)로,
  `setThinkingExpanded(true)`면 전문 markdown. **스트리밍 tail 예외**: `#streaming`이고 그 블럭이 세그먼트
  마지막 항목이면 항상 전문(라이브 tail 가독성). `setExpanded()` = setThinkingExpanded 별칭 — 도구와 같은
  duck-type 프로토콜이라 **ctrl+o 전역 스윕(`setToolsExpanded`의 isExpandable 순회)에 자동 합류**.
  ⚠️ stopReason은 스트리밍 신호로 못 씀(부분 메시지도 `stopReason:"stop"` 캐리, anthropic.ts:1030) →
  명시적 setStreaming 라이프사이클 채택.
- `input-controller.ts` — `toggleThinkingBlockVisibility`(ctrl+t) 재정의: `ctx.thinkingExpanded` 토글 →
  채팅 내 AssistantMessageComponent 전체 + streamingComponent에 적용. **세팅 미영속**(세션 스코프,
  toolOutputExpanded와 동일 결). legacy `hideThinkingBlock=true`였다면 첫 ctrl+t가 hidden 해제 →
  collapsed로 복귀(보이지 않는 상태를 토글하는 헛손질 방지). `toggleToolOutputExpansion`(ctrl+o)은
  thinkingExpanded를 동기로 끌고 가 전체 트리거화.
- `event-controller.ts` — streamingComponent 생성 2곳(신규 메시지·083.3 세그먼트 컷)에서
  `setThinkingExpanded(ctx.thinkingExpanded)+setStreaming(true)`, 세그먼트 컷 시 이전 세그먼트
  `setStreaming(false)`(tail 정착), message_end에서 `setStreaming(false)` 후 최종 updateContent.
- `ui-helpers.ts` 리빌드 경로 + `types.ts`/`interactive-mode.ts` `thinkingExpanded` 상태 추가.
- 테스트: `thinking-collapse.test.ts` 신규 5건(기본 접힘 / ctrl+t 펼침·재접힘 / ctrl+o 프로토콜 /
  스트리밍 tail 유지→정착 / legacy hideThinkingBlock 라벨) + abort-render 페이크에 setStreaming 스텁.
  전체 스위트 5382 pass — 실패 6건은 전부 기존 베이스라인(4) + 동시 세션 WIP(2), 본 변경 회귀 0.
- upstream vs fork: 083.1과 같은 결의 일반 UX 개선 — brand 게이트 없음, 업스트림 PR 후보.
