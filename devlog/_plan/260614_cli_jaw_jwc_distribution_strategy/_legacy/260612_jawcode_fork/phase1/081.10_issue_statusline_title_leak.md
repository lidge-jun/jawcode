# 081.10 — issue: 상태줄에 세션 타이틀 전문 누출 → 우측 스탯 잘림

> 상태: ✅ 수정 완료 (2026-06-12, fork). 소속: 081 밴드 (composer 행동 + 그 여파).
> 관련: [081.2](./081.2_issue_title_hallucination.md) (환각 타이틀 캡 — 업스트림 PR #515 머지).
> 입력: 사용자 스크린샷 — 상태줄이 "ㅎㅇㅎㅇ! 무엇을 도와드릴까요? 코딩, 디버깅, 프로젝트 설정 등
> 편하게 말해 주세요 / ⤴50.7/s / ⊞48.2%/200K / (sub)" — "내가 제대로 보고 싶은 건 맨 오른쪽인데
> 텍스트 누출 때문에 잘린다. 그 텍스트는 TUI에 렌더하지 않도록 해."

상태줄(에디터 상단 보더)에 어시스턴트 인사말 전문이 박혀 우측 스탯(토큰 속도·컨텍스트%·과금)을
밀어내고 있었다. 정체는 **`session_name` 세그먼트** — 자동 생성 세션 타이틀을 그대로 렌더하는
우측 그룹의 첫 세그먼트다. composer가 타이틀 생성에서 인사말 답변 전체를 타이틀로 저장하는
081.2 계열 습성과 만나면 한 문장짜리 "타이틀"이 상태줄 폭을 통째로 먹는다. 사용자 지시대로
세그먼트가 **아무것도 렌더하지 않게** 껐다 — 세션 액센트 색(갭 페인팅)은 타이틀 텍스트와
무관하게 계속 동작한다.

---

## 1. 경로 (코드 사실)

| 위치 | 역할 |
|---|---|
| `status-line/presets.ts` | 모든 프리셋의 `rightSegments`가 `["session_name", …]`로 시작 |
| `status-line/segments.ts:478` `sessionNameSegment` | `sessionManager.getSessionName()` 텍스트를 액센트 색으로 렌더 (길이 캡 없음) |
| `session-manager.ts:2858` `getSessionName()` | 자동 생성 타이틀(title-generator) 또는 /rename 값 |
| `status-line.ts:801` | 갭 페인팅용 액센트 — 세그먼트와 **독립**으로 name을 읽음 |

누출 조건: composer 자동 타이틀이 길 때(081.2의 캡은 환각 *저장*을 줄였지만 짧음을 보장하지
않음). 우측 그룹이 길어지면 잘리는 건 그룹 끝쪽 = 스탯이다.

## 2. 수정 (fork)

`segments.ts` `sessionNameSegment.render` → 항상 `{ content: "", visible: false }`.
타이틀 텍스트는 상태줄에서 영구 제거(사용자 지시 "걍 렌더하지 않도록"). 갭 액센트 색·터미널
타이틀·/resume 목록 등 다른 타이틀 소비처는 무변경.

검증: `status-line-overflow.test.ts` — 액센트 갭 테스트 2건은 우측 그룹을 `path`로 유지하도록
조정(빈 우측 그룹이면 갭 자체가 안 그려지므로), 신규 테스트 1건("타이틀 텍스트가 상태줄에 절대
안 나옴") 추가. status-line 3파일 **17/17 pass**, `cli-command-surface` 10 pass, biome clean.

## 3. 남김

- 근본 원인(긴 자동 타이틀)은 별개 — title-generator에 표시용 길이 캡(예: 32열)을 두면
  `/resume` 목록 가독성도 좋아짐. 후속 결정감.
