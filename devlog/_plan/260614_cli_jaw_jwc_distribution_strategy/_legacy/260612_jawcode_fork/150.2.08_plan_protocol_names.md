> 상태: ⬜ 연기 — 외부 호환 파괴. 150.1 P-3 `migrateHermesSetupOnce()`와 묶음, 최후.

## 리스크 / 호환

- **MCP 도구명 변경 = 계약 파괴**: 외부 Hermes/MCP 클라이언트가 `gjc_coordinator_*`를 호출 중이면 깨짐 → `migrateHermesSetupOnce()`(150.1 D-150-3, **현재 미구현 — 차단**) 선행.
- **ACP `name`**: cli-jaw acp-host(112.3)는 호스트라 영향 적으나, 외부 ACP 클라이언트(Zed 등)가 agentInfo.name으로 분기하면 영향.
- harness 식별자: `harness start --input '{"harness":"gajae-code"}'` 사용처(`commands/harness.ts:465`) 동시 갱신 — alias 기간 권장.

## 절차

1. `migrateHermesSetupOnce()` 설계·구현(150.1 D-150-3) 선결.
2. MCP 도구명: `gjc_*` → `jwc_*` + 구명 alias 등록(전환 기간 양쪽 수신).
3. harness 식별자: `"gajae-code"` 수신 유지 + `"jawcode"` 추가(읽기 호환), 발행만 신명.
4. ACP `name` 전환 + 클라이언트 호환 확인.

## 완료 기준

- [ ] `migrateHermesSetupOnce()` 구현(150.1 P-3 차단 해소).
- [ ] MCP/harness alias 기간 동작(구·신 양쪽 수신) + coordinator e2e.
- [ ] ACP `initialize` 신명 + 호스트(112.3) 무영향 확인.
