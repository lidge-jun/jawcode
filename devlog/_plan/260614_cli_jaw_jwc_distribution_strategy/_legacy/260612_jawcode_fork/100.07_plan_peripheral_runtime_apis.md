## 목표

Node SDK import와 기본 스트리밍 경로를 막는 주변 Bun API를 처리한다.

대상:

| 항목 | API | 처리 |
|------|-----|------|
| J | `Bun.serve` | `node:http` / `ws` thin wrapper 또는 스코프 제외 분리 |
| O | `Bun.Archive` | `node-tar` + `fflate` |
| P | `Bun.gc` | Node no-op |
| M | `Bun.semver` | Node SDK entry에서 제외 확인 |

## 범위

### 포함

- `bun-serve.ts` 최소 구현 또는 명시적 deferred guard
- `bun-archive.ts` tar/zip 읽기·쓰기 분기
- `bun-gc.ts` no-op
- `Bun.semver`가 Node SDK 번들에 들어오지 않는지 확인

### 제외

- auth-broker/gateway 서버 제품화
- bridge-mode WebSocket 전체 e2e
- archive tool 전체 regression matrix

## 결정 기준

`Bun.serve`는 세 그룹으로 나눈다.

| 그룹 | 처리 |
|------|------|
| OAuth callback/auth broker/gateway | M2 엔진 포팅 핵심 밖이면 deferred 가능 |
| `bridge-mode.ts` / `eval/py/tool-bridge.ts` | Node SDK import에 걸리면 thin wrapper 필요 |
| TUI-only 경로 | 제외 |

## 구현 순서

1. Node SDK bundle graph에 `Bun.serve` 사용처가 포함되는지 확인
2. 포함되면 `node:http` wrapper 작성
3. WebSocket upgrade가 필요한 경우 `ws` 의존성 사용 여부 결정
4. Archive는 tar/gzip 우선, zip은 기존 `fflate` 경로 재사용
5. `Bun.gc`는 no-op로 연결
6. `Bun.semver`는 `cli.ts`가 bundle graph에 없는지 확인

## 완료 기준

- Node SDK import가 serve/archive/gc/semver에서 막히지 않음
- deferred API는 명시적 에러 메시지와 문서 기록을 가짐
- archive 경로를 건드렸다면 focused archive test 통과
