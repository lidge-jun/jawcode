# 100.00 — Node 포팅 착수 전 준비 게이트 정리

> 상태: ✅ 완료
> 상위 문서: [100_moc_node_porting.md](./100_moc_node_porting.md)
> 작성일: 2026-06-12

## 목적

100 밴드(Node 포팅) 구현에 들어가기 전에, Phase A의 기본 준비 게이트가 실제로 실행 가능한지 확인하고 게이트를 막는 생성물/정리 드리프트를 먼저 고정한다.

이 문서는 **구현 착수 전 정리 작업**만 다룬다. `packages/jwc/src/shims/`, `dist-node/`, Node 런타임 셰임, esbuild 빌드 파이프라인 구현은 100 본작업/100.1 이후 범위다.

## 범위

### 포함

- `bun run check:jwc-ui` 실행 가능 상태 확인
- `bun run check:ts`의 준비 게이트 드리프트 확인
- Biome가 error로 처리하는 단순 unused/import-order 정리
- 생성 JSON schema 드리프트 갱신
- 100 구현 착수 전 현재 게이트 상태 기록

### 제외

- Bun API 셰임 구현
- Node dist 빌드 파이프라인 작성
- `bun:sqlite` → `better-sqlite3` 치환
- `Bun.JSONL.parseChunk` 재구현
- `Bun.serve`/archive/spawn/hash 포팅
- 테스트 러너를 vitest/node:test로 바꾸는 작업

## 현재 변경

## 확인된 게이트 상태

| 게이트 | 결과 | 비고 |
|--------|------|------|
| `bun run check:jwc-ui` | ✅ 통과 | `verify-jwc-ui-redesign.ts` + `rebrand-inventory.ts --strict` 모두 green |
| `bun run check:ts` | ✅ 통과 | `check:tools`, `check:node20-baseline`, `check:schemas`, `check:jwc-ui`, workspace `check` 완료 |

초기 확인 시 발견된 준비 드리프트:

- `check:tools` Biome error 2건
- `check:schemas` generated schema drift

현재 변경으로 위 드리프트는 정리되었다. 남은 Biome 항목은 warning/info 수준이며 게이트 실패를 만들지 않는다.

## 100 본작업 착수 전 상태

100 구현 착수 전 Phase A 기본 게이트는 통과 상태다.

```bash
bun run check:ts
bun run check:jwc-ui
```

그리고 100 본작업의 실제 착수 조건은 여전히 [100_moc_node_porting.md](./100_moc_node_porting.md)의 Phase A/B 기준을 따른다.
