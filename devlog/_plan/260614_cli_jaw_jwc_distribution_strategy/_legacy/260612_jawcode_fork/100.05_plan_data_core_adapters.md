## 목표

Node SDK의 실제 스트리밍/인증/세션 경로에 필요한 데이터 핵심 API를 구현한다.

대상:

| 항목 | API | Node 대응 |
|------|-----|-----------|
| D | `bun:sqlite` | `better-sqlite3` |
| F | `Bun.hash`, `Bun.CryptoHasher`, `Bun.SHA256` | `xxhash-wasm`, `node:crypto` |
| H | `Bun.JSONL.parseChunk` | 자체 구현 |
| I | `Bun.JSON5.parse` | `json5` |
| L | `Bun.stripANSI` | `strip-ansi` |

## 범위

### 포함

- SQLite import/constructor adapter
- hash bigint 반환 호환
- SHA-256 digest 호환
- JSONL parseChunk 자체 구현
- JSON5/ANSI 단순 wrapper
- 핵심 단위 테스트 실행

### 제외

- archive/serve
- auth-broker 전체 서버 e2e
- 실 프로바이더 스트리밍

## 설계

### SQLite

`bun:sqlite`는 esbuild에서 직접 resolve할 수 없으므로 두 가지 중 하나를 택한다.

기본값:

1. esbuild alias/plugin으로 `bun:sqlite` → `packages/jwc/src/shims/bun-sqlite.ts`
2. shim 내부에서 `better-sqlite3` import
3. `Database`, `Statement` 호환 surface 제공

필수 호환:

```ts
new Database(path)
db.prepare(sql).get(...)
db.prepare(sql).all(...)
db.prepare(sql).run(...)
db.exec(sql)
db.close()
```

### Hash

- `Bun.hash(input)`는 bigint 반환 인터페이스 유지
- `.toString(36)` 체인이 깨지지 않아야 함
- xxhash wasm async init 문제는 module-level lazy promise로 처리
- SHA-256은 `node:crypto.createHash` 사용

### JSONL

`packages/utils/src/stream.ts`의 `parseChunk(input, start?, stop?)` 시그니처를 보존한다.

검증 기준:

```bash
bun test packages/utils/test/stream.test.ts
```

### JSON5 / stripANSI

- `json5` ESM import
- `strip-ansi` ESM import
