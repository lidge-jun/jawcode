# 99.10.01 — plan: thinking 셀 포커스 링 편입 + alt+t 오버레이 포함

> 상태: ⬜ 플랜 (260612) — [083.5 §잔여](./083.5_plan_thinking_collapse.md) 마이그레이션.
> 소속: [99.10.00 MOC](./99.10.00_moc_frontend_uiux.md).
> 선행: 083.5 본편 ✅ (thinking 1줄 접힘 기본 + 전역 전환 + 스트리밍 tail 예외 — 구현 완료).

083.5에서 thinking 블럭은 완료 시 1줄 요약(`Thinking … +N lines`)으로 접히고 전역 입구로만
일괄 펼침이 가능하다. 도구 셀은 ctrl+↑ 포커스 링으로 **개별** 펼침이 되는데 thinking 셀만
동선에서 빠져 있다 — 같은 "접힌 셀"인데 조작 모델이 달라 UX 비대칭. 본 플랜은 thinking 셀을
포커스 링에 편입해 도구 셀과 동형 동선을 만들고, alt+t transcript 오버레이의 thinking 포함
여부를 함께 결정한다.

## 작업

1. **포커스 링 편입** — ctrl+↑ 순회 대상에 접힌 thinking 셀 추가. 포커스 상태에서
   Enter(또는 기존 도구 셀과 동일 키)로 `setThinkingExpanded(true)` 개별 토글.
   - 진입점: `assistant-message.ts`의 `setExpanded()` 별칭이 이미 도구와 같은 인터페이스 —
     포커스 링 등록부에서 thinking 셀을 제외하는 필터만 제거하면 될 가능성 (착수 시 실측).
2. **alt+t 오버레이** — transcript 오버레이에 thinking 전문 포함 여부 결정 후 반영.
   - 기본 제안: 포함하되 도구 출력과 동일한 dim/italic 스타일 — 오버레이는 "전체 기록 열람"
     맥락이라 누락이 더 어색함. 사용자 1회 확인 후 확정.

## 완료 기준

- [ ] ctrl+↑ 링에서 접힌 thinking 셀에 포커스 도달 + 개별 펼침/접힘 토글
- [ ] 도구 셀 포커스 링 회귀 0
- [ ] alt+t 오버레이 thinking 포함 방향 확정 + 반영
- [ ] 스트리밍 tail 예외(083.5) 무간섭 — 라이브 중 전문 표시 유지
- [ ] 테스트 + 083.5 §잔여에 본 문서 참조 기입
