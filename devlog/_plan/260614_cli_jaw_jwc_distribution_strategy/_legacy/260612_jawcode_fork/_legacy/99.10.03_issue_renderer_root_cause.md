# 99.10.03 — issue: 스크롤백 아티팩트 렌더러 단일 원인 추적

> 상태: ⬜ 조사 대기 (260612) — 라이브 로그 확보가 선행 조건.
> 소속: [99.10.00 MOC](./99.10.00_moc_frontend_uiux.md).
> 출처: [086.1 배너 출렁임](./086.1_issue_banner_scroll.md) §잔여 + [086.2 잔상/조각 redraw](./086.2_issue_scrollback_fragments_redraw.md) — 동일 계열 이월.

086.1(수직 반응형 컴팩트 배너)과 086.2(`/redraw` 커맨드)로 증상은 둘 다 완화됐지만, 두 이슈가
공유하는 **렌더러 단일 원인은 미특정**이다. 완화 패치는 증상 차단이지 수리가 아니라서 — 새 TUI
컴포넌트(99.04 HUD 띠, 99.09 하단 고정 컴포저)가 들어올 때마다 같은 계열 아티팩트가 재발할
리스크가 남는다. 본 문서는 근본 원인 추적의 소유자다.

## 가설 공간 (086.1/086.2에서 이월)

- 스크롤백 영역 재그리기 시 diff 기준선 어긋남 (부분 redraw가 이전 프레임 좌표 참조)
- 터미널 리사이즈 이벤트와 렌더 루프 경합 (배너 출렁임이 수직 리사이즈에서 빈발한 점과 정합)
- 완화책이 가린 제3 원인 — 컴팩트 배너·수동 `/redraw` 둘 다 "다시 그리면 사라짐" 패턴

## 조사 계획

1. **라이브 로그 채널 확보** — 재현 시점의 렌더 프레임 로그(diff 입력·출력 좌표) 캡처 훅.
   재현이 산발적이라 상시 로깅 토글(`JWC_TUI_RENDER_LOG=1` 류) 신설이 선행.
2. 재현 트리거 좁히기 — 수직 리사이즈 / 긴 스트리밍 중 자동접힘 / 오버레이 진입·이탈 각각 단독 재현 시도.
3. 단일 원인 특정 시 → 수리 plan으로 승격 (`99.10.0N` 신규). 30분 룰: 로그 확보 후에도 특정
   실패가 반복되면 "재현 불가 — 완화로 종결" 판정을 명시 기록하고 닫는다.

## 완료 기준

- [ ] 렌더 로그 토글 배선 (재현 대기 상태 진입)
- [ ] 단일 원인 특정 → 수리 plan 승격 **또는** 재현 불가 판정 명시 기록
- [ ] 086.1/086.2 §잔여에 본 문서 참조 기입
