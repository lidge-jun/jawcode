# 150.2.03 — M-G3: 가드 스크립트 브랜드 상수화 (1차·즉시)

> 상태: ⬜ 실행 대기 — **즉시 가능** (063.1 M3 승계).
> 상위: [150.2 §1-E](./150.2_plan_namespace_rename.md) (M-G3) · 설계: [phase1/063.1 M3](./phase1/063.1_plan_package_scope_migration.md).
> 핵심: 하드코딩된 `@gajae-code`/`gajae-code`를 **단일 상수**로 추출 → 미래 rename(150.2.07) 시 1파일만 변경, 업스트림 충돌 국한.

## 목표

가드/릴리스 스크립트의 브랜드 분모를 `WORKSPACE_SCOPE`/`ROOT_PACKAGE_NAME` 상수로 모은다. **값은 현행 유지**(`@gajae-code`/`gajae-code`) — 추출만. 이게 150.2.07(스코프 전수치환) 실행의 반자동화 발판.

## 실행 절차

1. 신규 `scripts/_brand-constants.ts` (또는 기존 공용 모듈): `export const WORKSPACE_SCOPE = "@gajae-code"`·`ROOT_PACKAGE_NAME = "gajae-code"`·`SCHEMA_DOMAIN = "gajae.ai"`.
2. 위 .ts 스크립트들이 상수 import로 전환 (`.sh`는 변수로 — `SCOPE="@gajae-code"`).
3. **값 불변** → g002/rebrand 가드 결과 동일해야 함.
4. `generate-json-schemas.ts`의 `gajae.ai` 도메인은 별도 결정 필요(스키마 `$id` 변경 = 외부 참조 영향) — 상수화만 하고 값 변경은 150.2.07로 연기.
