# 150.2.09 — M-H: `GJC_*` 폴백 제거 (2차·최후)

> 상태: ⬜ 연기 — alias 안정화 + 포크 독립 후 최후. 150.2.04(헬퍼) 선행 필수.
> 상위: [150.2 §1-C](./150.2_plan_namespace_rename.md) (M-H).

## 목표

150.2.04에서 도입한 `readBrandEnv` = `JWC_X ?? GJC_X`의 **GJC_ 폴백(읽기)을 제거** → `JWC_X` 단독. 이때 비로소 `GJC_` 흔적이 코드에서 사라진다.

## 전제 (전부 충족 후에만)

- 150.2.04 헬퍼로 전 사용처가 `readBrandEnv` 경유 (인라인 `process.env.GJC_*` 잔여 0).
- 포크 독립 선언(150.2.07과 동일 트리거) — 업스트림이 같은 env명을 쓰는 한 폴백 제거는 분기 발생.
- 외부 사용자/스크립트가 `GJC_*`를 직접 세팅하지 않음을 공지·확인 (breaking change).

## 절차

1. `readBrandEnv`를 `JWC_X`만 읽도록 축소(폴백 제거).
2. `grep -rn "GJC_" packages/ scripts/` → 보존 분모(migrate-config-dir 등) 외 잔여 0 확인.
3. E2E/test 전용 `GJC_*`(150.2.04 미전환분)는 별도 판단 — 테스트 픽스처는 보존 가능.

## 리스크

- breaking: `GJC_*` 직접 세팅 사용자 → 릴리스 노트 공지 + 1릴리스 deprecation 경고 권장.

## 완료 기준

- [ ] 폴백 제거 + `JWC_*` 단독 e2e.
- [ ] `GJC_*` 잔여 grep = 보존 분모만.
- [ ] breaking change 릴리스 노트 기재.
