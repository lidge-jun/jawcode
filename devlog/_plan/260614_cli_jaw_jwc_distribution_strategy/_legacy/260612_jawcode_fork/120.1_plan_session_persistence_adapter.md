## 2. 엔진 측 — SessionManager 외부 주입 (안 A 채택)

`createAgentSession`이 이미 지원: `options.sessionManager` 있으면 SDK가 파일을 새로 만들지 않음 (`sdk.ts:877`), 기존 branch 존재 시 `agent.replaceMessages` 자동 (`sdk.ts:916`).

```typescript
// JawRuntime.init() (110.3 A)
const sessionManager = fresh
  ? SessionManager.create(cwd, SessionManager.getDefaultSessionDir(cwd, agentDir))
  : await SessionManager.continueRecent(cwd, SessionManager.getDefaultSessionDir(cwd, agentDir));
this.#session = (await createAgentSession({ cwd, agentDir, hasUI: false, sessionManager })).session;
```

- 기각안 기록: **안 B** SessionStorage 교체(12메서드 구현 + 동기화 버그 면적 — 과잉) · **안 C** `_persist` 훅 미러링(`@internal` API 불안정 + 스코프 초과). 검색 인덱스 필요 시 [140.1](./140.1_plan_chat_federation_adapter.md) FTS 경로가 담당.
- in-memory 모드 기존재: `SessionManager.inMemory()` (`session-manager.ts:3667`, `MemorySessionStorage` `session-storage.ts:319`) — 테스트 더블로 사용.

## 3. UI 측 — 턴 완료 시 1회 기록 (098 §3.1 계약 유지)

- **스트리밍 중간 jaw.db write 금지.** [110.4](./110.4_map_agent_event_bus.md) `agent_done` 트리거에서:
