## 목표

Node SDK 번들이 가장 먼저 밟는 고빈도 Bun API를 Node 22 대응물로 구현한다. 대상은 100.1 인벤토리의 A/B/E/K/N이다.

| 항목 | Bun API | Node 대응 |
|------|---------|-----------|
| A | `Bun.file` | `node:fs/promises` |
| B | `Bun.write` | `node:fs/promises.writeFile` |
| E | `Bun.sleep` | `node:timers/promises.setTimeout` |
| K | `Bun.stdin/stdout/stderr` | `process.stdin/stdout/stderr` |
| N | `Bun.env` | esbuild `define`으로 `process.env` |

## 구현 세부

### `Bun.file`

필요 최소 인터페이스:

```ts
Bun.file(path).text()
Bun.file(path).json()
Bun.file(path).bytes()
Bun.file(path).exists()
Bun.file(path).stat()
Bun.file(path).write(data)
```

Node 대응:

- `.text()` → `fs.readFile(path, "utf8")`
- `.json()` → `JSON.parse(await text())`
- `.bytes()` → `new Uint8Array(await fs.readFile(path))`
- `.exists()` → `fs.stat` try/catch
- `.stat()` → `fs.stat`
- `.write(data)` → `fs.writeFile(path, data)`

### `Bun.write`

초기 입력 지원:

- `string | URL` path
- `string | Uint8Array | ArrayBuffer | Blob` data

복잡한 `Response`/stream 입력은 실제 실패 지점에서 확장한다.

### `Bun.sleep`

`node:timers/promises`의 `setTimeout(ms)`를 사용한다. `Promise.race` 패턴과 호환되어야 한다.

### stdio

- `Bun.stdin.text()`는 process.stdin을 끝까지 읽어 문자열 반환
- `Bun.stdin.stream()`은 Node 22 Web ReadableStream 변환 가능 여부를 실측 후 지원
- stdout/stderr는 `process.stdout`/`process.stderr` pass-through
