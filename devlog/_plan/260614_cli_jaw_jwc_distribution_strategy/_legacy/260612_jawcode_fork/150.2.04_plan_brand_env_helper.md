# 150.2.04 — M-env-helper: `GJC_*` → `JWC_ ?? GJC_` 폴백 헬퍼 (1차)

> 상태: ⬜ 실행 대기 — 즉시 가능 (alias라 저충돌).
> 상위: [150.2 §1-C](./150.2_plan_namespace_rename.md) (M-env-helper). 조사 §3: 고유 `GJC_*` ~130개, JWC_ 미보유 ~75개.
> 핵심: 일괄 제거 금지 — **읽기 폴백 헬퍼**로 점진. GJC_ 읽기 보존 = 업스트림 분모.

## 목표

`JWC_X ?? GJC_X` 우선순위를 단일 헬퍼로 통일하고, JWC_ alias 미보유 75개에 적용. **GJC_ 폴백(읽기)은 유지** — 제거는 150.2.09(M-H).

## 헬퍼 (신규)

`packages/utils/src/brand-env.ts`:
```ts
export function readBrandEnv(suffix: string): string | undefined {
  return process.env[`JWC_${suffix}`] ?? process.env[`GJC_${suffix}`];
}
```
- 이미 `JWC_X ?? GJC_X`를 인라인으로 쓰는 곳(`orchestrate-runtime.ts:118-126` 등)도 점진적으로 헬퍼로 흡수(중복 제거, dev §6 copy-paste).

## 대상 클러스터 (조사 §3, JWC_ 미보유 ~75개)

| 클러스터 | 예시 | 우선순위 |
|---|---|---|
| `GJC_TEAM_*` (~25) | `GJC_TEAM_NAME`·`GJC_TEAM_MAX_WORKERS`·`GJC_TEAM_LEADER_CWD`·`*_EVIDENCE_STATUSES` | 중 (team 표면) |
| `GJC_TMUX_*` (~12) | `GJC_TMUX_LAUNCHED_ENV`(10.007)·`GJC_TMUX_PROFILE*`·`GJC_TMUX_BRANCH_*` | 중 |
| `GJC_HARNESS_*` | `GJC_HARNESS_ROOT_REGISTRY_DIR`·`_RPC_COMMAND`·`_SOCKET_DIR`·`_STATE_ROOT` | 중 (headless) |
| `GJC_PYTHON_*`·`GJC_PLUGIN_*`·`GJC_CRASH_DIAGNOSTICS*` | — | 하 |
| 단발 | `GJC_HARMONY_DEBUG`(agent/harmony-leak.ts:301)·`GJC_RECEIPT_SPOOL_DIR`(receipt-spool.ts:7)·`GJC_STATE_DIR`·`GJC_NO_PTY`/`GJC_NO_TMUX`/`GJC_MOUSE` | 중 |
| `GJC_E2E_*`·`GJC_*_TEST*` | 사용자 비노출 | **최하** (스킵 가능) |

> 실행 시 `grep -rn "GJC_" packages/ scripts/ | grep -v "JWC_"` 로 미보유 전수 파일:라인 재추출 (본 표는 클러스터 분류).

## 실행 절차 (클러스터 단위)

1. `readBrandEnv` 헬퍼 도입 + 단위 테스트(JWC_ 우선·GJC_ 폴백·둘 다 없음).
2. 클러스터별로 `process.env.GJC_X` 읽기 → `readBrandEnv("X")` 치환 (한 클러스터 = 한 커밋).
3. **쓰기**(`env.GJC_X = …`, 자식 spawn 전파)는 JWC_ 우선 + GJC_ 동시 세팅(자식이 어느 쪽 읽든 호환) — 또는 JWC_만 세팅하고 자식도 헬퍼로 읽게.
4. E2E/테스트 전용 클러스터는 별도 후순위 또는 스킵 기록.

## 완료 기준

- [ ] 헬퍼 + 테스트.
- [ ] 사용자 노출 클러스터(TEAM·TMUX·HARNESS·단발) 헬퍼 전환.
- [ ] `JWC_X`로 단독 동작 + `GJC_X` 폴백 동작 둘 다 e2e.
- [ ] E2E/test 전용 미전환분 명시 기록 (150.2.09 제거 대상에서 제외 판단).
