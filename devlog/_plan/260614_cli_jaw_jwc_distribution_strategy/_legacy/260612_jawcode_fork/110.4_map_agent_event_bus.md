## 2. 발화 순서 불변식 (vendor-CLI 경로와 동형 보장)

`agent_status{running:true}` → `agent_output`* → (`agent_tool` 쌍)* → `agent_done` → `agent_status{running:false}` — vendor 경로 실측 순서(`spawn.ts:1863→1921→NDJSON→lifecycle:516`)와 동일하게 유지. **중복/순서 변형 = 098 R2 리스크** — 매퍼 단위 테스트로 시퀀스 고정.

## 3. traceRun 어댑터

- turn 시작 시 `trace_runs` insert (`id, cli:"jwc", working_dir, audience, status:"running"`) — 스키마 `db.ts:144-179`.
- `trace_events`: jwc 이벤트를 `seq` 단조 증가로 기록 (`source:"jwc"`, `event_type`, `preview`, raw는 retention 정책 따름) — **seq = 조항 1 `traceSeq` 커서 원천**.
- turn 종료 시 `finished_at`/`event_count`/`byte_count` 마감.
