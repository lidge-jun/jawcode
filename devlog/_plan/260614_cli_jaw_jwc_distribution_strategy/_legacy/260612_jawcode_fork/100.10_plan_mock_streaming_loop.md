> 상태: ✅ **완료 (260613)** — 로컬 OpenAI-호환 SSE 모의 서버 + model 직주입으로 Node에서
> agent loop 1턴 완주: agent_start→turn_start→message_start→message_update→message_end→
> turn_end→agent_end, "Hello from mock" 세션 상태 왕복 확인, credential 불요, exit 0.
> 재현: `scripts/smoke-node-streaming.mjs`. 경유 수리 없음 (첫 실행 green).
> 선행: [100.09_plan_create_agent_session_smoke.md](./100.09_plan_create_agent_session_smoke.md)
> 상위 문서: [100_moc_node_porting.md](./100_moc_node_porting.md)

## 목표

실 provider 없이 Node 22에서 agent loop가 prompt 입력을 받고 assistant delta/event를 방출하는지 확인한다.

## 범위

### 포함

- mock provider 또는 test provider wiring 확인
- `session.prompt("hello")` 실행
- streaming event 수신
- tool 없는 단순 assistant 응답 검증
- agent loop에서 Bun-only API 잔존 확인

### 제외

- 실제 Claude/OpenAI 호출
- 도구 실행 e2e
- cli-jaw SSE mapping

## 접근

가능한 경로 중 기존 테스트 구조에 가장 가까운 것을 택한다.

| 후보 | 장점 | 단점 |
|------|------|------|
| 기존 `agent/test/agent.test.ts` 로직 재사용 | 검증 의미 큼 | bun:test 의존 가능 |
| SDK smoke 전용 mock provider 삽입 | 빠름 | 제품 경로와 다를 수 있음 |
| local fake model provider 추가 | 반복 가능 | 구현 면적 증가 |

기본값은 **기존 agent 테스트의 mock 구조 재사용**이다.

## 검증 포인트

- prompt 호출이 Promise로 완료됨
- assistant text delta가 1개 이상 발생
- completion/final event가 발생
- Node event loop가 hanging 없이 종료됨
- compaction/system prompt 준비 timeout이 Node sleep으로 동작

## 실패 분류

| 실패 | 처리 |
|------|------|
| stream parser 오류 | 100.05 JSONL 보정 |
| prompt text undefined | 100.06 text embed 보정 |
| sleep/timeout 오류 | 100.03 sleep 보정 |
| subprocess/tool 관련 오류 | mock에서 tool 비활성화 또는 100.04 보정 |

## 완료 기준

- Node 22 mock streaming 1턴 완료
- assistant delta + final event 관측
- credential 없이 통과
- 실패 없이 process exit
