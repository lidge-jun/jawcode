# 150.2.05 — M-D→M-E: ENGINE_NAME 플립 + 상태 마이그레이션 (2차)

> 상태: ⬜ 연기 — 1차(150.2.01~04) 완료 후. 사용자 상태 마이그레이션 동반.
> 상위: [150.2 §1-B](./150.2_plan_namespace_rename.md). 선행: **150.2.02**(리터럴 상수 경유) 필수.

## 목표

`ENGINE_NAME = "gjc"` → `"jwc"` 단일 상수 전환. 11개 소비처 자동 정합(로그·XDG·테마 분기). 런타임 config 디렉터리 `~/.gjc`→`~/.jwc` 이동을 기존 마이그레이션으로 흡수.

## 단계

### M-D — 상태 마이그레이션 검증 (선행)
- `packages/coding-agent/src/migrate-config-dir.ts` 기존재 — `~/.gjc`→`.jwc` 1회 복사. 플립 전 **이 경로가 실제로 전 상태(세션·auth·skills·memory)를 옮기는지** e2e 확인.
- status-line 세그먼트 config 키 `"gajae"`(`segments.ts:72`·`presets.ts:36,60`) 사용자 설정 마이그레이션 포함 (150.2.01에서 보류한 것).

### M-E — 상수 플립
- `dirs.ts:20` `"gjc"`→`"jwc"`. 소비처 11곳(150.2 §1-B) 자동.
- 150.2.02 리터럴이 `${ENGINE_NAME}` 경유로 전환돼 있어야 로그파일명까지 반영.

## 리스크

- 기존 사용자: `~/.gjc` 상태가 마이그레이션 안 되면 "설정 날아감" 체감 → M-D e2e가 게이트.
- 업스트림이 `dirs.ts`를 건드리는 커밋과 충돌 가능 → ENGINE_NAME 선언 1줄이라 면적 작음.
