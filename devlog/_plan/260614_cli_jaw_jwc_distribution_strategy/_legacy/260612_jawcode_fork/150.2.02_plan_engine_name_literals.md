# 150.2.02 — M-lit: ENGINE_NAME 미경유 하드코딩 리터럴 (1차·즉시)

> 상태: ⬜ 실행 대기 — **즉시 가능** (저충돌).
> 상위: [150.2 §1-G](./150.2_plan_namespace_rename.md) (M-lit).
> 핵심: ENGINE_NAME 플립(150.2.05)으로 **자동 해결 안 되는** 직접 하드코딩 — 별도 패치 필수.

## 목표

`ENGINE_NAME` 상수를 거치지 않고 `gjc`/`.gjc`를 문자열로 박은 곳을 `ENGINE_NAME` 경유로 전환 → 150.2.05 플립 시 자동 정합. (지금 당장 `jwc`로 바꾸지 말고 **상수 경유로 리팩터** — 그래야 분모 보존 + 단일 진실점.)

## 대상 (조사 §8)

| 파일:라인 | 현재 | → 전환 |
|---|---|---|
| `packages/utils/src/dirs.ts:277` | `"gjc-plugins.lock.json"` | `` `${ENGINE_NAME}-plugins.lock.json` `` |
| `packages/utils/src/dirs.ts:445` | `"gjc-crash.log"` | `` `${ENGINE_NAME}-crash.log` `` |
| `packages/utils/src/logger.ts:50` | `"gjc.%DATE%.log"` | `` `${ENGINE_NAME}.%DATE%.log` `` (import ENGINE_NAME) |

`dirs.ts`는 이미 `ENGINE_NAME`을 export/사용 중 → `:277`/`:445`는 동일 모듈 내 참조로 즉시 가능. `logger.ts:50`은 `import { ENGINE_NAME } from "./dirs"` 추가.

## 보존 (변경 금지)

- `packages/coding-agent/src/migrate-config-dir.ts:16` `LEGACY_DIR_NAME = ".gjc"` — **마이그레이션 소스**. `~/.gjc`→`.jwc` 이전을 위해 `.gjc`를 알아야 함. 보존이 정상 (150.2 §2).
- `test/migrate-config-dir.test.ts`의 `.gjc` fixture (6곳) — 동일 이유 보존.

## 의존성

- 150.2.05(ENGINE_NAME 플립)의 **선행** — 리터럴이 상수 경유로 바뀌어 있어야 플립이 전부 반영됨. 본 슬라이스를 먼저.

## 실행 절차

1. 3개 리터럴을 `${ENGINE_NAME}` 보간으로 교체 (`logger.ts`는 import 추가).
2. `grep -rn '"gjc-\|"gjc\.\|'\''\.gjc'\''' packages/utils/src` → migrate-config-dir 외 잔여 0 확인.
3. 현 상태(ENGINE_NAME="gjc")에서 로그파일명·플러그인 락 경로가 **기존과 동일** 유지됨을 확인 (리팩터라 동작 불변).
