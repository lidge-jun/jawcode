## 목표

Node 22에서 `createAgentSession()`을 실제로 호출해 세션 객체를 만들고 dispose까지 완료한다. 이 단계는 아직 provider 스트리밍을 요구하지 않는다.

## 범위

### 포함

- 임시 cwd/agentDir 생성
- `GJC_BRAND_NAME=jwc` 또는 `JWC_BRAND_NAME=jwc` env 확인
- `createAgentSession({ cwd, agentDir })` 호출
- 반환 객체 shape 확인
- dispose/cleanup 호출

### 제외

- LLM API 호출
- 도구 실행
- cli-jaw 서버 attach
- session resume

## smoke 형태

권장 스크립트:

```ts
const sdk = await import("../dist-node/sdk.js");
const session = await sdk.createAgentSession({ cwd, agentDir });
console.log(Object.keys(session));
await session.dispose?.();
```

## 검증 포인트

| 항목 | 기대 |
|------|------|
| config load | 빈 agentDir에서도 기본값 생성 또는 graceful handling |
| auth load | credential 부재가 import/creation을 막지 않음 |
| defaults load | bundled jwc defaults 로드 가능 |
| session object | prompt/interrupt/dispose 등 공개 메서드 존재 |

## 실패 분류

| 실패 | 처리 |
|------|------|
| auth storage sqlite 오류 | 100.05 sqlite adapter 보정 |
| defaults resource 오류 | 100.06 text/defaults 보정 |
| config/env path 오류 | dirs/env adapter 보정 |
| TUI 초기화 오류 | sdk entry graph에서 TUI 제거/분리 |

## 완료 기준

- Node 22에서 세션 생성 성공
- provider credential 없이도 생성 단계가 실패하지 않음
- dispose가 호출 가능
- 생성 smoke 스크립트가 재현 가능
