# 100.13 — Node 셰임 적대적 감사 루프 (loop-until-dry)

> 상태: ✅ **완료 (260613)** — 치명 0 수렴. 7라운드 병렬 적대적 감사 + red-green 수리,
> 마지막 2라운드 연속 클린(치명·high·medium 0)으로 종료.
> 선행: [100.12](./100.12_plan_node_porting_closeout.md). 입력: 사용자 "치명적 이슈
> 안 나올 때까지 병렬 감사/백그라운드로 돌려".

## 방법

차원별(sqlite·archive·file I/O·proc/shell·data·build) 감사자 6명 병렬 fan-out →
critical/high 발견마다 **적대적 검증**(기본 refute, 실제 업스트림 호출 사이트를 직접 읽고
런타임 재현) → 확정 이슈만 반환하는 워크플로우를 라운드마다 재실행. 매 라운드 dev 스킬
가이드대로 **근본원인 → 수정 → red-green 회귀 → 정적분석(tsc·biome)+스모크 게이트 → 커밋**.

## 보안 관련 (강조)

- **bridge TLS 평문(R1 치명)**: `Bun.serve`가 `tls{cert,key}`를 버려 bridge 베어러 토큰이
  평문 전송되던 것 → node:https 전환.
- **request.signal disconnect(R4)**: 스트리밍 중 끊겨도 업스트림 LLM 요청이 안 끊겨
  토큰 과금 지속 → AbortController를 클라이언트 disconnect에 배선.
- **archive 경로순회(이전 100.12 B-3)**: 유지.
