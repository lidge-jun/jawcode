# 71.5 — P2.0a A-stage audit synthesis round 1

Planner verdict: FAIL (`71.3_a_p2_0a_planner_audit_r1.md`)
Architect verdict: FAIL (`71.4_a_p2_0a_architect_audit_r1.md`)

## Root cause

The narrowed P2.0a plan is correctly scoped, but it still used high-level lifecycle language where the TUI code requires exact hook semantics: `#doRender` has early returns, the process-wide singleton is reused by replay harnesses, and `TUI.stop()` is synchronous. The auditors found the plan would otherwise be implemented at the wrong location or leak/duplicate rows.

## Decisions

- Accept PLANNER-A1 / ARCH-A1: record frame and placeholder rows in one guarded block immediately after `#truncateLinesToWidth(newLines, width)` and before diff/full-render branching. This block runs on every `#doRender` that reaches final composition.
- Accept PLANNER-A2: placeholder `recordEvent` calls fire on every anchored `#doRender` when metrics are enabled.
- Accept PLANNER-A3 / ARCH-A5: successful flush appends buffered rows then clears the buffer. Failed flush records `perf.write.error` and retains buffered rows for a later explicit flush; second stop is idempotent after successful flush.
- Accept PLANNER-A4 / ARCH-A3: `RenderMetrics.reset()` clears event buffer, counters, `eventsBuffered`, and `eventsDropped` while preserving enabled state.
- Accept PLANNER-A5: each `recordCounter` and `recordEvent` appends exactly one JSONL event row when enabled.
- Accept PLANNER-A6 / ARCH-A4: default path is `path.join(getLogsDir(), "perf.jsonl")`; `JWC_PERF_LOG` overrides it.
- Accept PLANNER-A7: write-error assertion is `snapshot().counters["perf.write.error"] === 1`.
- Accept PLANNER-A8 / ARCH-A2 / ARCH-A9: use a minimal TUI fixture, not `runReplay`, for the stop-flush integration test; enable via `renderMetrics.enable()` / `reset()` and set `JWC_PERF_LOG` to a temp path. Auto-flush from `TUI.stop()` is gated by explicit perf logging intent (`JWC_PERF=1` or `JWC_PERF_LOG` set), not programmatic enable alone, preventing replay harness pollution.
- Accept PLANNER-A9: broad plan 70 coding-agent/session/sdk blocks remain deferred to P2.0b and must not be implemented in P2.0a.
- Accept PLANNER-A10: snapshot counter key rules are normative: `recordCounter(source, name)` aggregates `${source}.${name}`; `recordEvent(source, counters)` aggregates each short key as `${source}.${key}`.
- Accept ARCH-A6: tests assert JSONL placeholders only; no comparison to `helperStats.text.visibleWidth`.
- Accept ARCH-A7: disabled no-op expected snapshot includes `counters: {}`, `eventsBuffered: 0`, `eventsDropped: 0`.
- Accept ARCH-A8: focused gates are pre-PR smoke; package default test/check must also remain compatible.

## Plan changes

- `metrics.ts` section now defines event row lifecycle, reset behavior, path resolver, and auto-flush predicate.
- `tui.ts` section pins the post-`#truncateLinesToWidth` hook anchor and placeholder cadence.
- `perf-events_p2_0` test section chooses a minimal TUI fixture and forbids replay-harness side effects.
- Red-team and done checklist now cover reset, write-error counter, zero append, idempotent stop flush, and helperStats non-comparison.

## Residual risk

The B-stage must still find a clean minimal TUI fixture. If constructor/setup friction is higher than expected, implementation may use the smallest existing TUI test harness as long as it sets `JWC_PERF_LOG` and proves `TUI.stop()` sync flush without touching replay-harness global behavior.
