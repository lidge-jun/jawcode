## Shared root cause

The R1 plan had the right slice boundaries, but several implementation contracts were not pinned tightly enough:

- P3.1 adoption logic named the migration/resident operations but did not fully mirror the current `setSessionFile()` state transitions.
- P2.2 cache behavior named the main preparation call sites but did not fully account for existing defensive truncation emit branches, ASCII fast-path requirements, and exact metrics assertion shape.
- Test contracts named outcomes but left wrapper/fixture details loose.

## Disposition by finding

### Planner findings

- PLANNER-A1 — Accepted. Added an explicit printable-ASCII fast path rule and made `packages/tui/src/utils.ts` conditional only if the helper needs to be shared.
- PLANNER-A2 — Accepted. The plan now requires `this.#needsFullRewriteOnNextPersist = migrateToCurrentVersion(this.#fileEntries)` inside adoption.
- PLANNER-A3 — Accepted. The plan now distinguishes `setSessionFile()` pre-load setup from fresh-manager `open()` and requires adoption to reset resident stores before assigning entries.
- PLANNER-A4 — Accepted. Added `CountingSessionStorage` wrapper contract: count async `readText(path)` for the resolved path used by `loadEntriesFromFile()`.
- PLANNER-A5 — Accepted. P2.2 metrics now require exactly one aggregated render-owned `recordEvent()` per anchored render, short JSONL keys, and source-qualified snapshot assertions.
- PLANNER-A6 — Accepted. Prepared-line byte fixture now includes Thai/Lao normalization-sensitive content.
- PLANNER-A7 — Accepted. Forced render cache clear is explicitly inside `requestRender(true, ...)` before scheduling.
- PLANNER-A8 — Accepted. Stop invalidation fixture now uses a new `VirtualTerminal` + new `TUI` at the same width with the same lines.
- PLANNER-A9 — Accepted. The session test now inlines minimum large resident text/image constants and exact bounded/materialized predicates.

### Architect findings

- ARCH-A1 — Accepted. Migration return assignment and migrated-session cold rewrite test are required.
- ARCH-A2 — Accepted. Adoption starts with `#resetResidentStores()` and `setSessionFile()` retains current pre-load teardown/error reset behavior.
- ARCH-A3 — Accepted. Counting storage counts async `readText()` and asserts using `path.resolve(sessionFile)`.
- ARCH-A4 — Accepted. The plan now calls out viewport/differential inline truncation branches and requires deletion after preparation or use of a single shared truncation terminator helper.
- ARCH-A5 — Accepted. Width-change cache clearing is now at the start of `#doRender()` before preparation; the width-change test expects normalize/truncate misses.
- ARCH-A6 — Accepted. `open()` resolves once and passes the resolved path to both single load and adoption.
- ARCH-A7 — Accepted as clarification. Verification text remains a focused pre-PR smoke bundle; `bun run check` is the workspace hygiene gate, not a claim of a separate CI job.

## Residual risk
