> Result: Both lenses FAIL. Revise plan before delta re-audit.

## Original design intent

The approved I/P design requires native gate enforcement from the first patch: fresh-fork interview scoring can conclude readiness, but the workflow must not enter P until the main agent has shown a short full-spec summary and the user has explicitly acknowledged it. Direct/manual P compatibility should survive only outside the strict jaw-interview handoff path.

## Shared root cause

The plan made `handoff_status` the strict-gate activation predicate but did not require every new jaw-interview final-write/handoff path to set it. This leaves the current Phase 5b path (`jwc state jaw-interview write ...` then `jwc orchestrate p`) and legacy `--write --stage final` / `--handoff` combinations able to bypass the summary gate.
