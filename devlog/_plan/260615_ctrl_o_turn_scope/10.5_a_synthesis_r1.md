# A Synthesis R1 — Ctrl+O current-turn scope

Revision: `ctrlo-r1`

## Verdicts

- Planner lens: FAIL (`agent://87-CtrlOPlannerAudit`)
- Architect lens: FAIL (`agent://88-CtrlOArchitectAudit`)

## Shared root cause

Both auditors found the plan was too narrow around user-boundary movement. It covered `message_start`, but the UI appends optimistic local user prompts in `InteractiveMode.startPendingSubmission()` before any agent event arrives. In that window Ctrl+O could still target the previous assistant turn.

## Finding decisions

### Planner

- PLANNER-A1: accept. Add optimistic submit boundary hook and test before message_start.
- PLANNER-A2: accept. Make event-controller fixture mutation a hard requirement.
- PLANNER-A3: accept. Test 4 must use an InteractiveMode public rebuild entrypoint, not a UiHelpers-only path.
- PLANNER-A4: accept. Make post-rebuild Ctrl+O exclusion mandatory and keep Ctrl+T/keybinding coverage in focused tests.
- PLANNER-A5: accept. State rebuild hook site as the end of `InteractiveMode.rebuildChatFromMessages()` after `this.renderSessionContext(context)`.
- PLANNER-A6: accept. Add real `ToolExecutionComponent` render assertion for minimized summary mode.

### Architect

- ARCH-A1: accept. Add `currentTurnStartIndex = chatContainer.children.length` after optimistic real-user append in `startPendingSubmission()`.
- ARCH-A2: accept. Rewire `event-controller-message-start.test.ts` fixture with real `Container` and mutating `addMessageToChat`.
- ARCH-A3: accept. Do not minimize active/pending tools; skip values still in `ctx.pendingTools`.
- ARCH-A4: accept. Add `currentTurnStartIndex: 0` and `toolOutputExpanded: false` to touched fixtures.
- ARCH-A5: accept. Pin the new `interactive-mode-current-turn-boundary.test.ts` to the lightweight InteractiveMode construction pattern used by `interactive-mode-status.test.ts`, or an equivalent dedicated harness.
- ARCH-A6: accept. Hook only `renderInitialMessages()` and `rebuildChatFromMessages()` wrappers.
- ARCH-A7: accept. Add `UserMessageComponent` import to `interactive-mode.ts` during implementation.
- ARCH-A8: accept as non-goal. `toggleThinkingBlockVisibility()` remains on the old eligibility contract in this cycle; Ctrl+T/full-transcript and thinking visibility are separate work.
