## Why this is a decision, not a port

The card's gap table (NEW `local-token-detect.ts`, HARD-EDIT `auth-storage.ts`) is **stale**:
jwc has since evolved its own auth lineage. The live diff shows large **bidirectional**
divergence, almost all of it independent feature work + intentional jaw fork — not a security
gap jwc is "behind" on.

- `diff auth-storage.ts` = ~400 lines, features on **both** sides.
- `diff -qr utils/oauth`: gjc-only `fugu.ts`, `glm-zcode.ts` (NEW providers); **jwc-only**
  `kiro.ts`, `local-token-detect.ts` (the card's "NEW import" already exists in jwc, jaw-built).

## Independent security audit (A-phase) — verdict

A read-only sub-agent classified every gjc-side delta as SECURITY-FIX / FEATURE-ADD /
REFACTOR. **HEADLINE: NO security fix outstanding; jwc is at-or-ahead of gjc on every safety
axis.** Key findings:

1. **Shared OAuth security primitives diff CLEAN** — `pkce.ts`, `callback-server.ts`,
   `google-oauth-shared.ts`, `github-copilot.ts`, `cursor.ts` byte-identical → no
   PKCE/state/nonce/callback fix exists on the gjc side.
2. **`chmod 0o600` + WAL + `busy_timeout`** present and identical on both sides (shared baseline,
   not a gjc-only fix). jwc additionally has `isSqliteBusyError` busy-retry (4-attempt backoff) —
   **jwc ahead** on credential-DB-open race hardening.
3. 🔴 **gjc HARDCODES Google OAuth client secrets** (`atob("R09DU1BY…")` in
   `google-gemini-cli.ts` / `google-antigravity.ts`); jwc externalized them to `$env.*` +
   `requireOAuthConfig()`. **Importing gjc wholesale would REINTRODUCE a hardcoded-secret
   exposure** — a net security regression.
4. gjc-only providers (`fugu`, `glm-zcode`, `xiaomi-token-plan-*`) are **self-contained
   FEATURE-ADD** (use only shared helpers; touch no `serializeCredential` / perms / redaction /
   SQLite store).
5. gjc-only `auth-storage` items (`AuthCredentialIfAbsent*`, `CredentialRankingMode
   "balanced"/"earliest-reset"`) are **feature/ergonomics**; IfAbsent preserves jwc's existing
   refresh-token redaction (`REMOTE_REFRESH_SENTINEL`) — no leak fixed or introduced.

## Judgment (resolves the card's "판정 1줄")

**adapt → SPLIT + conscious hold. No code migration. No security fix outstanding.**
- **Core / storage / OAuth-security**: keep jwc as-is (intentional fork; at-or-ahead of gjc).
  Explicitly do **NOT** import gjc's `auth-storage.ts` open path or `google-*` provider files
  (would delete jwc's busy-retry and reintroduce gjc's hardcoded OAuth client secrets).
- **Preserve** jaw-only lineage: `kiro` provider, `LocalTokenImportMode`/local-token-detect,
  broker/gateway, `isSqliteBusyError` (Decision B: do not delete).
- **Optional, no-urgency follow-up candidates** (only if product wants them; each its own
  security-reviewed ticket per Decision G): port providers `fugu` / `glm-zcode` /
  `xiaomi-token-plan-*`; optionally `AuthCredentialIfAbsent*` (idempotent import) +
  `CredentialRankingMode` (quota spreading). One non-security NEEDS-HUMAN-REVIEW: whether to adopt
  gjc's `resolveOAuthStorageProvider` (`openai-codex-device`→`openai-codex` storage pool dedup).
