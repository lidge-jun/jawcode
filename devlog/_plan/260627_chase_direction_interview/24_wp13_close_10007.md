## Key insight — the round-trip IS the verification

gjc's self-heal (`team-runtime.ts:1871-1886`) resolves "real leader (adopt) vs foreign (reject)"
without a launch marker: when the current session lacks `@gjc-profile`, **write** the tag
(`set-option`) then **read it back** (`show-options`). A real tmux round-trips the user option →
adopt; a provider that drops it (psmux, non-tmux mux) fails the readback → stay rejected as
`unmanaged_tmux_session`. This is exactly the Decision's "verified by tmux readback, not
restricted to launch-marker sessions." Safety boundary preserved: we only reach this code because
`display-message -t $TMUX_PANE` already proved our process is **inside** this tmux session, so
tagging the session we are executing in is not a foreign-takeover — and the round-trip blocks
fake multiplexers. jwc already hard-rejects untagged leaders (`team-runtime.ts:1655-1656`), so
self-heal is purely additive; jwc never had a `GJC_TMUX_LAUNCHED`-gated retag to widen away from.

## Source — `packages/coding-agent/src/jwc-runtime/team-runtime.ts`

   return `exitCode === 0`. Uses jwc's existing exact-target form `=${sessionName}` (matches
   `readJwcTmuxProfileValue`).
2. **MODIFY** `readCurrentTmuxLeaderContext` (jwc L1655-1656): replace the hard
   `throw unmanaged_tmux_session` with write-then-readback self-heal:
   ```ts
   if (readJwcTmuxProfileValue(tmuxCommand, sessionName) !== GJC_TMUX_PROFILE_VALUE) {
       // Self-heal: adopt any real tmux leader (incl. a session created outside
       // `gjc --tmux`, or a `gjc --tmux` pane that lost its @gjc-profile tag mid-startup)
       // by writing the ownership tag and reading it back. Real tmux round-trips the user
       // option and is adopted; a provider that drops it stays rejected as foreign/unmanaged.
       // (gjc team-runtime.ts:1871-1886)
       const tagged = tagJwcTmuxSessionAsLeader(tmuxCommand, sessionName);
       if (!tagged || readJwcTmuxProfileValue(tmuxCommand, sessionName) !== GJC_TMUX_PROFILE_VALUE)
           throw new Error(buildTeamTmuxLeaderRequirementMessage(`unmanaged_tmux_session:${sessionName}`));
   }
   ```
3. Scope kept tight: keep jwc's existing `invalid_tmux_context` strict check and the existing
   reject message format (no port of gjc's `Bun.which` install check / refined `not_inside_tmux`
   messaging / untagged-session hint — out of this card's self-heal gap).

## Risk — backend, security-sensitive (tmux ownership)

Threat model: self-heal must never retag a session we are not legitimately inside. Guaranteed by
the precondition (`display-message -t $TMUX_PANE` succeeded ⇒ our pane is in this session) + the
round-trip gate (fake multiplexers rejected). No persistence/state migration. Foreign rejection
stays strict. A-phase reviewer must adversarially confirm the foreign-reject path.
