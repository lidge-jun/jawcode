### 1. Esc draft-clear тАФ `packages/coding-agent/src/modes/controllers/input-controller.ts`

omp source (`input-controller.ts:318-335`): when Esc fires with **typed text** (not streaming/
python/bash), the draft is cleared and the double-Esc timer reset, **before** any empty-editor
double-Esc action. jwc `onEscape` currently has only `else if (!this.ctx.editor.getText().trim())`
(empty-editor double-Esc tree/branch); the **text-present case is an unhandled no-op**.

- **MODIFY** the tail branch (~L155). Split the single empty-editor branch into:
  ```ts
  } else if (this.ctx.editor.getText().trim()) {
      // Esc with typed text clears the draft and resets the double-Esc window
      // (omp input-controller.ts:318-323). Purely additive: this state was a no-op.
      this.ctx.editor.setText("");
      this.ctx.ui.requestRender();
      this.ctx.lastEscapeTime = 0;
  } else {
      // ...existing empty-editor double-Esc (tree/branch) body, unchanged...
  }
  ```
- `this.ctx.ui.requestRender()` is verified present (jwc `interactive-mode.ts` uses it widely).
- **No conflict** with jwc's richer branches above (loadingAnimation / steerConsumePending /
  isBashRunning / isBashMode / isEvalRunning / isPythonMode / isStreaming-steering) тАФ those all
  precede and `return`/handle first; the new branch only catches the previously-unhandled
  idle+text state. Double-Esc exit chord is decoupled (`onForcedExit`/`onExitPending`), untouched.

### 2. ast-edit status whitespace collapse тАФ `packages/coding-agent/src/tools/ast-edit.ts`
