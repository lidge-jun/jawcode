## Source — two surfaces

### 1. Esc draft-clear — `packages/coding-agent/src/modes/controllers/input-controller.ts`

omp source (`input-controller.ts:318-335`): when Esc fires with **typed text** (not streaming/
python/bash), the draft is cleared and the double-Esc timer reset, **before** any empty-editor
double-Esc action. jwc `onEscape` currently has only `else if (!this.ctx.editor.getText().trim())`
(empty-editor double-Esc tree/branch); the **text-present case is an unhandled no-op**.

- **MODIFY** the tail branch (~L155). Split the single empty-editor branch into:
  ```ts
  } else if (this.ctx.editor.getText().trim()) {
      // Esc with typed text clears the draft and resets the double-Esc window
      // (omp input-controller.ts:318-323). Purely additive: this state was a no-op.
      this.ctx.editor.setText("");
      this.ctx.ui.requestRender();
      this.ctx.lastEscapeTime = 0;
  } else {
      // ...existing empty-editor double-Esc (tree/branch) body, unchanged...
  }
  ```
- `this.ctx.ui.requestRender()` is verified present (jwc `interactive-mode.ts` uses it widely).
- **No conflict** with jwc's richer branches above (loadingAnimation / steerConsumePending /
  isBashRunning / isBashMode / isEvalRunning / isPythonMode / isStreaming-steering) — those all
  precede and `return`/handle first; the new branch only catches the previously-unhandled
  idle+text state. Double-Esc exit chord is decoupled (`onForcedExit`/`onExitPending`), untouched.

### 2. ast-edit status whitespace collapse — `packages/coding-agent/src/tools/ast-edit.ts`

omp `3d646d8` collapses whitespace runs in the AST pattern echoed into the status line. jwc has
structurally diverged (zod/v4 vs arktype, different helpers), so this is **not** a cherry-pick — it
is the same intent applied to jwc's own `renderCall` (L431): when `rewriteCount === 1` the
status `description` is the raw `args.ops[0].pat`, so a multi-line / multi-space pattern renders
ugly. Minimal faithful adapt:

- **MODIFY** L431: `args.ops?.[0]?.pat` → `args.ops?.[0]?.pat?.replace(/\s+/g, " ").trim()`.

### 3. selector `resetDisplay` — DEFER (evidence-backed)

omp calls `this.ctx.ui.resetDisplay()` on tree/message-selector entry. jwc has **no
`ui.resetDisplay()`** API (`grep -rn 'resetDisplay' packages/coding-agent/src` → 0 hits; jwc ui
exposes `requestRender`, `compactViewportFill`). Per the Decision ("add a narrowly scoped secondary
TUI API surface **or** explicitly defer that subpart with evidence"), this micro-card defers it:
adding a display-reset API is a broader TUI-render change with its own risk, out of an
esc_and_ast_only micro bundle. Recorded as intentionally deferred (Decision F monitored).
